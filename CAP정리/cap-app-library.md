# cap-app/library (common.lib) 문서

## 1. 개요

`common.lib` 라이브러리를 정의하는 `cap-app/library` 모듈은 이 프로젝트의 여러 프론트엔드 애플리케이션에서 공유되는 재사용 가능한 UI5 라이브러리입니다. 그 목적은 공통 UI 구성 요소, 로직 및 유틸리티를 중앙 집중화하여 코드 중복을 피하고 일관된 사용자 경험을 보장하는 것입니다.

이 라이브러리에서 제공하는 주요 기능은 다음과 같습니다:
- 다른 모든 애플리케이션 컨트롤러가 확장해야 하는 `BaseController`.
- 강력하고 재사용 가능한 사용자 정의 UI 컨트롤 모음.
- 공통 작업을 위한 유틸리티 클래스 및 함수 집합.
- 공통 데이터 모델(사용자, 공통 코드, 메뉴 등)의 중앙 집중식 로딩.

## 2. 핵심 구성 요소

### 2.1. `BaseController.js`

이것은 라이브러리에서 가장 중요한 구성 요소 중 하나입니다. 애플리케이션 컨트롤러는 이 `BaseController`를 상속하여 광범위한 내장 기능에 액세스할 수 있습니다.

**주요 특징:**

- **자동 모델 로딩**: `onInit` 메소드에서 여러 필수 모델을 자동으로 로드하고 `sap.ui.getCore()`를 통해 전역적으로, 그리고 뷰에 로컬로 사용할 수 있도록 합니다. 여기에는 다음이 포함됩니다:
    - `User`: 현재 로그인한 사용자에 대한 정보.
    - `CommonCode`: 드롭다운 및 값 도움말을 위한 시스템 전체 코드 목록.
    - `Menu`: 애플리케이션의 탐색 메뉴 구조.
    - `Message`: 시스템 메시지.
    - `MenuFunction`: 특정 기능 또는 권한에 대한 정보.
- **Promise 기반 OData 도우미**: 표준 OData `read` 메소드를 `Promise`로 래핑하는 `readOdata` 함수를 제공하여 비동기 데이터 가져오기를 단순화합니다.
- **테이블 관리 유틸리티**: 테이블 정렬/필터링 재설정(`resetTableSettings`), 테이블 개인 설정 관리, 셀 선택/복사 활성화와 같은 기능을 포함합니다.
- **유틸리티 함수**: `getResourceBundle` 및 `fnDeepCopy`와 같은 공통 도우미를 제공합니다.

이 `BaseController`를 사용함으로써 개별 애플리케이션은 상용구 코드를 크게 줄일 수 있습니다.

### 2.2. 사용자 정의 컨트롤 (예: `CodeValueHelp.js`)

라이브러리에는 향상된 기능을 제공하기 위한 사용자 정의 UI5 컨트롤이 포함되어 있습니다. 대표적인 예가 `CodeValueHelp`입니다.

**`CodeValueHelp` 컨트롤:**

이것은 고도로 구성 가능하고 재사용 가능한 값 도움말 대화 상자 컨트롤입니다.
- 단일 및 다중 선택 시나리오 모두에 사용할 수 있습니다.
- OData 서비스에 직접 연결하여 데이터를 가져오고 표시합니다.
- 내장된 검색 및 필터링 기능을 제공합니다.
- 사용자가 목록에서 값을 선택해야 하는 양식 및 테이블에 쉽게 통합되도록 설계되었습니다.

### 2.3. 유틸리티 클래스

라이브러리는 공통 개발 작업을 지원하기 위해 여러 유틸리티 클래스(`src/util/`에 위치)를 제공합니다.

**`UI5Util.js`:**
다음을 위한 도우미 메소드가 있는 정적 클래스:
- 특정 유형의 부모 컨트롤 찾기(`findParentAs`).
- 테이블 정렬 및 필터링 재설정(`resetSortedAndFiltered`).
- `sap.m.Input`(`value`) 및 `sap.m.Select`(`selectedKey`)와 같은 컨트롤 간의 차이점을 추상화하여 컨트롤의 값을 가져옵니다.

**로더 (예: `UserSessionLoader.js`, `CommonCodeLoader.js`):**
이 클래스들은 `BaseController`에서 사용하는 핵심 데이터 모델을 가져오는 역할을 합니다. 이벤트 기반 접근 방식(`attachEventOnce('ready', ...)`으로 설계되어 데이터가 한 번만 로드된 다음 필요한 애플리케이션의 모든 부분에서 사용할 수 있도록 보장합니다.

## 3. 결론

`common.lib` 라이브러리는 프론트엔드 아키텍처의 기반입니다. 코드 재사용, 일관성 및 관심사 분리를 촉진합니다. 견고한 `BaseController`, 재사용 가능한 컨트롤 및 공통 유틸리티를 제공함으로써 개발자는 반복적인 상용구 작업 대신 개별 애플리케이션의 특정 비즈니스 로직에 집중할 수 있습니다.
