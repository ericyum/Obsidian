# cap-app/portal 문서

## 1. 개요

`cap-app/portal` 애플리케이션(`comportal`)은 사용자를 위한 기본 프론트엔드 진입점입니다. 동적 "셸" 또는 "런치패드" 역할을 합니다. 주요 책임은 전체 페이지 구조(헤더, 탐색, 콘텐츠 영역)를 제공하고 로그인한 사용자의 권한을 기반으로 탐색 메뉴를 동적으로 구축하는 것입니다.

정적 페이지 세트를 갖는 대신, 포털 애플리케이션은 백엔드에서 사용자의 승인된 메뉴 항목을 읽은 다음 즉석에서 UI5 라우터를 구성하여 다른 별도의 UI5 애플리케이션으로의 탐색을 허용합니다.

## 2. 핵심 로직 (`Component.js`)

가장 중요한 로직은 `Component.js`에 있습니다.

### 2.1. 초기화 (`init`)

초기화 시 구성 요소는 두 가지 주요 비동기 작업을 수행합니다.

1.  **사용자 개인화**: `common.lib` 라이브러리의 `UserSessionLoader`를 사용하여 현재 사용자의 프로필을 가져옵니다. 검색되면 언어, 날짜/숫자 형식 및 테마에 대한 전역 UI5 설정을 구성하여 UI가 사용자에게 개인화되도록 합니다.
    ```javascript
    new UserSessionLoader().attachEventOnce('ready', function(oEvent){
        let oUserData = oEvent.getParameter("model").getData();
        // 언어, 테마 및 형식 설정
        sap.ui.getCore().getConfiguration().setLanguage(oUserData.language_code || "EN");
        Theming.setTheme(oUserData.theme_code || "sap_fiori_3");
    });
    ```
2.  **동적 라우팅 설정**: `common.lib`의 `MenuLoader`를 사용하여 사용자의 탐색 메뉴를 가져옵니다. 이것은 애플리케이션의 탐색이 정의되는 중요한 단계입니다.

### 2.2. 동적 경로 생성 (`_setRouteAndTarget`)

이 함수는 메뉴 데이터가 로드된 후 호출됩니다. 메뉴 항목을 반복하고 라우터를 동적으로 구성합니다.

-   **메뉴의 각 애플리케이션에 대해** 새로운 **Route**와 새로운 **Target**을 생성합니다.
-   **Target**은 `Component` 유형으로, 포털이 기본 콘텐츠 영역 내에 다른 전체 UI5 애플리케이션을 포함하고 표시할 수 있도록 합니다.
-   또한 포털의 매니페스트에 `componentUsage` 항목을 동적으로 추가합니다. 이것은 외부 구성 요소를 로드하기 위한 UI5 요구 사항입니다.
-   모든 동적 경로를 생성한 후 자체 페이지(예: "홈" 대시보드 및 "검색" 페이지)에 대한 몇 가지 정적 경로를 추가합니다.
-   마지막으로 이 완전하고 동적으로 생성된 구성으로 라우터를 초기화합니다.

이 아키텍처는 포털을 매우 유연하게 만듭니다. 사용자 보기에 새 애플리케이션을 추가하려면 백엔드 데이터베이스의 `Menu` 테이블에 새 항목을 추가하기만 하면 됩니다. 포털 자체에는 프론트엔드 배포가 필요하지 않습니다.

## 3. UI 구조

### 3.1. 기본 레이아웃 (`Layout.view.xml`)

기본 UI는 표준 애플리케이션 레이아웃을 제공하는 `sap.tnt.ToolPage` 컨트롤을 사용하여 빌드됩니다.
-   **헤더**: 회사 로고, 전역 메뉴 검색 필드, 사용자 정보 및 로그아웃/프로필 버튼을 포함합니다.
-   **메인 콘텐츠 영역**: 이 영역에는 `<sap.m.App>` 컨트롤이 포함되어 있습니다. 이 컨트롤은 라우터가 포털 자체의 "홈" 뷰이든 외부 애플리케이션 전체가 구성 요소로 로드되든 모든 다른 뷰와 구성 요소를 배치하는 컨테이너 역할을 합니다.

### 3.2. 홈페이지 (`Home.view.xml`)

기본 페이지(`pattern: ""`)는 `Home` 뷰입니다. 이 뷰는 대시보드 역할을 하여 다음을 표시합니다.
-   `GenericTile` 컨트롤을 사용하는 핵심 성과 지표(KPI).
-   중요한 마스터 데이터 관리 메뉴에 대한 빠른 링크.
-   데이터 추세를 시각화하기 위한 차트(`VizFrame`).

## 4. 결론

`portal` 애플리케이션은 프론트엔드의 중앙 허브입니다. 단순한 웹 페이지가 아니라 사용자 권한을 기반으로 런타임에 자체 탐색 기능을 구성하는 정교하고 데이터 기반의 셸입니다. 일관된 모양과 느낌을 제공하며 프로젝트 범위 내의 다른 모든 비즈니스 애플리케이션을 위한 런치패드 역할을 합니다.
