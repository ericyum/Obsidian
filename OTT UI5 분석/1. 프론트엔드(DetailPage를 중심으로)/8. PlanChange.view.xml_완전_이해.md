## 개요: 이 파일의 정체

`PlanChange.view.xml`은 **구독할 플랜을 선택하는 카드 그리드 화면**이다. MyMembership 페이지에서 "플랜 변경" 또는 "구독 시작" 버튼을 누르면 이 화면으로 이동한다. `sap.uxap.ObjectPageLayout`을 루트로 사용하며 (7/15 Page에서 전환), `FlexBox` aggregation binding으로 플랜 카드 목록을 동적 생성한다.

**연결된 컨트롤러**: `myPage.controller.PlanChange`
**사용 모델**: `viewModel` (JSONModel), `myPage` (OData V2), `i18n` (ResourceModel)
**루트 레이아웃**: `sap.m.Page` (ObjectPageLayout ❌)
**라우트**: `#/PlanChange`

## 1. 전체 구조 (7/15 ObjectPageLayout 기준)

```
uxap:ObjectPageLayout
├── uxap:headerTitle
│   └── uxap:ObjectPageDynamicHeaderTitle
│       ├── uxap:heading → Title ("플랜 선택", H2)
│       └── uxap:actions → 뒤로가기 버튼
│
└── uxap:sections
    └── uxap:ObjectPageSection (showTitle="false")
        └── VBox (중앙 정렬)
            ├── FlexBox (items=PlanList, wrap="Wrap") ← 카드 그리드
            │   └── VBox × N ← 각 플랜 카드
            │       └── planCard (흰색 둥근 CSS 카드)
            │           ├── Title (플랜명)
            │           ├── Text (label: Monthly Fee)
            │           ├── Text (가격)
            │           ├── Text (label: Simultaneous Streams)
            │           ├── Text (동시시청 값)
            │           ├── Button "현재 플랜" (disabled)
            │           └── Button "선택"/"선택됨"
            │
            └── Button "구독 시작"/"플랜 변경" (planSubmitBtn)
```

## 3. FlexBox Aggregation Binding — 카드 그리드

```xml
<FlexBox
    items="{
        path: 'viewModel>/PlanList',
        sorter: { path: 'price' }
    }"
    justifyContent="Center"
    wrap="Wrap"
    class="sapUiLargeMarginBottom">
```

### 3.1 Aggregation Binding vs 순수 레이아웃

| 용법 | 코드 | 동작 |
|------|------|------|
| **Aggregation Binding** | `items="{viewModel>/PlanList}"` | 배열 길이만큼 자식 템플릿을 **복제** (for문) |
| **순수 레이아웃** | `justifyContent="Center"` | 자식 정렬 컨테이너 |

> 이 화면에서는 두 가지 용법을 **동시에** 사용한다: `items`로 플랜 수만큼 카드를 찍어내고, `justifyContent`와 `wrap`으로 레이아웃을 제어한다.

### 3.2 속성 설명

| 속성 | 값 | 의미 |
|------|------|------|
| `items` | `{path: 'viewModel>/PlanList', sorter: {path: 'price'}}` | PlanList 배열을 가격순 정렬해서 각 요소마다 자식 템플릿 생성 |
| `justifyContent` | `Center` | 카드가 화면 중앙에 정렬 |
| `wrap` | `Wrap` | 화면 너비가 줄면 카드가 자동으로 다음 줄로 넘어감 (enum 타입!) |

> **⚠️ `wrap="true"`가 아니다!** `FlexBox.wrap`은 열거형(enum) 속성으로 `"Wrap"` (대문자) 문자열을 써야 한다. `"true"`로 잘못 쓰면 동작하지 않는다. (6/26 디버깅에서 겪은 교훈)

### 3.3 카드 흐름 (7/15 CSS 스타일링)

```
PlanList: [{code:"BASIC", name:"Basic", price:9900, maxStreams:2}, {...}, {...}]

FlexBox items=PlanList
  ├─ VBox → planCard (흰색 배경, border-radius, 그림자, padding)
  │   ├─ Title (플랜명)
  │   ├─ Text "Monthly Fee" (회색 .planCardLabel)
  │   ├─ Text "9,900원/월" (formatPrice)
  │   ├─ Text "Simultaneous Streams" (회색 .planCardLabel)
  │   ├─ Text "2명" (formatMaxStreamsValue)
  │   ├─ Button "현재 플랜" (disabled) / "선택" (Default) / "선택됨" (Emphasized)
```

> **7/15 변경**: `backgroundDesign="Solid"` → CSS `.planCard` 클래스로 대체. 라벨(Monthly Fee, Simultaneous Streams)과 값 분리. `formatMaxStreams`(풀문장) → `formatMaxStreamsValue`(값만)로 변경.

## 4. `CustomData` — 클릭 이벤트에 planCode 전달하기

```xml
<VBox class="sapUiMediumMargin">
    <customData>
        <core:CustomData key="planCode" value="{viewModel>code}" />
    </customData>
    ...
</VBox>
```

### 4.1 왜 필요한가

플랜 카드가 클릭되었을 때 **어떤 플랜이 선택되었는지** 컨트롤러에 알려줘야 한다. 그런데 Expression Binding만으로는 `onSelectPlan` 함수에 planCode를 직접 전달할 수 없다.

**해결**: `CustomData`로 DOM 요소에 데이터를 숨겨두고, 컨트롤러에서 `oEvent.getSource().getCustomData()[0].getValue()`로 꺼낸다.

```
사용자가 "STANDARD" 카드의 선택 버튼 클릭
  → oEvent.getSource() = 클릭된 Button
  → getCustomData() = [{key: "planCode", value: "STANDARD"}]
  → getValue() = "STANDARD"
```

### 4.2 두 군데에 `CustomData`가 있다

| 위치 | 목적 |
|------|------|
| 카드 VBox 최상단 | _백업용. 실제로는 버튼 쪽에서 읽음_ |
| 선택 버튼 | `onSelectPlan`에서 읽는 실제 planCode |

## 5. 플랜 카드 내부 구조 (7/15 개선)

```xml
<VBox class="planCard sapUiSmallMarginBottom">
    <Title text="{viewModel>name}" level="H3" textAlign="Center" />

    <Text text="{i18n>label.monthlyFee}" textAlign="Center" class="planCardLabel" />
    <Text text="{path: 'viewModel>price', formatter: '.formatPrice'}" textAlign="Center" />

    <Text text="{i18n>label.maxStreams}" textAlign="Center" class="planCardLabel" />
    <Text text="{path: 'viewModel>maxStreams', formatter: '.formatMaxStreamsValue'}" textAlign="Center" />

    <Button text="{i18n>label.currentPlan}" enabled="false"
        visible="{= ${viewModel>code} === ${viewModel>/Membership/subscription_plan} }" />
    <Button
        text="{= ${viewModel>code} === ${viewModel>/SelectedPlan} ? ${i18n>button.selected} : ${i18n>button.select} }"
        type="{= ${viewModel>code} === ${viewModel>/SelectedPlan} ? 'Emphasized' : 'Default' }"
        visible="{= ${viewModel>code} !== ${viewModel>/Membership/subscription_plan} }"
        press="onSelectPlan">
        <customData>
            <core:CustomData key="planCode" value="{viewModel>code}" />
        </customData>
    </Button>
</VBox>
```

> **7/15 변경**: 라벨/값 분리 (`{i18n>label.monthlyFee}` + `formatPrice`), 회색 라벨(`.planCardLabel`), 카드 패딩 1.5rem 2rem, 버튼 둥글게 확대.

### 5.1 두 개의 버튼 — 상호 배타적 visible

```
현재 구독 중인 플랜:        [현재 플랜] (disabled, 회색)
다른 플랜 (아직 선택 안 함):  [선택] (Default)
다른 플랜 (이미 선택함):      [선택됨] (Emphasized, 파란색)
```

### 5.2 Expression Binding 3쌍

| 속성 | Expression | 설명 |
|------|-----------|------|
| `text` | `code === SelectedPlan ? "선택됨" : "선택"` | 선택 상태에 따라 버튼 텍스트 변경 |
| `type` | `code === SelectedPlan ? 'Emphasized' : 'Default'` | 선택된 플랜 강조 (파란색) |
| `visible` | `code !== subscription_plan` | 현재 구독 중인 플랜은 선택 버튼 숨김 |

### 5.3 `backgroundDesign="Solid"` 

카드에 흰색 배경 + 미세한 그림자를 줘서 카드 느낌을 만든다. `sap.f.Card`를 쓸 수 없는 UI5 1.136 CDN 제약을 CSS 없이 순수 속성만으로 우회했다.

| 속성 | 설명 |

## 8. 핵심 암기 사항

| # | 내용 |
|---|------|
| 1 | `Page`는 `ObjectPageLayout` 없이 독립 네비게이션이 가능한 페이지 |
| 2 | `FlexBox items=배열` = Aggregation Binding = for문처럼 템플릿 복제 |
| 3 | `FlexBox wrap="Wrap"` — enum 타입. `"true"`로 쓰면 안 됨! |
| 4 | `CustomData` = DOM 요소에 데이터 숨기기. `getCustomData()[0].getValue()`로 꺼냄 |
| 5 | Expression Binding으로 버튼 text·type·visible 삼중 동적 제어 |
| 6 | 두 버튼을 `visible`로 상호 배타적 전환 (현재 플랜 vs 다른 플랜) |
| 7 | `backgroundDesign="Solid"` = 순수 속성만으로 카드 스타일 |
| 8 | 양쪽 `ToolbarSpacer`로 버튼 중앙 정렬 |
| 9 | `sorter: {path: 'price'}` — Aggregation Binding에서 가격순 정렬 |
| 10 | `Page.showNavButton` + `navButtonPress` = 표준 네비게이션 패턴 |
