## 개요: 이 파일의 정체

`MyMembership.controller.js`는 **사용자의 멤버십(구독) 정보를 로드하고, 해지·재개·플랜 변경을 처리하는 컨트롤러**다. detailPage의 DetailMain과 동일한 상속 구조를 가지며, `MessageBox.warning`을 통해 Fiori 표준 확인 다이얼로그를 제공한다.

**상속**: `MyMembership` → `myPage.BaseController` (껍데기) → `library_ott.BaseController` (진짜 로직)
**연결된 뷰**: `myPage.view.MyMembership`
**사용 모델**: `viewModel` (JSONModel), `myPage` (OData V2), `User` (Core)
**라우트**: `#/MyMembership`

onInit: function () {

## 4. 데이터 로딩 — `_loadMembership`

```javascript
_loadMembership: function () {
    var oModel = this.getView().getModel("myPage") || this.getOwnerComponent().getModel("myPage");
    var oViewModel = this.getOwnerComponent().getModel("viewModel");
    var oView = this.getView();

    // ★ 7/7 중첩 컴포넌트로 로드 시 모델이 없으면 수동 생성
    if (!oModel) {
        oModel = new sap.ui.model.odata.v2.ODataModel("/srv-api/odata/v2/ott/Mypage", {
            useBatch: false
        });
        this.getView().setModel(oModel, "myPage");
    }

    oView.setBusy(true);

    this.readOdata({
        model: oModel,
        path: "/UserSet",
        param: {
            urlParameters: { "$expand": "plan" }
        }
    }).then(function (oData) {
        var oMembership = oData.results && oData.results.length > 0 ? oData.results[0] : null;
        oViewModel.setProperty("/Membership", oMembership);
        oView.setBusy(false);
    }).catch(function () {
        oView.setBusy(false);
        MessageToast.show(this.i18n.getText("message.membershipLoadFail"));
    });
},
```

### 4.1 DetailMain과 비교

| | DetailMain `_loadDetail` | MyMembership `_loadMembership` |
|---|---|---|
| OData 경로 | `/Contents('C001')` | `/UserSet` |
| $expand | 5개 (partner, genres, tags, casts, reviews) | 1개 (plan) |
| Token 패턴 | 있음 (race condition 방지) | 없음 (단일 멤버십, 빠른 전환 없음) |
| 결과 처리 | 첫 번째 요소 → `/Detail` | 전체 results[0] → `/Membership` |
| 추가 작업 | `_checkMyReview()` 호출 | 없음 |

### 4.2 왜 Token 패턴이 없나

detailPage는 사용자가 C001 → C002 → C003 같이 **연속으로 다른 리소스를 조회**할 수 있어서 race condition 방지가 필요했다. MyMembership은 항상 같은 `/UserSet`을 조회하고, URL 파라미터도 없어서 **연속된 중복 호출이 발생할 이유가 없다**.

### 4.3 OData 모델이 "myPage"

`manifest.json`에서 `myPage`라는 이름으로 OData V2 모델이 선언되어 있다:

```json
"myPage": {
    "type": "sap.ui.model.odata.v2.ODataModel",
    "settings": { "useBatch": false },
    "dataSource": "mainService",
    "preload": true
}
```

> **detailPage는 "detail"**, **myPage는 "myPage"** — 같은 `_ott` 접미사 패턴으로 네임스페이스 분리.

## 5. 구독 해지·재개 — `MessageBox.warning` 표준화

### 5.1 전체 흐름

```
onCancelConfirm() / onResumeConfirm()
  ↓
_showConfirmDialog(action, title, message)
  ↓  MessageBox.warning (Fiori 표준)
  ↓  사용자 OK 클릭
  ↓
_callAction("cancel") / _callAction("resume")
  ↓  callFunction("/cancel") 또는 callFunction("/resume")
  ↓
_loadMembership() — 서버 재조회
```

### 5.2 `_showConfirmDialog` — 공통 다이얼로그

```javascript
_showConfirmDialog: function (sAction, sTitle, sMessage) {
    var that = this;
    MessageBox.warning(sMessage, {
        title: sTitle,
        actions: [MessageBox.Action.OK, MessageBox.Action.CANCEL],
        emphasizedAction: MessageBox.Action.OK,
        contentWidth: "30rem",
        onClose: function (sResult) {
            if (that._bDestroyed) return;           // ★ 파괴 가드
            if (sResult === MessageBox.Action.OK) {
                that._callAction(sAction);           // OK일 때만 액션 실행
            }
        }
    });
},
```

### 5.3 `MessageBox.warning` 속성 설명

| 속성 | 값 | 설명 |
|------|------|------|
| `actions` | `[OK, CANCEL]` | 두 개의 버튼 |
| `emphasizedAction` | `OK` | OK 버튼을 파란색(Emphasized)으로 강조 |
| `contentWidth` | `"30rem"` | 다이얼로그 폭 고정 (영문 긴 문장 방지) |
| `onClose` | 콜백 | **모든 닫힘 동작**(OK, CANCEL, X)에서 실행 |

> **⚠️ `emphasizedAction`은 시각적 강조만 할 뿐, `onClose`가 OK일 때만 로직이 실행된다.** 사용자가 CANCEL이나 X를 누르면 `onClose`는 실행되지만 `sResult !== OK`라서 `_callAction`이 호출되지 않는다.

### 5.4 i18n 키 구조

```properties
# 다이얼로그 제목
dialog.cancelTitle=구독 해지
dialog.resumeTitle=구독 재개

# 다이얼로그 본문
dialog.cancelBody=구독을 해지하시겠어요? 해지 후에도 구독 기간이 끝나기 전까지는 서비스를 계속 이용하실 수 있습니다.
dialog.resumeBody=구독을 재개하시겠어요? 구독이 재개되면 다음 결제 주기부터 요금이 청구됩니다.
```

제목과 본문을 분리한 이유: 제목은 짧게, 본문에서 상세 설명을 제공. 이전에는 제목 하나에 모든 텍스트를 담아서 잘리는 문제가 있었다 (7/2 리팩토링).

## 6. `_callAction` — 백엔드 호출

```javascript
_callAction: function (sAction) {
    var oModel = this.getView().getModel("myPage");
    var that = this;

    oModel.callFunction("/" + sAction, {
        method: "POST",
        success: function () {
            if (that._bDestroyed) return;
            MessageToast.show(
                sAction === "cancel"
                    ? that.i18n.getText("message.cancelSuccess")
                    : that.i18n.getText("message.resumeSuccess")
            );
            that._loadMembership();  // 서버 재조회
        },
        error: function () {
            if (that._bDestroyed) return;
            MessageToast.show(that.i18n.getText("message.processError"));
        }
    });
},
```

### 6.1 `callFunction` — DetailMain과 동일한 V2 콜백 패턴

```javascript
oModel.callFunction("/cancel", {
    method: "POST",
    success: function () { ... },  // ← Promise 안 됨. 반드시 콜백
    error: function () { ... }
});
```

### 6.2 성공 후 `_loadMembership()` 재조회

DetailMain의 `_loadDetail(contentId)`와 같은 원리. 액션 성공 후 **서버에서 전체 멤버십 데이터를 다시 가져온다**.

### 6.3 모든 콜백에서 `_bDestroyed` 확인

```javascript
if (that._bDestroyed) return;
```

사용자가 다이얼로그를 띄운 상태에서 페이지를 이탈할 경우, 응답 도착 시 파괴된 컨트롤러를 조작하지 않도록 방어.

## 7. Info Popover — 동시시청·기간 설명 (7/15 신규)

```javascript
onInfoMaxStreams: function (oEvent) {
    this._showInfoPopover(oEvent,
        this.i18n.getText("info.maxStreamsTitle"),
        this.i18n.getText("info.maxStreamsBody"));
},

onInfoPeriod: function (oEvent) {
    this._showInfoPopover(oEvent,
        this.i18n.getText("info.periodTitle"),
        this.i18n.getText("info.periodBody"));
},

_showInfoPopover: function (oEvent, sTitle, sBody) {
    var oLayout = new VerticalLayout({ width: "100%" });
    oLayout.addStyleClass("sapUiContentPadding");
    oLayout.addContent(new Text({ text: sBody }));

    if (!this._oInfoPopover) {
        this._oInfoPopover = new Popover({
            title: sTitle,
            content: [oLayout],
            contentWidth: "22rem"
        });
    } else {
        this._oInfoPopover.setTitle(sTitle);
        this._oInfoPopover.removeAllContent();
        this._oInfoPopover.addContent(oLayout);
    }
    this._oInfoPopover.openBy(oEvent.getSource());
},
```

> **7/15 신규**: 동시시청·구독기간 옆 ⓘ 아이콘 클릭 시 Popover로 설명 표시. `VerticalLayout` + `sapUiContentPadding` 필수 (Text만 넣으면 패딩 없음). `addStyleClass`는 메서드, 생성자 속성 아님.

## 9. 방어적 패턴

| 패턴 | 코드 | 목적 |
|------|------|------|
| `that = this` | 콜백 내 `this` 보존 | MessageBox.onClose, callFunction 콜백 |
| `_bDestroyed` | `if (that._bDestroyed) return;` | 페이지 이탈 후 안전 중단 |
| MessageBox 가드 | `if (sResult === MessageBox.Action.OK)` | OK 눌렀을 때만 액션 실행 |

> **DetailMain보다 방어적 패턴이 적은 이유**: 연속 클릭 race condition이 없고, 다이얼로그 자체가 중복 실행을 막아주기 때문.
