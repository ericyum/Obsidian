## 피드백 #1 — 상단 배너 '기본 정보', '출연진', '리뷰' 클릭 이동 원리

### 현재 상태
detailPage 상단에 `ObjectPageLayout`의 anchor bar가 표시되고, 각 섹션 타이틀(기본 정보, 출연진, 리뷰, 리뷰 목록, 내 리뷰)이 버튼 형태로 나타난다. 클릭하면 해당 섹션으로 스크롤 이동한다.

### 원리 분석
- **담당 라이브러리**: `sap.uxap.ObjectPageLayout`
- **동작 방식**: ObjectPageLayout에 `ObjectPageSection`을 추가하면, 각 Section의 `title` 속성이 자동으로 anchor bar에 버튼으로 표시된다. 클릭 시 브라우저의 scrollIntoView 방식으로 해당 섹션으로 이동한다.
- **추가 설정**:
  - `upperCaseAnchorBar="false"` → 버튼 텍스트 대문자 변환 비활성화
  - `toggleHeaderOnTitleClick="true"` → 헤더 타이틀 클릭 시 헤더 확장/축소
- **개발자가 한 일**: 없음. 완전히 ObjectPageLayout의 **내장 기능**이다.

### 결론
별도 수정 불필요. 이미 정상 동작 중이며 SAP 표준 동작이다.

## 피드백 #2 — 장르 파란 글자가 링크처럼 보이는 문제

### 현재 상태
```xml
<!-- DetailMain.view.xml -->
<ObjectStatus
    text="{viewModel>genre/name}"
    state="Information"          ← ⚠ 이게 파란색 원인
    icon="sap-icon://tag"
    class="sapUiTinyMarginEnd" />
```

### 원인 분석
- `sap.m.ObjectStatus` 컨트롤의 `state="Information"` 속성은 **파란색 텍스트**를 렌더링한다
- 일반적으로 상태 표시용(Information/Success/Warning/Error)이지만, 장르 태그에는 의미상 맞지 않음
- 사용자 입장에서 파란 글자 = 클릭 가능한 링크로 오인

### 수정 방안
**방안 A: `state="None"`** (가장 간단)
```xml
<ObjectStatus
    text="{viewModel>genre/name}"
    state="None"
    icon="sap-icon://tag"
    class="sapUiTinyMarginEnd" />
```
→ 텍스트 기본 색상(검정)으로 변경, 링크 오인 해소

**방안 B: ObjectStatus 대신 Text + Icon** (완전 분리)
```xml
<HBox>
    <core:Icon src="sap-icon://tag" size="0.875rem" />
    <Text text="{viewModel>genre/name}" class="sapUiTinyMarginBegin" />
</HBox>
```

### 추천: 방안 A
ObjectStatus의 아이콘+텍스트 조합 UI는 유지하되, 색상만 변경.

| 파일 | 변경 위치 |
|------|-----------|
| `DetailMain.view.xml` | 장르 ObjectStatus의 `state="Information"` → `state="None"` |

## 피드백 #3 — 출연진 표 행이 너무 많음

### 현재 상태
```xml
<!-- DetailMain.view.xml -->
<t:Table
    id="castTable"
    rows="{viewModel>/Detail/casts/results}"
    enableSelectAll="false"
    selectionMode="None"
    enableColumnFreeze="true"
    width="auto">
    <!-- visibleRowCount 설정 안 됨 ⚠ -->
```

### 원인 분석
- `sap.ui.table.Table`은 `visibleRowCount` 속성으로 **한 번에 보여줄 행 수**를 제한할 수 있다
- 현재 이 속성이 설정되지 않아 **모든 행이 한 번에 렌더링**됨
- 출연진 수가 많으면(예: 50명 이상) 테이블이 끝없이 길어짐 → 스크롤 문제

### `visibleRowCount` 속성 설명
| 속성 | 기본값 | 설명 |
|------|:------:|------|
| `visibleRowCount` | 10 | 테이블에 표시될 행 개수. 이를 초과하면 **세로 스크롤** 생성 |
| `minAutoRowCount` | 5 | `visibleRowCount="auto"`일 때 최소 행 수 |
| `rowActionCount` | 1 | 행 작업 버튼 개수 |

> 참고: `visibleRowCount`를 설정하면 테이블 자체에 스크롤바가 생기고, 테이블 높이가 고정된다.
> 또한 `visibleRowCount`는 **expression binding이 가능**하므로 데이터 길이에 따라 동적으로 행 수를 조절할 수 있다.

### 수정 방안

**방안 A: 데이터 개수만큼 + 최대 15행 제한** ← 추천
```xml
<t:Table
    id="castTable"
    rows="{viewModel>/Detail/casts/results}"
    visibleRowCount="{= Math.min(${viewModel>/Detail/casts/results}.length, 15) }"
    enableSelectAll="false"
    selectionMode="None"
    enableColumnFreeze="true"
    width="auto">
```
> 출연진 3명 → 3행, 15명 → 15행, 50명 → 15행(+스크롤). 적으면 군더더기 없고 많으면 제한.

**방안 B: 고정 10행** (가장 단순)
```xml
visibleRowCount="10"
```
> 데이터가 3건이어도 빈 공간 7행이 남음 → "왜 이렇게 비었지?" 의문 유발 가능.

**방안 C: 컨트롤러에서 동적 설정** (완전 제어)
```javascript
// DetailMain.controller.js _loadDetail() 성공 후
var aCasts = oData.casts && oData.casts.results;
var nCount = aCasts ? aCasts.length : 0;
this.byId("castTable").setVisibleRowCount(Math.min(nCount, 15));
```

| 방안 | 데이터 3개 | 데이터 20개 | 장점 |
|:----:|:---------:|:---------:|------|
| **A (추천)** | 3행 | 15행 | 깔끔 + 자동 |
| B | 10행(7행 공백) | 10행 | 단순 |
| C | 3행 | 15행 | 완전 제어 |

| 파일 | 변경 사항 |
|------|-----------|
| `DetailMain.view.xml` | `visibleRowCount="10"` 속성 추가 |

## 피드백 #4 — 엑셀 파일 통합 또는 버튼 위치 변경

### 현재 상태
헤더 actions 영역에 두 개의 버튼이 있음:
```xml
<uxap:actions>
    <Button icon="sap-icon://excel-attachment" text="기본정보 내보내기" press="onExportInfo" />
    <Button icon="sap-icon://excel-attachment" text="출연진 내보내기"   press="onExportCast" />
    <Button icon="sap-icon://nav-back"          text="뒤로가기"         press="onNavBack" />
</uxap:actions>
```

### 문제점
- "기본정보"와 "출연진"이 각각 따로 엑셀 파일로 다운로드됨
- 버튼이 헤더 actions에 몰려 있어서 해당 섹션과의 연관성이 떨어짐

### 수정 방안

**방안 A: 하나의 엑셀 버튼으로 통합**
- 하나의 버튼 클릭 시 기본정보 + 출연진을 하나의 엑셀 파일에 시트 2개로 저장
- ⚠ **제약**: UI5 1.136 CDN의 `sap.ui.export.Spreadsheet`는 멀티시트 미지원 (과거 확인됨)

**방안 B: 버튼을 각 섹션으로 이동**  ← 추천
- 기본정보 섹션에 "기본정보 내보내기" 버튼 배치
- 출연진 섹션에 "출연진 내보내기" 버튼 배치
- 섹션과 버튼의 연관성 ↑, 사용자 직관성 ↑

**방안 C: 헤더에 "엑셀 전체 다운로드" 하나만 배치**
- zip 파일로 묶어서 한 번에 다운로드 (복잡도 ↑)

### 추천: 방안 B
| 파일 | 변경 사항 |
|------|-----------|
| `DetailMain.view.xml` | actions에서 엑셀 버튼 2개 제거 |
| `DetailMain.view.xml` | 기본정보 섹션 하단에 `onExportInfo` 버튼 추가 |
| `DetailMain.view.xml` | 출연진 섹션 하단에 `onExportCast` 버튼 추가 |

## 피드백 #5 — 리뷰 '수정' '삭제' 버튼 중복

### 현재 상태
리뷰 목록의 **각 항목**에 수정·삭제 버튼이 있고, **"내 리뷰" 섹션**에도 별도 수정·삭제 버튼이 있음:

```
리뷰 목록
├── 리뷰 항목 1: [✏️ 편집] [🗑️ 삭제]     ← 위치 1
├── 리뷰 항목 2:                        (내 리뷰가 아니면 버튼 숨김)

내 리뷰
├── "이미 리뷰를 작성하셨습니다"
└── [리뷰 수정] [삭제]                    ← 위치 2 (중복!)
```

### 원인
"내 리뷰" 섹션과 리뷰 목록 항목의 버튼이 **기능적으로 중복**된다. 사용자 입장에서 "어떤 버튼을 눌러야 하지?" 혼란 발생.

### 수정 방안
"내 리뷰" 섹션에서 버튼을 제거하고, 안내 메시지만 표시:

```xml
<!-- MyReview 섹션: 버튼 제거, 메시지만 -->
<uxap:ObjectPageSubSection
    visible="{= !!${viewModel>/HasReviewed} }"
    title="{i18n>label.myReview}">
    <MessageStrip
        text="{i18n>message.alreadyReviewed}"
        type="Information"
        showIcon="true" />
    <!-- 버튼 제거! -->
</uxap:ObjectPageSubSection>
```

대신 리뷰 목록 항목의 버튼(✏️/🗑️)만으로 수정·삭제를 수행한다. 추가로 "내 리뷰" 항목은 시각적으로 강조(배경색 등)하여 구분.

| 파일 | 변경 사항 |
|------|-----------|
| `DetailMain.view.xml` | "내 리뷰" ObjectPageSubSection에서 버튼 2개 제거 |
| `DetailMain.view.xml` | (선택) 내 리뷰 항목 배경 강조 |

## 피드백 #6 — MyMembership 텍스트 높낮이 어긋남

### 현재 상태
```xml
<form:SimpleForm layout="ResponsiveGridLayout" ...>
    <form:content>
        <VBox>
            <Label text="플랜" design="Bold" />
            <Text text="{viewModel>/Membership/plan/name}" />
            <layoutData>
                <layout:GridData span="L6 M6 S12" />
            </layoutData>
        </VBox>
        <!-- 플랜, 월 요금, 동시시청, 기간 동일 패턴 -->
    </form:content>
</form:SimpleForm>
```

### 원인 분석
- `SimpleForm` + `ResponsiveGridLayout`에서 `Label`과 `Text`는 각각 고유한 height/margin을 가짐
- `Label`의 상하 margin과 `Text`의 padding이 달라서 **두 텍스트의 baseline이 수직으로 어긋남**
- 특히 `Label design="Bold"`는 일반 Text보다 두꺼워서 더 도드라짐

### 근본 원리: ResponsiveGridLayout의 텍스트 정렬
- `ResponsiveGridLayout`은 폼 요소를 2열 그리드로 배치
- 각 `VBox`(form content)는 grid cell 하나를 차지
- `Label` + `Text` 조합을 VBox로 감싸면, Label과 Text 사이의 간격은 균일하나, **다른 VBox와 수평 비교 시 텍스트 높이가 미묘하게 다를 수 있음**
- `Label`은 기본적으로 `margin: 0.25rem 0` 상하 여백이 자동 추가됨

### 수정 방안
CSS로 정렬 보정:

```css
/* detailPage or myPage style.css */
.membershipFormVBox .sapMLabel {
    margin-bottom: 0.125rem;
}
.membershipFormVBox .sapMText {
    margin-top: 0;
    line-height: 1.4;
}
```

또는 각 VBox에 `class="membershipFormVBox"` 추가하여 균일한 정렬 적용.

| 파일 | 변경 사항 |
|------|-----------|
| `myPage/webapp/css/style.css` | 멤버십 폼 정렬 CSS 추가 |
| `MyMembership.view.xml` | 각 VBox에 class 추가 |

## 피드백 #7 — 팝업창 텍스트 짤림 + 아이콘 개선

### 현재 상태
```javascript
// MyMembership.controller.js
_showConfirmDialog: function (sAction, sTitle, sMessage) {
    MessageBox.confirm(sMessage, {
        title: sTitle,
        onClose: function (sResult) { ... }
    });
},
```

i18n 메시지:
```
message.cancelConfirm=구독을 해지하시겠어요?
message.cancelConfirmDetail=재개로 다시 이용할 수 있어요.   ← 짧은데도 혹시 모를 길이 문제
message.resumeConfirm=구독을 재개하시겠어요?
message.resumeConfirmDetail=바로 다시 이용 가능합니다.
```

### 문제점 A: 아이콘이 모두 '?' (QUESTION)
- `MessageBox.confirm()`은 기본 아이콘으로 `sap-icon://question-mark`(?)를 사용
- 해지·재개·구독 시작 등 **성격이 다른 액션이 모두 같은 아이콘** → 진부함, 구분 어려움

### 문제점 B: 텍스트 짤림
- `MessageBox`의 기본 width는 약 20rem(320px)으로 제한됨
- 긴 영문 텍스트의 경우 `text-overflow: ellipsis`로 '...' 처리됨
- 실제 텍스트는 짧은 편이지만, MessageBox 자체의 구조적 한계

### 수정 방안
**`sap.m.Dialog`로 직접 구현** (완전한 제어 가능):

```javascript
_showConfirmDialog: function (sAction, sTitle, sMessage, sIcon) {
    var that = this;
    var oDialog = new Dialog({
        title: sTitle,
        type: "Message",
        icon: sIcon,                    // ← 아이콘 직접 지정
        contentWidth: "auto",           // ← 텍스트에 맞춰 너비 자동 조정
        content: [
            new Text({ text: sMessage })
        ],
        beginButton: new Button({
            text: that.i18n.getText("button.confirm"),
            type: "Emphasized",
            press: function () {
                that._callAction(sAction);
                oDialog.close();
            }
        }),
        endButton: new Button({
            text: that.i18n.getText("button.cancelDialog"),
            press: function () {
                oDialog.close();
            }
        })
    });
    oDialog.open();
},
```

아이콘 매핑:

| 액션 | 아이콘 | 의미 |
|------|--------|------|
| 해지(cancel) | `sap-icon://warning2` | ⚠ 경고 (세모+!) |
| 재개(resume) | `sap-icon://refresh` | 🔄 재시작 |
| 구독 시작(subscribe) | `sap-icon://add-coursebook` | 📘 신규 등록 |

| 파일                            | 변경 사항                                   |
| ----------------------------- | --------------------------------------- |
| `MyMembership.controller.js`  | `MessageBox.confirm` → 커스텀 `Dialog`로 교체 |
| `myPage/i18n/i18n.properties` | 확인/취소 버튼 텍스트 키 추가                       |
| `PlanChange.controller.js`    | `onSubmitPlan`에도 동일 Dialog 패턴 적용 (선택)   |
