## 0. 전체 아키텍처

### 0.1 원칙 — 템플릿 무변경

기존 `cap-app/` 템플릿(approuter, portal, sysmgt 등)은 **일체 수정하지 않는다**. 모든 OTT 전용 모듈은 `_ott` 접미사를 붙인 별도 폴더로 관리한다.

```
cap-app/
├── approuter/         ← 원본 (손 안 댐)
├── portal/            ← 원본 (손 안 댐)
├── sysmgt/            ← 원본 (손 안 댐)
│
├── approuter_ott/     ← [신규] OTT 전용 진입점
├── portal_ott/        ← [신규] OTT 전용 포털
└── sysmgt_ott/        ← [신규] OTT 전용 기능 모듈
    └── webapp/
        ├── detailPage/   ← 기능2
        └── myPage/       ← 기능4
```

### 0.2 사용자 요청 흐름

```
브라우저
  │  approuter_ott 주소 접속
  ▼
approuter_ott
  │  인증 (xsuaa)
  │  URL = (.*) → portal_ott 라우팅
  ▼
portal_ott (Layout.view.xml)
  │  MenuLoader로 DB 메뉴 동적 로딩
  │  상단/사이드 메뉴바 출력
  │
  ├─ "컨텐츠" 클릭 → /sysmgt_ott/webapp/detailPage/
  │   └─ DetailMain.view.xml (기능2)
  │
  └─ "마이페이지" 클릭 → /sysmgt_ott/webapp/myPage/
      └─ MyMembership.view.xml (기능4)
```

### 0.3 백엔드 연결

| 항목 | 값 |
|------|-----|
| 프로토콜 | OData V2 (`sap.ui.model.odata.v2.ODataModel`) |
| 경로 | `/srv-api/odata/v2/detail/` (기능2) · `/srv-api/odata/v2/ott/Mypage/` (기능4) |
| V4→V2 변환 | `@cap-js-community/odata-v2-adapter` (ott/package.json dependencies) |
| 로컬 개발 | `ui5.yaml`에 fiori-tools-proxy → `http://localhost:8083/srv-api` 백엔드 연결 |
| 배포 | approuter_ott가 `/srv-api/` 경로를 CAP 서비스로 프록시 |

## 2. 핵심 설정 파일 상세

### 2.1 approuter_ott/xs-app.json — 라우팅 규칙

```json
{
  "routes": [
    { "source": "^.*/srv-api/(.*)$",                  "destination": "onCapService" },
    { "source": "^.*/portal_ott/~[0-9]+~/(.*)$",     "target": "/comportalott-1.0.0/$1" },
    { "source": "^.*/portal_ott/(.*)$",               "target": "/comportalott-1.0.0/$1" },
    { "source": "/common.lib/(.*)",                   "target": "/commonlib-1.0.0/$1" },
    { "source": "^.*/sysmgt_ott/~[0-9]+~/(.*)$",     "target": "/comsysmgtott-1.0.0/$1" },
    { "source": "^.*/sysmgt_ott/(.*)$",               "target": "/comsysmgtott-1.0.0/$1" },
    { "source": "(.*)",                                "target": "/comportalott-1.0.0/$1" }
  ]
}
```

> 각 UI 모듈은 버전 있는 패턴(`~[0-9]+~`)과 일반 패턴 두 개를 가진다.

| 우선순위 | 패턴 | 대상 |
|:--:|------|------|
| 1 | `/srv-api/*` | CAP 백엔드 (onCapService) |
| 2 | `/portal_ott/*` | OTT 포털 UI |
| 3 | `/common.lib/*` | 공통 라이브러리 |
| 4 | `/sysmgt_ott/*` | OTT 기능 모듈 |
| 5 | 그 외 `(.*)` | portal_ott (기본 진입점) |

> 로컬 개발 시 `dev/xs-app.json`을 사용하며, 여기에는 `/(*ott.*)` 패턴으로 OTT API를 `onOttService`(8083)로 분기 라우팅하는 규칙이 추가되어 있다.

### 2.4 detailPage/ui5.yaml — 로컬 개발 백엔드 프록시

```yaml
metadata:
  name: detailPage
server:
  customMiddleware:
    - name: fiori-tools-proxy
      configuration:
        backend:
          - path: /srv-api
            url: http://localhost:8083          # ← CAP 백엔드 연결
    - name: fiori-tools-servestatic
      configuration:
        paths:
          - path: /common.lib
            src: "../../library/src"
```

> `myPage/ui5.yaml`도 동일한 프록시 설정 적용.

## 3. 백엔드 API 매핑

### 3.1 기능2 — DetailService

| 용도 | OData V2 URL |
|------|-------------|
| 컨텐츠 목록 | `GET /odata/v2/detail/Contents` |
| 컨텐츠 상세 + 장르·태그·출연진·리뷰 | `GET /odata/v2/detail/Contents('C001')?$expand=genres($expand=genre),tags($expand=tag),casts,reviews($expand=user)` |
| 리뷰 등록 (action) | `POST /odata/v2/detail/submitReview { content_id, rating, review_text }` |
| 리뷰 삭제 (action) | `POST /odata/v2/detail/deleteReview { review_id }` |

### 3.2 기능4 — MembershipService

| 용도 | OData V2 URL |
|------|-------------|
| 내 멤버십 | `GET /odata/v2/ott/Mypage/UserSet?$expand=plan` |
| 플랜 목록 | `GET /odata/v2/ott/Mypage/PlanSet` |
| 구독 시작 | `POST /odata/v2/ott/Mypage/subscribe { plan }` |
| 플랜 변경 | `POST /odata/v2/ott/Mypage/changePlan { plan }` |
| 해지 | `POST /odata/v2/ott/Mypage/cancel` |
| 재개 | `POST /odata/v2/ott/Mypage/resume` |

### 4.1 detailPage (기능2)

```
┌─────────────────────────────────────┐
│  ← 목록      ⭐ 4.2  (리뷰 12개)      │
├─────────────────────────────────────┤
│  {title}                            │

│  태그: #명작 #두뇌싸움                   │

│  리뷰 목록 (별점·내용·작성자)           │
│  리뷰 작성 (별점 + 텍스트 + 등록 버튼)   │

```

- URL 라우팅: `Detail/{content_id}`
- 리뷰 삭제 버튼: `visible="{= ${viewModel>user_user_id} === ${User>/userId} }"` (본인만)
- 리뷰 등록/삭제 시 화면 자동 갱신 (평균 평점 포함)

### 4.2 myPage (기능4)

**MY-01: 내 멤버십**
```
┌──────────────────────────┐
│ 👤 마이페이지              │
│ {user_name} 님  [상태배지] │
│ 플랜: {plan/name}         │
│ 월 요금: {plan/price}원    │
│ 기간: {start}~{end}       │
│ [플랜 변경] [해지]          │
└──────────────────────────┘
```

**MY-02: 플랜 선택** (`PlanChange` 라우팅)
```
┌──────────────────────────┐
│ BASIC    STANDARD  PREMIUM│
│ 9,900원  13,900원  17,900원│
│ [선택]   [선택]    [현재]  │

└──────────────────────────┘
```

**상태별 버튼 분기**:
| `subscription_status` | 표시 버튼 |
|----------------------|---------|
| `active` | [플랜 변경] [해지] |
| `cancelled` / `suspended` | [재개] |
| `none` | [구독 시작] |
