# OTT 프로젝트 데이터 정의 (백엔드: 기능6 자유 게시판)

본 문서는 `cap-node/ott`의 데이터 모델 중 **기능6(자유 게시판)**에 해당하는 2개 테이블을 정의한다. 팀 공용 `DATA_SPEC.md`(사용자/컨텐츠/시청기록/플랜/파트너/정산 6개 테이블) 및 기능3(구독자 추세 분석 4개 테이블)에 추가된다.

- 스키마 파일: `cap-node/ott/db/cds/*-model.cds` (namespace `com.cap.ott`, 엔티티당 1파일)
- 시드 데이터: 불필요 (사용자 생성 데이터)
- DB: 개발 = SQLite in-memory, 운영 = HANA (`.cdsrc.json` 프로파일)
- 인증: `@(requires: 'authenticated-user')` — 로그인 사용자만 접근

---

## 1. 게시글 테이블 (`Post`) — 신규, 시드 없음

| 컬럼명 | 타입 | 설명 | 비고 |
|---|---|---|---|
| `post_id` | UUID | 게시글 고유 식별자 | PK (CAP `cuid`) |
| `title` | String(200) | 제목 | NOT NULL |
| `content` | String(5000) | 내용 | NOT NULL |
| `author_id` | String(255) | 작성자 | FK → `Users.user_id` |
| `view_count` | Integer | 조회수 | 기본값 0 |
| `deleted_flag` | Boolean | 삭제 여부 | 기본값 false (소프트 삭제) |
| (+ managed 감사 필드) | | | 자동 관리 (`createdAt`, `createdBy`, `modifiedAt`, `modifiedBy`) |

**주요 활용**:
- 게시글 CRUD: `GET /board/FreeBoard/PostSet`, `POST`, `PATCH`, `DELETE`.
- 조회수 증가: `READ` 후 `view_count += 1`.
- 소프트 삭제: `deleted_flag = true` → 목록에서 제외, 댓글은 보존.

---

## 2. 댓글 테이블 (`Comment`) — 신규, 시드 없음

| 컬럼명 | 타입 | 설명 | 비고 |
|---|---|---|---|
| `comment_id` | UUID | 댓글 고유 식별자 | PK (CAP `cuid`) |
| `post_id` | UUID | 게시글 | FK → `Post.post_id` |
| `content` | String(2000) | 내용 | NOT NULL |
| `author_id` | String(255) | 작성자 | FK → `Users.user_id` |
| (+ managed 감사 필드) | | | 자동 관리 |

**주요 활용**:
- 댓글 CRUD: `GET /board/Comment/CommentSet`, `POST`, `PATCH`, `DELETE`.
- 게시글 삭제 시 댓글도 cascade 삭제 (`Composition`).

---

## 테이블 간 관계

```
Users (1) ──< (N) Post (1) ──< Composition ── (N) Comment
```

- `Post.author_id` → `Users.user_id`: 작성자 식별 (Association 생략, FK 컬럼만).
- `Post` ↔ `Comment`: **Composition** 관계. 게시글 삭제 시 댓글도 함께 삭제된다.
- `Post.deleted_flag = true`인 게시글은 OData 쿼리에서 기본 제외 (`$filter=deleted_flag eq false`).

---

## API 설계

### FreeBoard 서비스 (`/board/FreeBoard`)

| 메서드 | 엔드포인트 | 설명 |
|--------|-----------|------|
| `GET` | `/PostSet` | 게시글 목록 (기본: `deleted_flag=false`, `$orderby=createdAt desc`) |
| `GET` | `/PostSet({id})` | 게시글 상세 (조회수 +1) |
| `POST` | `/PostSet` | 게시글 작성 (`author_id`는 서버에서 `$user` 자동 주입) |
| `PATCH` | `/PostSet({id})` | 게시글 수정 (작성자 본인만) |
| `DELETE` | `/PostSet({id})` | 소프트 삭제 (`deleted_flag = true`, 작성자 본인만) |

### Comment 서비스 (`/board/Comment`)

| 메서드 | 엔드포인트 | 설명 |
|--------|-----------|------|
| `GET` | `/CommentSet?$filter=post_id eq {id}` | 특정 게시글의 댓글 목록 |
| `POST` | `/CommentSet` | 댓글 작성 |
| `PATCH` | `/CommentSet({id})` | 댓글 수정 (작성자 본인만) |
| `DELETE` | `/CommentSet({id})` | 댓글 삭제 (작성자 본인만) |

## CDS 스키마

### Post-model.cds
```cds
namespace com.cap.ott;
using { managed } from '@sap/cds/common';

entity Post {
    key post_id      : UUID;
        title        : String(200)  not null;
        content      : String(5000) not null;
        author_id    : String(255)  not null;
        view_count   : Integer      default 0;
        deleted_flag : Boolean      default false;

        comments     : Composition of many Comment
                         on comments.post_id = post_id;
}
extend Post with managed;
```

### Comment-model.cds
```cds
namespace com.cap.ott;
using { managed } from '@sap/cds/common';

entity Comment {
    key comment_id   : UUID;
        post_id      : UUID         not null;
        content      : String(2000) not null;
        author_id    : String(255)  not null;

        post         : Association to Post
                         on post.post_id = post_id;
}
extend Comment with managed;
```
