## 1. 이 프로젝트가 무엇인가?

**Zen OTT Onboarding**은 SAP BTP(Business Technology Platform) 위에서 동작하는 엔터프라이즈 SaaS 애플리케이션이다. 크게 두 개의 축으로 나뉜다:

| 축 | 설명 |
|---|---|
| **cap-node/core** | 공통 핵심 서비스 (사용자·역할·메뉴·코드 관리 + 이메일·PDF·파일 첨부) |
| **cap-node/ott** | OTT 스트리밍 플랫폼 비즈니스 서비스 (콘텐츠·구독·정산·추세 분석·게시판) |
| **cap-app/\*** | SAPUI5 기반 프론트엔드 애플리케이션들 (포털, 시스템 관리 등) |

**간단히 말해:** 이 프로젝트는 "기업 내에서 사용자와 메뉴를 관리하는 공통 플랫폼" 위에 "OTT 스트리밍(넷플릭스 같은) 비즈니스 로직"을 올려서 만든 통합 시스템이다.

## 4. 주요 개념 설명 (비개발자도 이해할 수 있게)

### 4.1 SAP CAP이란?
SAP CAP(Cloud Application Programming Model)은 SAP가 만든 클라우드 앱 개발 프레임워크다. 데이터 모델(CDS)을 정의하면 DB 테이블과 REST/OData API가 자동으로 생성된다. 마치 "스프레드시트로 테이블 구조를 정하면, 자동으로 백엔드 API가 나온다"고 생각하면 된다.

### 4.2 CDS (Core Data Services)란?
CDS는 데이터 모델을 정의하는 언어다. SQL의 CREATE TABLE 문과 비슷하지만 더 강력하다. 테이블 간 관계(Association, Composition)를 선언적으로 정의하며, 이걸 기반으로 OData API가 자동 생성된다.

### 4.3 OData란?
REST API의 SAP 표준 버전이다. HTTP를 통해 데이터를 주고받으며, URL에 `$filter`, `$expand` 같은 파라미터를 붙여서 복잡한 쿼리를 할 수 있다. 예를 들어 `/UserSet?$filter=use_flag eq true`는 "사용 중인 사용자만 조회"라는 뜻이다.

### 4.4 MTA (Multi-Target Application)란?
SAP BTP에 배포할 때 사용하는 패키지 형식이다. 여러 모듈(백엔드+프론트엔드+DB)을 하나의 아카이브(.mtar)로 묶어서 한 번에 배포한다.

### 4.5 AppRouter란?
모든 HTTP 요청의 진입점이다. 사용자가 접속하면 AppRouter가 URL을 분석해서 적절한 목적지(백엔드 API, UI5 앱 등)로 연결해준다. 인증(XSUAA)도 여기서 처리한다.

### 4.6 XSUAA란?
SAP BTP의 인증 서비스다. 사용자가 로그인하면 JWT 토큰을 발급하고, 이 토큰을 기반으로 어떤 API에 접근할 수 있는지 제어한다.

## 5. 서비스 간 의존 관계

```
사용자 브라우저
      │
      ▼
┌──────────────┐
│  AppRouter   │  ← HTTP 진입점. XSUAA 인증 처리. URL 라우팅.
└──────┬───────┘
       │
       ├──▶ /srv-api/*  ──▶ cap-node/core (공통 핵심 API) ──▶ HANA DB
       │
       ├──▶ /portal/*   ──▶ HTML5 Repository → Portal UI
       ├──▶ /sysmgt/*   ──▶ HTML5 Repository → System Management UI
       │
       └──▶ /common.lib/* ──▶ 공통 UI5 라이브러리
```

**CAP core 서비스** (`cap-node/core`)는 사용자·역할·메뉴·공통코드·메시지·메일·PDF를 담당하는 **공통 인프라 서비스**다.

**CAP ott 서비스** (`cap-node/ott`)는 OTT 스트리밍 비즈니스(콘텐츠·구독·시청·정산·추세·게시판)를 담당하는 **도메인 서비스**다.

> ⚠️ 현재 이 프로젝트에서 `cap-node/ott`는 독립 서비스로 존재하지만, AppRouter 라우팅에는 아직 ott 전용 경로가 설정되어 있지 않다. (향후 추가 필요)

## 6. 배포 순서 (반드시 지켜야 함)

```
순서 1: cap-node/core     → 공통 핵심 서비스 배포 (HANA DB 테이블 생성)
순서 2: cap-node/ott       → OTT 비즈니스 서비스 배포
순서 3: cap-app/library    → 공통 UI5 라이브러리 배포 (다른 UI가 의존)
순서 4: cap-app/portal     → 포털 UI 배포
순서 5: cap-app/sysmgt     → 시스템 관리 UI 배포
순서 6: cap-app/approuter  → AppRouter 배포 (마지막에)
```

## 7. 로컬 개발 실행 방법

```bash
# 최상위 디렉토리에서
npm run init:all          # 모든 모듈 의존성 설치

# 전체 서비스 동시 실행
npm run watch             # CAP core + AppRouter + UI5 동시 기동

# 개별 실행
npm run cap               # CAP core만 (포트 8080)
npm run router            # AppRouter만 (포트 5000)
npm run ui5               # UI5 dev 서버 (포트 5001)
npm run portal            # 포털 UI
npm run sysmgt            # 시스템 관리 UI
```
