# ZEN-OTT — 프론트엔드 데드 코드 정리 보고서 (2026-07-15)

**범위**: detailPage(기능2) + myPage(기능4) 컨트롤러·i18n·CSS
**결과**: 함수 5개 · i18n 키 14개 · CSS 3줄 제거, 총 약 120줄 감소
**사유**: 7/14~7/15 UI 전면 개선(FeedListItem·FeedInput·ObjectPageLayout 전환)으로 인한 불필요 코드 누적

---

## 🗑️ 제거된 함수 (5개, ~115줄)

### 1. `DetailMain.controller.js` — `onSubmitReview()` → `onFeedPost` 대체

```javascript
// ❌ 제거 (~45줄)
onSubmitReview: function () {
    // SimpleForm + Button 기반 리뷰 등록 로직
    // viewModel.getProperty("/NewReview/review_text")로 텍스트 획득
}
```

**사유**: 7/14 `SimpleForm` + `Button` → `FeedInput` 전환. FeedInput은 자체 등록 버튼을 내장하며 `post` 이벤트로 값을 전달. `onFeedPost(oEvent)`가 `oEvent.getParameter("value")`로 텍스트를 획득하는 방식으로 변경되어 기존 함수 완전 불필요.

### 2. `DetailMain.controller.js` — `formatActions()` → `_checkMyReview` 내부 이관

```javascript
// ❌ 제거 (~15줄)
formatActions: function (oReview) {
    // FeedListItemAction 배열 생성 로직
}
```

**사유**: 7/14 도입된 `FeedListItem`의 `actions` aggregation에 필요한 `_actions` 배열을 `_checkMyReview()`에서 직접 `setProperty`로 주입하도록 변경. 별도 formatter 호출 불필요.

### 3. `DetailMain.controller.js` — `onEditReview()` → `onFeedAction` 통합

```javascript
// ❌ 제거 (~20줄)
onEditReview: function (oEvent) {
    // ✏️ 버튼 press 핸들러 — 기존 리뷰 데이터를 NewReview에 prefill
}
```

**사유**: 7/14 ✏️/🗑️ 버튼이 `FeedListItemAction`의 `···` 메뉴로 통합됨. `onFeedAction(oEvent)`에서 `key === "edit"` 분기로 동일 기능 수행. `press` 이벤트 바인딩도 `press="onEditReview"` → `press=".onFeedAction"`으로 변경.

### 4. `DetailMain.controller.js` — `onDeleteMyReview()` 제거

```javascript
// ❌ 제거 (~15줄)
onDeleteMyReview: function () {
    // "내 리뷰" 하위 섹션의 🗑️ 버튼 전용 핸들러
}
```

**사유**: 7/2 리팩토링에서 "내 리뷰" 하위 섹션 자체가 제거됨. 이후 모든 리뷰 삭제는 `onDeleteReview` 단일 경로로 처리. 뷰에서도 이미 `press` 바인딩 제거되어 완전 사장.

### 5. `PlanChange.controller.js` — `isCurrentPlan()` → Expression Binding 대체

```javascript
// ❌ 제거 (~20줄)
isCurrentPlan: function (sPlanCode) {
    var sMembershipPlan = this.getView().getModel("viewModel")
        .getProperty("/Membership/subscription_plan");
    return sPlanCode === sMembershipPlan;
}
```

**사유**: 뷰에서 Expression Binding으로 직접 처리하도록 변경.
```xml
<!-- AS-IS: formatter 호출 -->
visible="{path: 'viewModel>code', formatter: '.isCurrentPlan'}"

<!-- TO-BE: Expression Binding 직접 비교 -->
visible="{= ${viewModel>code} === ${viewModel>/Membership/subscription_plan} }"
```
