# OTT 프로젝트 데이터 정의 (백엔드: 기능3 구독자 추세 분석)

본 문서는 `cap-node/ott`의 데이터 모델 중 **기능3(구독자 추세 분석 & 이상 탐지)**에 해당하는 4개 테이블을 정의한다. 팀 공용 `DATA_SPEC.md`(사용자/컨텐츠/시청기록/플랜/파트너/정산 6개 테이블)에 추가된다.

- 스키마 파일: `cap-node/ott/db/cds/*-model.cds` (namespace `com.cap.ott`, 엔티티당 1파일)
- 시드 데이터: `cap-node/ott/db/data/com.cap.ott-*.csv` (네임스페이스 접두사 필수)
- DB: 개발 = SQLite in-memory(시드 자동 적재), 운영 = HANA (`.cdsrc.json` 프로파일)
- 인코딩: 한글 값이 포함된 CSV는 **UTF-8 BOM** — 엑셀에서 바로 열어도 깨지지 않음

---

## 1. 구독자 그룹 테이블 (`com.cap.ott-SubscriberGroup.csv`) — 신규

분석의 기준이 되는 구독자 그룹을 정의하는 마스터 테이블. 연령대, 성별, 구독 플랜별로 그룹을 나눈다.

| 컬럼명 | 타입 | 설명 | 비고 |
|---|---|---|---|
| `group_id` | String(30) | 그룹 고유 식별자 | PK, `AGE_20S` / `GENDER_F` / `PLAN_PREMIUM` |
| `group_type` | String(30) | 그룹 분류 기준 | `AGE` / `GENDER` / `PLAN` |
| `group_name` | String(100) | 그룹 표시명 | 20대 / 여성 / 프리미엄 |
| (+ managed 감사 필드) | | | 자동 관리 |

**주요 활용**:
- 분석 실행 시 `group_type`에 따라 Users 테이블을 필터링하여 그룹별 집계 수행.
- 프론트에서 그룹 유형 선택 → 해당 그룹 목록 조회.

---

## 2. 그룹별 추세 테이블 (`com.cap.ott-GroupTrend.csv`) — 신규, 시드 없음

그룹-월 단위 구독 및 시청 지표를 스냅샷으로 저장하는 핵심 테이블. **`calculateTrends` 액션으로만 생성**되며, 같은 월 재실행 시 삭제 후 재생성(멱등).

| 컬럼명 | 타입 | 설명 | 비고 |
|---|---|---|---|
| `trend_id` | UUID | 추세 고유 식별자 | PK (CAP `cuid`) |
| `group_id` | String(30) | 그룹 | FK → `SubscriberGroup.group_id` |
| `analysis_month` | String(7) | 분석 월 | 'YYYY-MM' |
| `total_members` | Integer | 그룹 내 전체 사용자 수 | |
| `active_count` | Integer | 활성 구독자 수 | `subscription_status = 'active'` |
| `churn_rate` | Decimal(5,2) | 이탈률(%) | `(해지자 / 전체) × 100` |
| `avg_watch_seconds` | Integer64 | 1인당 평균 시청시간(초) | `총시청시간 / 시청자수` |
| (+ managed 감사 필드) | | | 자동 관리 |

**주요 활용**:
- 그룹별 시계열 그래프: `total_members`, `active_count`, `churn_rate`, `avg_watch_seconds` 추이.
- 그룹 간 비교: "20대 vs 40대 이탈률", "BASIC vs PREMIUM 시청시간".
- 이상 탐지의 기준 데이터: `calculateTrends` 실행 시 전월과 비교하여 이상 변화 계산.

## 3. 이상 탐지 알림 테이블 (`com.cap.ott-TrendAnomaly.csv`) — 신규, 시드 없음

`calculateTrends` 실행 중 임계값을 초과하는 이상 변화가 감지되면 자동 생성되는 알림 레코드.

| 컬럼명 | 타입 | 설명 | 비고 |
|---|---|---|---|
| `anomaly_id` | UUID | 알림 고유 식별자 | PK (CAP `cuid`) |
| `group_id` | String(30) | 그룹 | FK → `SubscriberGroup.group_id` |
| `anomaly_month` | String(7) | 이상 발생 월 | 'YYYY-MM' |
| `anomaly_type` | String(30) | 이상 유형 | `CHURN_SPIKE` / `WATCH_DROP` / `WATCH_SURGE` |
| `severity` | String(10) | 심각도 | `INFO` / `WARN` / `CRITICAL` |
| `change_rate` | Decimal(8,2) | 변화율(%) | 전월 대비 얼마나 변했는지 |
| `alert_text` | String(500) | 알림 메시지 | 한글 자동 생성 (예: "20대 그룹 이탈률이 +11.3%p 급증") |
| `is_read` | Boolean | 확인 여부 | 기본값 `false` |
| (+ managed 감사 필드) | | | 자동 관리 |

**이상 탐지 기준**:
| 유형 | WARN | CRITICAL |
|------|------|----------|
| CHURN_SPIKE (이탈률↑) | +10%p 이상 | +20%p 이상 |
| WATCH_DROP (시청↓) | -30% 이상 | -50% 이상 |
| WATCH_SURGE (시청↑) | +40% 이상 (INFO) | — |

**주요 활용**:
- 관리자 대시보드 알림 목록: 읽지 않은 이상 알림을 심각도 순으로 표시.
- `markAsRead` 액션으로 확인 처리.

## 4. 그룹별 콘텐츠 소비 통계 (`com.cap.ott-ContentGroupStat.csv`) — 신규, 시드 없음

구독자 그룹이 어떤 장르의 콘텐츠를 많이 소비하는지 분석하는 보조 테이블. `calculateTrends` 실행 시 함께 생성된다.

| 컬럼명 | 타입 | 설명 | 비고 |
|---|---|---|---|
| `stat_id` | UUID | 통계 고유 식별자 | PK (CAP `cuid`) |
| `group_id` | String(30) | 그룹 | FK → `SubscriberGroup.group_id` |
| `analysis_month` | String(7) | 분석 월 | 'YYYY-MM' |
| `genre` | String(100) | 장르 | Contents.genre 값 (Sci-Fi, Romance 등) |
| `watch_count` | Integer | 시청 횟수 | |
| `total_seconds` | Integer64 | 총 시청 시간(초) | |
| (+ managed 감사 필드) | | | 자동 관리 |

**주요 활용**:
- "20대가 가장 많이 보는 장르", "40대의 시청 시간이 긴 장르" 등 그룹-장르 교차 분석.

## 테이블 간 관계

```
                          (팀 공용 6개)
Users ──< ViewingHistory >── Contents
  │                              │
  │ (age_group, gender, plan)    │ (genre)
  │                              │
  ▼                              ▼
SubscriberGroup (1) ──< (N) GroupTrend
       │                       │
       │                       ▼ (이상 감지 시)
       │                  TrendAnomaly
       │
       └──< (N) ContentGroupStat
```

- `SubscriberGroup`은 분석의 기준 축. `group_type`에 따라 Users 테이블을 필터링한다.
- `GroupTrend`는 그룹-월 단위의 스냅샷. `calculateTrends` 액션으로 생성된다.
- `TrendAnomaly`는 `GroupTrend` 생성 시 전월과 비교하여 임계값 초과 시 자동 생성된다.
- `ContentGroupStat`은 그룹별 장르 소비를 분석한 통계로, 같은 액션에서 함께 생성된다.

## 분석 실행 흐름

```
POST /odata/v4/ott/TrendAnalysis/calculateTrends
  ↓
1. SubscriberGroup 목록 조회 (use_flag = true 인 그룹)
2. 각 그룹별로 Users 테이블 필터링 (group_type에 따라 AGE→age_group, GENDER→gender, PLAN→subscription_plan)
3. ViewingHistory + Contents JOIN → 그룹 내 사용자의 시청 데이터 집계
4. GroupTrend INSERT (그룹-월 스냅샷)
5. ContentGroupStat INSERT (장르별 집계)
6. 전월 GroupTrend 조회 → 변화율 계산 → 임계값 초과 시 TrendAnomaly INSERT
7. 결과 반환: { trend_count, anomaly_count, anomalies[] }
```

## Users 테이블 확장 필요사항

기능3을 위해 Users 테이블에 아래 2개 컬럼이 추가되어야 한다 (팀 공용 스펙 확장):

| 컬럼명 | 타입 | 설명 | 비고 |
|---|---|---|---|
| `age_group` | String(10) | 연령대 | `10s` / `20s` / `30s` / `40s` / `50s` |
| `gender` | String(1) | 성별 | `M` / `F` |

이 컬럼들은 `SubscriberGroup.group_type = 'AGE'` 또는 `'GENDER'`인 그룹의 필터링에 사용된다.
