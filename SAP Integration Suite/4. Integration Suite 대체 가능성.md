# 4. Integration Suite 대체 가능성

## 결론

**모든 연동에 Integration Suite가 필요한 것은 아닙니다.** 단순하고 변경이 적은 요구에는 더 작은 선택지가 비용과 운영 면에서 적합할 수 있습니다. 반대로 SAP 중심의 다수 시스템·B2B·API·이벤트·하이브리드 운영이 함께 필요한 경우에는 기능을 개별 제품으로 조합하는 비용과 운영 복잡성까지 비교해야 합니다.

## 상황별 판단

| 목적 | Integration Suite가 적합한 경우 | 더 작은/다른 선택지가 적합한 경우 | 판단 포인트 |
|---|---|---|---|
| **API 통합 관리만** | API를 여러 팀·외부 파트너에게 공개하고, 정책·개발자 포털·분석·API 수명주기를 관리해야 할 때 | API가 매우 적고 내부 한 시스템에서만 사용되며, 기존 API Gateway가 이미 표준화되어 있을 때 | API 수, 외부 공개 여부, 인증·감사 요구 |
| **두 SAP ERP 간 업무 연결만** | IDoc/RFC/OData 변환, 재처리, 여러 인터페이스, 모니터링, 향후 SaaS 연결까지 필요할 때 | 인터페이스가 1~2개이고 변경이 드물며, 이미 SAP 표준 기능·직접 RFC/IDoc·파일 교환으로 요구를 충족할 때 | 예외 처리, 모니터링, 변경 빈도 |
| **파일 기반 배치 교환만** | 파일 검증, 변환, 재시도, 여러 상대 시스템, 보안 인증서 관리가 필요할 때 | 단일 SFTP 파일을 단순 전달하고 운영 절차가 매우 안정적일 때 | 파일 포맷 변환과 실패 재처리 필요성 |
| **이벤트 기반 실시간 연동** | 여러 소비자가 같은 이벤트를 구독하고, 비동기 확장·추적·운영이 필요할 때 | 단일 시스템 호출로 충분하고 비동기 처리의 이점이 작을 때 | 소비자 수, 결합도, 실시간성 |
| **B2B/EDI** | 거래처가 많고 메시지 표준·매핑·파트너 설정을 체계적으로 관리해야 할 때 | 거래처가 매우 적고 VAN/EDI 전문 서비스가 이미 요구를 충족할 때 | 거래처 증가, 표준 다양성, 내부 운영 역량 |
| **기존 PI/PO 유지** | 클라우드 전환·신규 SaaS 연결·현대적 API/이벤트 요구가 커질 때 | 신규 요구가 거의 없고 전환 비용/리스크가 이점보다 큰 단기 상황 | PI/PO 지원 계획, 전환 로드맵, 신규 요구 |

## 현실적인 대안 범주

특정 상용 제품을 단정적으로 추천하지 않습니다. 조직의 기존 표준과 계약에 따라 다음 범주를 비교하세요.

1. **SAP 표준 기능 또는 직접 연동**: SAP 표준 IDoc, RFC, OData, 파일 교환으로 충분한 좁은 범위의 연동.
2. **기존 API Gateway / ESB / iPaaS**: 회사가 이미 표준으로 운영 중인 API 관리 또는 통합 플랫폼.
3. **클라우드 사업자 네이티브 서비스**: 단일 클라우드에 통합 대상과 운영 역량이 집중된 경우의 메시징·워크플로·API 서비스.
4. **B2B/EDI 전문 서비스 또는 VAN**: 파트너 수와 표준에 맞춰 아웃소싱이 더 경제적인 경우.
5. **개별 맞춤 개발**: 요구가 작고 수명이 길며 운영 부담이 명확히 감당 가능한 경우. 다만 인증서, 재시도, 모니터링, 변경 대응 비용을 처음부터 포함해야 합니다.

## 비용 비교 시 흔한 오류

- 구독료만 비교하고 개발·운영·장애 대응·보안 패치 비용을 빼는 것
- 현재 1개 인터페이스만 보고 향후 시스템·거래처·API 소비자 증가를 무시하는 것
- 이미 보유한 플랫폼의 라이선스와 운영 역량을 비교에서 빼는 것
- “SAP 제품이므로 항상 최적” 또는 “직접 개발이 항상 저렴”이라고 단정하는 것

## 권장 의사결정 절차

1. 인터페이스별로 대상, 데이터, 빈도, SLA, 보안, 오류 처리, 소유 조직을 표로 만듭니다.
2. 각 인터페이스를 API·메시지·이벤트·B2B 중 어떤 방식으로 풀지 분류합니다.
3. 단순 연동은 직접/기존 플랫폼과 비교하고, 공통 운영 기능이 필요한 연동만 Integration Suite 후보로 올립니다.
4. 파일럿에서 실제 메시지량과 개발·운영 시간을 측정한 뒤 비용을 산정합니다.

## 검토 기준

SAP 공식 문서는 Integration Suite가 여러 통합 방식을 제공함을 설명하지만, 특정 대체 제품의 우열이나 절감액을 보장하지 않습니다. 따라서 이 문서는 제공된 블로그의 제품/비용 주장을 근거로 쓰지 않았고, 선택 기준 중심으로 재작성했습니다.
