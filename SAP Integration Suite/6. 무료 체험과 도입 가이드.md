# 6. 무료 체험과 도입 가이드

## 테스트 목적으로는 무료인가?

**예. SAP의 FSD에는 SAP Integration Suite가 SAP BTP의 free trial에 포함된다고 명시되어 있습니다.** 또한 현재 SAP Help Portal은 Integration Suite에 **free service plan**이 제공된다고 안내합니다. 따라서 학습이나 PoC 목적이라면 BTP Cockpit에서 현재 계정·리전에 사용할 수 있는 무료 플랜과 할당량을 먼저 확인하는 것이 맞습니다.

다만 다음을 구분해야 합니다.

| 구분 | 의미 |
|---|---|
| 무료 체험 | 평가·학습·제한된 PoC를 위한 환경입니다. 종료 시점, 리전, 제공 범위, 쿼터는 계정 화면의 현재 조건을 확인해야 합니다. |
| 상용 사용 | 실제 업무 데이터와 운영 SLA를 전제로 하는 사용입니다. 적절한 계약/플랜과 보안·운영 설계가 필요합니다. |
| Free service plan | Help Portal에서 제공을 명시한 무료 서비스 플랜입니다. 범위·쿼터·리전은 Cockpit의 Entitlement와 플랜 설명에서 확인합니다. |

기존 문서에 있던 “30일” 같은 고정 기간이나 “Free tier에서는 불가”라는 단정은 공식 자료와 맞지 않거나 확인되지 않아 사용하지 않았습니다. 무료 환경을 실제 운영용으로 간주해서도 안 됩니다.

## 가장 안전한 체험 순서

1. [SAP BTP 무료 체험](https://www.sap.com/korea/products/technology-platform/integration-suite/pricing.html)으로 이동해 계정/리전을 선택합니다.
2. BTP Cockpit의 서비스 카탈로그에서 **Integration Suite** Entitlement와 서비스 플랜이 표시되는지 확인합니다.
3. 구독(Subscribe) 후 `Integration_Provisioner` 역할이 있는 사용자가 Integration Suite를 열고, 필요한 capability만 활성화합니다.
4. 실제 운영 데이터 대신 샘플 데이터와 비운영 API를 사용합니다.
5. 가장 작은 iFlow 하나로 연결·변환·모니터링을 검증합니다.
6. 결과를 바탕으로 기능, 월 메시지량, 환경 수, 운영 책임을 기록한 후 상용 견적을 요청합니다.

## 첫 PoC에 권하는 범위

| 포함할 것 | 이유 |
|---|---|
| 송신 시스템 1개, 수신 시스템 1개 | 연결 자체의 난이도를 빠르게 확인 |
| 대표 데이터 1개 | 실제 필드 매핑과 변환을 검증 |
| 정상·오류 메시지 각 1개 | 모니터링과 재처리 절차를 검증 |
| 인증 방식 1개 | 보안 구성 난이도 확인 |
| 배포 절차 1회 | 개발에서 운영으로 옮기는 흐름 확인 |

처음부터 여러 시스템, B2B 파트너, 대량 트래픽을 넣으면 제품 적합성보다 프로젝트 복잡성을 시험하게 됩니다.

## 상용 전환 체크리스트

- [ ] 대상 리전과 데이터 처리 위치가 요구 사항을 만족한다.
- [ ] 사용자 역할, 권한, 인증서·비밀값 관리 책임이 정해졌다.
- [ ] capability 활성화 담당자(`Integration_Provisioner`)와 기능별 role collection 할당 절차가 정해졌다.
- [ ] 개발·테스트·운영 환경 및 이관 절차가 정의됐다.
- [ ] 메시지 보관, 로그 접근, 오류 재처리 담당이 정해졌다.
- [ ] 월 메시지량·페이로드 크기·증가율을 추정했다.
- [ ] 필요한 capability와 플랜이 계약에 포함되는지 확인했다.
