**날짜**: 2026-07-16 (최종 업데이트 — 7/15 formatMaxStreamsValue·OData fallback·데드코드 제거 반영)
**범위**: `myPage/webapp/controller/PlanChange.controller.js` — 플랜 선택/변경 비즈니스 로직
**이전 단계**: [[9. MyMembership.controller.js_완전_이해]], [[8. PlanChange.view.xml_완전_이해]]

## 개요: 이 파일의 정체

`PlanChange.controller.js`는 **플랜 카드 목록을 로드하고, 사용자가 선택한 플랜으로 구독/변경하는 컨트롤러**다. MyMembership에서 "플랜 변경" 또는 "구독 시작"을 누르면 이 컨트롤러로 진입한다. 사용자의 현재 구독 상태에 따라 `/subscribe` 또는 `/changePlan`을 호출한다.

**상속**: `PlanChange` → `myPage.BaseController` (껍데기) → `library_ott.BaseController` (진짜 로직)
**연결된 뷰**: `myPage.view.PlanChange`
**사용 모델**: `viewModel` (JSONModel), `myPage` (OData V2)
**라우트**: `#/PlanChange`
**컨트롤러 크기**: ~120줄 (7/15 데드코드 제거로 소폭 감소)

```javascript
sap.ui.define([
    "./BaseController",

```

| 모듈 | 변수명 | 용도 |
|------|:------:|------|
| `./BaseController` | `BaseController` | 상속할 부모 클래스 |

### 2.1 `formatPrice` — 가격 표시

```javascript
formatPrice: function (nPrice) {
    return (nPrice || 0) + this.getResourceBundle().getText("label.priceUnit");
}
// 9900 → "9900원/월"
```

> **MyMembership의 `formatPrice`와 다른 i18n 키를 쓴다**: `label.currencyUnit`(원) vs `label.priceUnit`(원/월). 카드에서는 "월" 단위를 포함한다.

formatMaxStreamsValue: function (nMax) {

> **7/15 변경**: `formatMaxStreams`(풀문장 "동시시청 4명") → `formatMaxStreamsValue`(값만 "4명"). 뷰에서 `{i18n>label.maxStreams}` 라벨을 별도로 표시하므로 값만 필요.

## 3. 생명주기

```javascript
onInit: function () {
    BaseController.prototype.onInit.call(this);
    this.i18n = this.getResourceBundle();
    this.getOwnerComponent().getRouter()
        .getRoute("PlanChange")
        .attachPatternMatched(this._onRouteMatched, this);
},

onExit: function () {
    this._bDestroyed = true;
},

_onRouteMatched: function () {
    var oViewModel = this.getOwnerComponent().getModel("viewModel");
    oViewModel.setProperty("/SelectedPlan", "");  // ★ 선택 초기화
    this._loadPlans();
},
```

### 3.1 MyMembership과 차이 — `SelectedPlan` 초기화

```javascript
oViewModel.setProperty("/SelectedPlan", "");
```

**왜 필요한가**: 사용자가 플랜 변경을 위해 재진입했을 때, 이전 선택이 남아있으면 안 된다. 예를 들어 BASIC 선택 후 뒤로가기 → 다시 PlanChange 진입 시 BASIC이 선택된 채로 보이는 것을 방지한다.

### 3.2 DetailMain과의 유사성

DetailMain의 `_onRouteMatched`에서 `NewReview`를 초기화한 것과 같은 원리다:

```javascript
// DetailMain
oViewModel.setProperty("/NewReview", { rating: 0, review_text: "" });
this._checkRetryCount = 0;

// PlanChange
oViewModel.setProperty("/SelectedPlan", "");
```

## 4. 데이터 로딩 — `_loadPlans`

```javascript
_loadPlans: function () {
    var oModel = this.getView().getModel("myPage") || this.getOwnerComponent().getModel("myPage");
    var oViewModel = this.getOwnerComponent().getModel("viewModel");
    var oView = this.getView();

    // ★ 7/7 중첩 컴포넌트로 로드 시 모델이 없으면 수동 생성
    if (!oModel) {
        oModel = new sap.ui.model.odata.v2.ODataModel("/srv-api/odata/v2/ott/Mypage", {
            useBatch: false
        });
        this.getView().setModel(oModel, "myPage");
    }

    oView.setBusy(true);

    this.readOdata({
        model: oModel,
        path: "/PlanSet"
    }).then(function (oData) {
        oViewModel.setProperty("/PlanList", oData.results);
        oView.setBusy(false);
    }).catch(function () {
        oView.setBusy(false);
        MessageToast.show(this.i18n.getText("message.planLoadFail"));
    });
},
```

### 4.1 다른 컨트롤러와 비교

| | DetailMain | MyMembership | PlanChange |
|---|---|---|---|
| OData 경로 | `/Contents('C001')` | `/UserSet` | `/PlanSet` |
| $expand | 5개 | 1개 (plan) | 없음 |
| 저장 위치 | `/Detail` | `/Membership` | `/PlanList` (배열) |
| Token 패턴 | 있음 | 없음 | 없음 |

### 4.2 `PlanList`는 배열

```javascript
oViewModel.setProperty("/PlanList", oData.results);
```

`/Detail`이나 `/Membership`처럼 단일 객체가 아니라 **배열**을 저장한다. 이 배열이 뷰의 `FlexBox items="{viewModel>/PlanList}"`에 바인딩되어 카드 그리드를 생성한다.

## 5. 플랜 선택 — `onSelectPlan`

```javascript
onSelectPlan: function (oEvent) {
    var sPlanCode = oEvent.getSource().getCustomData()[0].getValue();
    this.getOwnerComponent().getModel("viewModel").setProperty("/SelectedPlan", sPlanCode);
},
```

### 5.1 `CustomData`로 planCode 전달

```xml
<!-- 뷰에서 -->
<Button press="onSelectPlan">
    <customData>
        <core:CustomData key="planCode" value="{viewModel>code}" />
    </customData>
</Button>
```

```javascript
// 컨트롤러에서
oEvent.getSource()                    // 클릭된 Button
    .getCustomData()                  // [CustomData {key:"planCode", value:"STANDARD"}]
    [0]                               // 첫 번째 CustomData
    .getValue();                      // "STANDARD"
```

### 5.2 왜 Binding Context가 아닌 CustomData인가

DetailMain의 `onDeleteReview`는 리스트 항목에서 `getBindingContext()`를 썼다. PlanChange는 **Aggregation Binding의 자식이지만 FlexBox 내에서 깊게 중첩**되어 있어서, binding context로 접근하기보다 **CustomData로 명시적으로 전달**하는 게 더 직관적이다.

| 방식 | DetailMain (`onDeleteReview`) | PlanChange (`onSelectPlan`) |
|------|--------------------------|---------------------------|
| 데이터 추출 | `getBindingContext("viewModel").getProperty("review_id")` | `getCustomData()[0].getValue()` |
| 특징 | 바인딩된 행의 컨텍스트를 따라감 | 버튼에 직접 값을 박아둠 |

### 5.3 ViewModel 업데이트 → UI 반응

```javascript
viewModel.setProperty("/SelectedPlan", "STANDARD");
```

이 한 줄로 뷰의 모든 Expression Binding이 즉시 반응한다:

| UI 요소 | 변화 |
|---------|------|
| STANDARD 카드의 버튼 | `text="선택"` → `text="선택됨"`, `type="Default"` → `type="Emphasized"` |
| 이전에 선택된 카드의 버튼 | `text="선택됨"` → `text="선택"`, `type="Emphasized"` → `type="Default"` |

## 6. 구독/변경 — `onSubmitPlan`

```javascript
onSubmitPlan: function () {
    if (this._bSubmitting) return;     // ★ 중복 제출 방지

    var oModel = this.getView().getModel("myPage");
    var oViewModel = this.getOwnerComponent().getModel("viewModel");
    var sPlan = oViewModel.getProperty("/SelectedPlan");
    var sCurrentStatus = oViewModel.getProperty("/Membership/subscription_status");

    if (!sPlan) {
        MessageToast.show(this.i18n.getText("message.selectPlan"));
        return;
    }

    this._bSubmitting = true;
    var sAction = (sCurrentStatus === "none") ? "/subscribe" : "/changePlan";
    var that = this;

    oModel.callFunction(sAction, {
        method: "POST",
        urlParameters: { plan: sPlan },
        success: function () {
            if (that._bDestroyed) return;
            MessageToast.show(
                sCurrentStatus === "none"
                    ? that.i18n.getText("message.subscribeSuccess")
                    : that.i18n.getText("message.planChangeSuccess")
            );
            that.getOwnerComponent().getRouter().navTo("MyMembership");
            that._bSubmitting = false;
        },
        error: function () {
            if (that._bDestroyed) return;
            MessageToast.show(that.i18n.getText("message.processError"));
            that._bSubmitting = false;
        }
    });
},
```

### 6.1 상태별 분기

```javascript
var sAction = (sCurrentStatus === "none") ? "/subscribe" : "/changePlan";
```

| `subscription_status` | 호출 API | 의미 |
|:---------------------|----------|------|
| `"none"` | `/subscribe` | 신규 구독 |
| `"active"`, `"cancelled"`, `"suspended"` | `/changePlan` | 플랜 변경 |

### 6.2 중복 제출 방지 — DetailMain과 동일한 `_bSubmitting`

```javascript
if (this._bSubmitting) return;   // 이미 제출 중이면 무시
this._bSubmitting = true;        // 잠금
// ... API 호출 ...
that._bSubmitting = false;       // 해제
```

### 6.3 입력 검증

```javascript
if (!sPlan) {
    MessageToast.show(this.i18n.getText("message.selectPlan"));
    return;
}
```

플랜을 선택하지 않고 "구독 시작"을 누르면 경고 메시지를 표시한다.

### 6.4 성공 후 네비게이션

```javascript
that.getOwnerComponent().getRouter().navTo("MyMembership");
```

DetailMain이 `_loadDetail()`로 재조회한 것과 달리, PlanChange는 **MyMembership 페이지로 이동**한다. MyMembership의 `_onRouteMatched`가 실행되면서 `_loadMembership()`이 호출되어 최신 상태를 반영한다.

### 6.5 V2 `callFunction` — DetailMain, MyMembership과 동일 패턴

```javascript
oModel.callFunction("/subscribe", {
    method: "POST",
    urlParameters: { plan: sPlan },
    success: function () { ... },  // Promise 미지원 → 콜백
    error: function () { ... }
});
```

> **3개 컨트롤러 모두 `callFunction`을 동일한 패턴으로 사용**한다. V2 OData는 Promise를 지원하지 않으므로 항상 `success`/`error` 콜백 패턴이 필요하다.

## 9. 전체 데이터 흐름

```
① URL 진입: #/PlanChange
│  (MyMembership에서 navTo로 이동)
│
② _onRouteMatched()
│   ├─ SelectedPlan → "" (초기화)
│   └─ _loadPlans()
│
③ readOdata("/PlanSet")
│   ├─ OData V2 모델 "myPage"로 HTTP 요청
│   └─ Promise 응답 → viewModel.setProperty("/PlanList", results)
│
④ FlexBox Aggregation Binding → 카드 렌더링
│   └─ 각 카드: 플랜명·가격·동시시청·선택 버튼
│
⑤ 사용자 인터랙션
│   ├─ 카드 "선택" 클릭 → onSelectPlan()
│   │    → CustomData에서 planCode 추출
│   │    → viewModel.setProperty("/SelectedPlan", planCode)
│   │    → 해당 카드 버튼이 "선택됨"으로 변경
│   │
│   ├─ "구독 시작" / "플랜 변경" → onSubmitPlan()
│   │    ├─ 입력 검증 (sPlan 존재 확인)
│   │    ├─ _bSubmitting 락
│   │    ├─ 상태 분기: /subscribe 또는 /changePlan
│   │    ├─ callFunction(sAction, { plan: sPlan })
│   │    └─ 성공 → navTo("MyMembership")
│   │
│   └─ ← 뒤로가기 → onNavBack()
│        → navTo("MyMembership")
```
