# 7. MyMembership.view.xml 완전 이해

**날짜**: 2026-07-07
**범위**: `myPage/webapp/view/MyMembership.view.xml` — 마이페이지 메인 UI
**이전 단계**: [[6. DetailMain.controller.js_완전_이해]], [[5. DetailMain.view.xml_완전_이해]], [[4. BaseController.js_완전_이해]]

## 개요: 이 파일의 정체

`MyMembership.view.xml`은 **사용자의 현재 멤버십(구독) 정보를 보여주는 마이페이지 메인 화면**이다. detailPage와 동일하게 `sap.uxap.ObjectPageLayout`을 루트 레이아웃으로 사용하며, 사용자명·구독 상태·플랜 정보를 표시하고 상태에 따라 해지/재개/변경 버튼을 제공한다.

**연결된 컨트롤러**: `myPage.controller.MyMembership`
**사용 모델**: `viewModel` (JSONModel), `myPage` (OData V2), `i18n` (ResourceModel)
**루트 레이아웃**: `sap.uxap.ObjectPageLayout`
**라우트**: `#/MyMembership`

---

## 0. 사용된 라이브러리

```xml
<mvc:View
    xmlns:mvc="sap.ui.core.mvc"
    xmlns="sap.m"
    xmlns:uxap="sap.uxap"
    xmlns:form="sap.ui.layout.form"
    xmlns:layout="sap.ui.layout"
    height="100%">
```

| 네임스페이스 | 라이브러리 | 이 화면에서의 용도 |
|:-----------|-----------|----------------|
| `sap.m` | 기본 UI 컨트롤 | Title, Text, Label, HBox, VBox, Button, ObjectStatus, OverflowToolbar, ToolbarSpacer |
| `sap.uxap` | ObjectPage 패턴 | ObjectPageLayout, ObjectPageDynamicHeaderTitle, ObjectPageSection, ObjectPageSubSection |
| `sap.ui.layout.form` | 폼 레이아웃 | SimpleForm (ResponsiveGridLayout) — 멤버십 정보 폼 |
| `sap.ui.layout` | 순수 배치 도구 | GridData — 폼 내 반응형 span 제어 |

> **detailPage와의 차이**: `sap.f`, `sap.ui.table`, `sap.ui.export` 등이 필요 없어서 포함하지 않았다. 오직 멤버십 정보 표현에 필요한 라이브러리만 선언했다.

## 1. 전체 구조

```
uxap:ObjectPageLayout
├── uxap:headerTitle
│   └── uxap:ObjectPageDynamicHeaderTitle
│       ├── uxap:heading → Title (사용자명 + 존칭) + ObjectStatus (구독 상태 배지)
│       └── uxap:snappedContent → ObjectStatus (스크롤 시 축약된 상태 배지)
│
└── uxap:sections
    └── uxap:ObjectPageSection (title="멤버십 정보")
        ├── uxap:ObjectPageSubSection
        │   └── form:SimpleForm (ResponsiveGridLayout, 2열)
        │       ├── VBox: 플랜명
        │       ├── VBox: 월 요금
        │       ├── VBox: 동시시청
        │       └── VBox: 구독 기간
        │
        └── uxap:ObjectPageSubSection
            └── OverflowToolbar (우측 정렬)
                ├── "플랜 변경" 버튼 (active일 때)
                ├── "구독 해지" 버튼 (active일 때)
                ├── "구독 재개" 버튼 (cancelled/suspended일 때)
                └── "구독 시작" 버튼 (none일 때)
```

## 2. 헤더 — `ObjectPageDynamicHeaderTitle`

```xml
<uxap:ObjectPageDynamicHeaderTitle>
    <uxap:heading>
        <HBox alignItems="Center">
            <Title
                text="{path: 'viewModel>/Membership/user_name', formatter: '.formatUserName'}"
                level="H2" />
            <ObjectStatus
                text="{path: 'viewModel>/Membership/subscription_status', formatter: '.statusFormatter'}"
                state="{path: 'viewModel>/Membership/subscription_status', formatter: '.statusStateFormatter'}"
                class="sapUiSmallMarginBegin" />
        </HBox>
    </uxap:heading>
    <uxap:snappedContent>
        <ObjectStatus
            text="{path: 'viewModel>/Membership/subscription_status', formatter: '.statusFormatter'}"
            state="{path: 'viewModel>/Membership/subscription_status', formatter: '.statusStateFormatter'}" />
    </uxap:snappedContent>
</uxap:ObjectPageDynamicHeaderTitle>
```

### 2.1 `ObjectPageDynamicHeaderTitle`의 3개 영역

| 영역 | 언제 보이나 | 이 화면에서 넣은 것 |
|------|:--------:|------|
| **heading** | **항상** | 사용자명 (H2) + 상태 배지 |
| expandedContent | 스크롤 최상단만 | _이 화면에선 사용 안 함_ |
| **snappedContent** | 스크롤 내리면 | 상태 배지만 축약 표시 |

> **detailPage와 비교**: detailPage는 `expandedContent`에 타입·별점·리뷰수를 표시했지만, MyMembership은 단순해서 heading만 사용하고 expandedContent는 생략했다.

### 2.2 포매터 체인

| 바인딩 | 포매터 | 입력 → 출력 예시 |
|--------|--------|-----------------|
| `user_name` | `formatUserName` | `"홍길동"` → `"홍길동님"` |
| `subscription_status` | `statusFormatter` | `"active"` → `"구독중"` |
| `subscription_status` | `statusStateFormatter` | `"active"` → `"Success"` (초록색) |

## 3. 멤버십 정보 — `SimpleForm` + `ResponsiveGridLayout`

```xml
<form:SimpleForm
    layout="ResponsiveGridLayout"
    editable="false"
    labelSpanL="4" labelSpanM="4"
    emptySpanL="0" emptySpanM="0"
    columnsL="2" columnsM="2">
```

### 3.1 SimpleForm 속성

| 속성 | 값 | 의미 |
|------|------|------|
| `layout` | `ResponsiveGridLayout` | 반응형 12컬럼 그리드. 화면 크기에 따라 자동 재배치 |
| `editable` | `false` | 읽기 전용 폼 (Label 굵게, Text 일반) |
| `labelSpanL` | `4` | Large 화면에서 라벨이 4컬럼 차지 |
| `emptySpanL` | `0` | 빈 컬럼 없음 — 타이트한 배치 |
| `columnsL` | `2` | Large 화면에서 2열 |

### 3.2 각 필드 구조 (공통 패턴)

```xml
<VBox class="membershipFormVBox">
    <Label text="{i18n>label.plan}" design="Bold" />
    <Text text="{viewModel>/Membership/plan/name}" />
    <layoutData>
        <layout:GridData span="L6 M6 S12" />
    </layoutData>
</VBox>
```

각 필드는 `VBox`로 감싼 `Label` + `Text` 쌍이다. `GridData`로 반응형 span을 지정한다.

| 화면 | span | 결과 |
|------|:----:|------|
| Large (≥1024px) | L6 | 2열 (6+6=12) |
| Medium (≥600px) | M6 | 2열 |
| Small (Phone) | S12 | 1열 (전체 너비) |

### 3.3 4개 필드

| 필드 | 바인딩 | 데이터 출처 | 포매터 |
|------|-------|----------|:--:|
| 플랜명 | `viewModel>/Membership/plan/name` | `$expand=plan` | - |
| 월 요금 | `viewModel>/Membership/plan/price` | `$expand=plan` | `.formatPrice` |
| 동시시청 | `viewModel>/Membership/plan/maxStreams` | `$expand=plan` | `.formatMaxStreamsText` |
| 구독 기간 | `viewModel>/Membership/subscription_start_date` ~ `subscription_end_date` | UserSet | OData Date type |

### 3.4 기간 표시 — 두 바인딩을 한 Text에

```xml
<Text text="{path:'.../subscription_start_date', type:'sap.ui.model.odata.type.Date', formatOptions:{pattern:'yyyy-MM-dd'}} ~ {path:'.../subscription_end_date', type:'sap.ui.model.odata.type.Date', formatOptions:{pattern:'yyyy-MM-dd'}}" />
```

OData Date 타입을 사용해 날짜 포맷을 `yyyy-MM-dd`로 지정. 두 바인딩 사이에 `~`를 직접 텍스트로 넣었다.

> **주의**: `formatOptions`를 쓸 때는 `{path: '...', type: '...', formatOptions: {...}}` 확장형 바인딩이 필요하다.

## 4. CSS — 2열 정렬 강제 (`style.css`)

```css
.sapUiFormResGridCont {
    display: flex !important;
    flex-wrap: wrap !important;
    align-items: flex-start !important;
}
.sapUiFormResGridCont > .sapUiRespGridSpanL6,
.sapUiFormResGridCont > .sapUiRespGridSpanM6 {
    flex: 0 0 50% !important;
    width: 50% !important;
}
```

**왜 필요한가**: `ResponsiveGridLayout`은 내부적으로 `float` + `display: block`을 사용해서, VBox 높이가 다르면 **높낮이가 틀어지는** 문제가 있다. `!important`로 강제로 `flex`로 전환해 2열이 항상 상단 정렬되도록 고정했다.

> **History**: 7/1 피드백 #6에서 발견해 해결. CSS 파일 캐시 문제로 JS `document.createElement("style")`까지 동원했던 이슈다. ([[컴팩트_2026-07-01]] 참조)

## 5. 액션 버튼 — 상태별 가시성

```xml
<OverflowToolbar>
    <ToolbarSpacer />
    <Button text="플랜 변경" press="onGoPlanChange"
        visible="{= ${viewModel>/Membership/subscription_status} === 'active' }" />
    <Button text="구독 해지" press="onCancelConfirm"
        visible="{= ${viewModel>/Membership/subscription_status} === 'active' }" />
    <Button text="구독 재개" press="onResumeConfirm"
        visible="{= ${viewModel>/Membership/subscription_status} === 'cancelled' || ... === 'suspended' }" />
    <Button text="구독 시작" press="onGoPlanChange"
        visible="{= ${viewModel>/Membership/subscription_status} === 'none' }" />
</OverflowToolbar>
```

### 5.1 `OverflowToolbar` + `ToolbarSpacer`

| 요소 | 역할 |
|------|------|
| `OverflowToolbar` | 버튼 바. 공간 부족 시 우측 버튼이 `...` 메뉴로 접힘 |
| `ToolbarSpacer` | 남은 공간을 모두 차지 → 버튼을 **우측 정렬** |

### 5.2 상태별 버튼 (Expression Binding)

```javascript
${viewModel>/Membership/subscription_status} === 'active'
```

| subscription_status | 보이는 버튼 |
|:-------------------|-----------|
| `"active"` | "플랜 변경" + "구독 해지" |
| `"cancelled"` / `"suspended"` | "구독 재개" |
| `"none"` | "구독 시작" (onGoPlanChange로 이동) |

> **"구독 시작"과 "플랜 변경"은 같은 `onGoPlanChange` 핸들러**를 사용한다. 둘 다 PlanChange 페이지로 이동하지만, PlanChange 컨트롤러에서 현재 상태를 보고 `/subscribe` 또는 `/changePlan`을 호출한다.

### 5.3 `ObjectStatus`의 `state` 속성

```xml
<ObjectStatus state="{path: '...', formatter: '.statusStateFormatter'}" />
```

`state` 속성은 **SAP Fiori 디자인 시스템의 시맨틱 컬러**를 적용한다:

| statusStateFormatter 반환값 | 색상 | 의미 |
|:--------------------------|------|------|
| `"Success"` | 초록색 | active (정상 구독) |
| `"Warning"` | 주황색 | cancelled (해지 예정) |
| `"Error"` | 빨간색 | suspended (정지) |
| `"None"` | 기본 (회색) | none (미구독) |

## 6. detailPage 뷰와의 비교

| 요소 | detailPage | MyMembership |
|------|-----------|-------------|
| 루트 레이아웃 | ObjectPageLayout | ObjectPageLayout (동일) |

| 조건부 visible | HasReviewed, editing_review_id | subscription_status (4가지) |

## 7. 핵심 암기 사항

| # | 내용 |
|---|------|
| 1 | `ObjectPageDynamicHeaderTitle`의 4개 영역 중 이 화면은 heading + snappedContent만 사용 |
| 2 | `SimpleForm` + `ResponsiveGridLayout` + `GridData span` = 반응형 폼 표준 패턴 |
| 3 | `Label design="Bold"` + `editable="false"` = 읽기 전용 폼 스타일 |
| 4 | CSS `!important`로 `float` → `flex` 강제 전환 (높낮이 정렬) |
| 5 | `OverflowToolbar` + `ToolbarSpacer` = 우측 정렬 버튼 바 |
| 6 | Expression Binding으로 상태별 버튼 visible 제어 (`=== 'active'` 등) |
| 7 | `ObjectStatus.state` = SAP 시맨틱 컬러 (Success/Warning/Error/None) |
| 8 | 두 바인딩을 하나의 Text에: `{path:...} ~ {path:...}` 직접 결합 |
| 9 | `$expand=plan`으로 플랜명·가격·동시시청 정보를 한 번에 가져옴 |
| 10 | detailPage보다 단순 — 섹션 1개, 조건 분기 단순, OData 호출 1회 |
