# 2026-06-26 CAP 온보딩 신입 공유

## Context

- CAP Template의 기본 소스 구조와 기존 UI5 개발 구조를 빠르게 파악한다.
- UI5 화면 개발 시 자주 사용하는 ViewModel, Filter, Control 탐색 방법을 익힌다.
- 상세 페이지 이동과 테이블 선택/액션 패턴을 SAPUI5 공식 예제 기준으로 확인한다.

## Discussion

### 공통 안내

#### Fiori Design Guidelines

- 화면 설계 전 SAP Fiori Design Guidelines를 먼저 확인한다.
- 주요 확인 대상:
  - `sap.m.Table`과 `sap.ui.table.Table`의 사용 상황 차이
  - `Object Page Layout` 사용 예시
  - `Dynamic Page Layout` 사용 예시

#### `sap.m.Table` vs `sap.ui.table.Table`

| 구분 | `sap.m.Table` | `sap.ui.table.Table` |
|---|---|---|
| 주 사용처 | 반응형 화면, 모바일/태블릿 친화 목록 | 데스크톱 중심의 대량 데이터 그리드 |
| UI 성격 | List 기반, 카드형/목록형 표현에 적합 | 행/열 기반 업무 테이블에 적합 |
| 권장 상황 | 간단한 목록, 상세 이동 중심 화면 | 컬럼이 많고 정렬/필터/행 액션이 중요한 화면 |
| 참고 포인트 | Fiori List Report 패턴과 함께 검토 | Row Action, Selection, Column 설정 확인 |

### CAP Template 소스 구조

CAP Template은 크게 백엔드 CAP 영역과 프론트엔드 UI5 영역으로 나누어 본다.

#### 백엔드 CAP 영역

- `cap-node/*/db`: CDS 모델과 초기 데이터 관리
- `cap-node/*/srv`: 서비스 CDS와 handler 구현
- `cap-node/*/srv/cds`: OData 서비스 정의
- `cap-node/*/srv/src`: 서비스별 TypeScript handler, feature 로직
- `mta.yaml`, `xs-security.json`, `default-env.json`: 배포/인증/로컬 실행 설정

#### 프론트엔드 UI5 영역

- `cap-app/*/webapp/manifest.json`: 앱 설정, 라우팅, 모델 선언
- `cap-app/*/webapp/view`: XML View와 Fragment
- `cap-app/*/webapp/controller`: 화면 Controller
- `cap-app/library/src`: 공통 Control, Util, Model, i18n 등 재사용 모듈
- `xs-app.json`, `ui5.yaml`, `package.json`: App Router, UI5 tooling, 실행/빌드 설정

### `messageManagement` 앱 구조 기준 UI5 개발 흐름

이번 공유에서는 `cap-app/sysmgt/webapp/messageManagement` 앱을 기준으로 실제 UI5 구조를 설명한다.

| 구분 | 파일 | 역할 |
|---|---|---|
| 앱 설정 | `webapp/manifest.json` | OData datasource, `viewModel`, rootView, routing 정의 |
| 앱 초기화 | `webapp/Component.js` | `UIComponent` 초기화 후 router initialize |
| 루트 View | `view/MainFlexibleColumnLayout.view.xml` | `sap.f.FlexibleColumnLayout` 배치 |
| 목록 View/Controller | `view/MainList.view.xml`, `controller/MainList.controller.js` | 검색 조건, 목록 조회, 상세 이동 |
| 상세 View/Controller | `view/DetailList.view.xml`, `controller/DetailList.controller.js` | 상세 조회, 편집모드, 다국어 메시지 행 추가/삭제 |
| 공통 기능 | `cap-app/library/src/controller/BaseController.js` | 공통 코드/사용자/메뉴 모델 로딩, `readOdata`, i18n helper |

읽는 순서:

1. `manifest.json`에서 model과 route를 먼저 확인한다.
2. `MainFlexibleColumnLayout.view.xml`에서 전체 레이아웃 바인딩을 확인한다.
3. `MainList.view.xml`의 검색조건과 테이블 row binding을 본다.
4. `MainList.controller.js`에서 검색조건을 읽어 OData 호출 후 `viewModel`에 결과를 넣는 흐름을 확인한다.
5. `onRowActionMoveDetail`에서 상세 route로 이동하는 방식을 확인한다.
6. `DetailList.controller.js`에서 route parameter를 받아 상세 데이터를 조회하고 편집모드를 제어하는 방식을 확인한다.

### `viewModel` 실제 사용 예시

#### `manifest.json` 선언

`messageManagement` 앱은 `viewModel`이라는 이름의 JSONModel을 선언한다. 기본 OData 모델은 이름 없는 모델 `""`로 선언되어 있고, 화면 상태와 조회 결과는 `viewModel`에 담는다.

```json
"models": {
  "": {
    "dataSource": "odataService",
    "preload": true,
    "settings": {
      "defaultCountMode": "Inline",
      "useBatch": true
    }
  },
  "viewModel": {
    "type": "sap.ui.model.json.JSONModel"
  }
}
```

#### 전체 레이아웃 상태

`MainFlexibleColumnLayout.view.xml`은 `FlexibleColumnLayout`의 `layout`을 `viewModel>/App/MainLayout`에 바인딩한다.

```xml
<f:FlexibleColumnLayout
    id="MainFlexibleColumnLayout"
    layout="{viewModel>/App/MainLayout}" />
```

`MainFlexibleColumnLayout.controller.js`와 `MainList.controller.js`에서는 `/App` 하위에 레이아웃과 버튼 상태를 초기화한다.

```js
oViewModel.setProperty("/App", {
  MainLayout: "OneColumn",
  EditMode: false,
  MainBtnScreen: "sap-icon://full-screen",
  detailHeader: ""
});
```

상세 페이지에 진입하면 `DetailList.controller.js`에서 `MainLayout`을 `TwoColumnsMidExpanded`로 바꿔 목록+상세 2단 화면을 만든다.

```js
oViewModel.setProperty("/App/MainLayout", "TwoColumnsMidExpanded");
```

#### 목록 검색 조건과 결과

`MainList.controller.js`의 `_setMainListViewModel`은 목록 화면에서 사용할 검색조건과 결과 리스트의 기본 구조를 만든다.

```js
oViewModel.setProperty("/MainListModel", {
  list: [],
  cond: {
    EQ: {},
    Contains: {},
    GE: {},
    LE: {}
  }
});
```

`MainList.view.xml`의 검색 조건은 이 구조에 직접 바인딩된다.

```xml
<Input value="{viewModel>/MainListModel/cond/Contains/message_code}" />
<Input value="{viewModel>/MainListModel/cond/Contains/message_contents}" />
<ComboBox selectedKey="{viewModel>/MainListModel/cond/Contains/message_type_code}" />
```

검색 결과 테이블은 `/MainListModel/list`에 바인딩한다.

```xml
<t:Table
    id="mainListTable"
    rows="{viewModel>/MainListModel/list}"
    rowActionCount="1">
```

Controller에서 OData 조회가 끝나면 결과를 `viewModel`에 넣고, View는 자동으로 갱신된다.

```js
this.readOdata({ path: sPath }).then(function (oReturnData) {
  oViewModel.setProperty("/MainListModel/list", oReturnData.value);
});
```

#### 상세 데이터와 편집모드

`DetailList.controller.js`는 상세 화면용으로 `/DetailListModel`을 사용한다.

```js
oViewModel.setProperty("/DetailListModel", {
  GeneralInfo: {},
  messageList: []
});
```

상세 기본정보는 `/DetailListModel/GeneralInfo`, 다국어 메시지 목록은 `/DetailListModel/messageList`에 바인딩된다.

```xml
<Text text="{viewModel>/DetailListModel/GeneralInfo/message_code}" />
<t:Table id="message" rows="{viewModel>/DetailListModel/messageList}">
```

편집모드는 `/layout/editmode`로 제어한다. 상세 View의 수정/취소/저장 버튼과 입력 필드가 이 값에 따라 표시된다.

```xml
<Button press="onPressEdit" visible="{=!${viewModel>/layout/editmode}}" />
<Button press="onPressEditCancel" visible="{=!!${viewModel>/layout/editmode}}" />
<Button press="onPressSave" visible="{=!!${viewModel>/layout/editmode}}" />
```

신규 생성 route(`message_code === "new"`)로 들어오면 처음부터 편집모드로 열린다.

```js
oViewModel.setProperty("/layout", {
  editmode: oArgs.message_code === "new"
});
```

### `messageManagement` 기준 Filter 생성 방법

이 앱에서는 검색 방식이 두 가지로 나뉜다.

#### 메인 목록 검색

`MainList.controller.js`의 `_bindListTable`은 `viewModel`의 검색조건을 읽어서 OData V4 `$filter` 문자열을 만든다.

```js
oCondData = oViewModel.getProperty("/MainListModel/cond");

if (oCondData.Contains.message_code) {
  const searchVal = oCondData.Contains.message_code.toLowerCase();
  filterArray.push("contains(tolower(message_code),'" + searchVal + "')");
}
```

조건이 있으면 function import 경로 뒤에 `$filter`를 붙인다.

```js
var sPath = `/srv-api/odata/v4/core/MessageManagement/MessageList('${languageCode?.toUpperCase()}')/Set`;

if (filterArray.length) {
  sPath += `?$filter=${filterArray.join(" and ")}`;
}
```

조회 결과는 OData binding에 직접 연결하지 않고 `readOdata` helper로 읽은 뒤 `/MainListModel/list`에 넣는다.

#### 상세 조회

상세 화면은 OData V2 모델의 `model.read`에 `sap.ui.model.Filter` 배열을 넘긴다.

```js
aFilters = [];
aFilters.push(new Filter("message_code", "EQ", oArgs.message_code));

this.readOdata({
  model: oDataModel,
  path: "/MessageSet",
  param: {
    filters: aFilters
  }
});
```

조회 결과에서 브라우저 언어와 맞는 행을 `GeneralInfo`로 잡고, 전체 결과는 `messageList`로 저장한다.

```js
const languageCode = (navigator.language || navigator.userLanguage).split("-")[0];
let oGeneralInfo = oReturnData.results.filter(
  o => o.language_code === languageCode?.toUpperCase()
)[0] || oReturnData.results[0];

oViewModel.setProperty("/DetailListModel/GeneralInfo", oGeneralInfo);
oViewModel.setProperty("/DetailListModel/messageList", oReturnData.results);
```

### UI5 Control 탐색 방법

브라우저에서 UI5 화면을 디버깅할 때 아래 단축키를 사용한다.

| 목적 | 방법 |
|---|---|
| Technical Information / Debug 정보 확인 | `Ctrl + Alt + Shift + P` |
| Control Tree 확인 | `/Control Tree` 메뉴 확인 |
| 클릭한 Control 정보 확인 | `Ctrl + Alt + Shift + S` 후 대상 Control 클릭 |

확인할 항목:

- Control ID
- 바인딩 path
- 적용된 model 이름
- parent/child 구조
- event handler 연결 여부

### Detail Page Navigation

효서 매니저님 질문 관련 내용이다.

- `messageManagement`는 `sap.ui.table.Table`의 `rowActionTemplate`을 사용한다.
- 목록 View의 `RowActionItem` 클릭 시 `onRowActionMoveDetail`이 실행된다.
- 선택한 row context에서 `message_code`를 꺼내 `DetailList` route로 넘긴다.
- 상세 Controller는 `DetailList` route의 `attachPatternMatched`에서 route parameter를 받아 상세 데이터를 다시 조회한다.

목록 View의 RowAction:

```xml
<t:rowActionTemplate>
  <t:RowAction>
    <t:RowActionItem
        type="Custom"
        icon="sap-icon://begin"
        press="onRowActionMoveDetail" />
  </t:RowAction>
</t:rowActionTemplate>
```

목록 Controller의 상세 이동:

```js
onRowActionMoveDetail: function (oEvent) {
  const oRow = oEvent.getParameter("row");
  const iSelectedIdx = oRow.getIndex();
  const oTable = this.byId("mainListTable");
  const oContext = oTable.getContextByIndex(iSelectedIdx);
  const oSelectItem = oContext.getObject();

  oViewModel.setProperty("/App/detailHeader", oSelectItem);
  oRouter.navTo("DetailList", {
    message_code: oSelectItem.message_code
  });
}
```

`manifest.json`의 route는 `message_code`를 받도록 정의되어 있다.

```json
{
  "pattern": "DetailList/:message_code:",
  "name": "DetailList",
  "target": ["MainList", "DetailList"]
}
```

상세 Controller에서는 route matched 시점에 argument를 `/Args`에 저장하고, 신규가 아니면 `_setGenralInfo`로 상세 데이터를 조회한다.

```js
oArgs = e.getParameter("arguments");
oViewModel.setProperty("/Args", oArgs);
oViewModel.setProperty("/App/MainLayout", "TwoColumnsMidExpanded");

this._setDetailListViewModel(oArgs);
if (oArgs.message_code !== "new") {
  this._setGenralInfo();
}
```
