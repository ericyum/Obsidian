## 개요: 왜 BaseController가 중요한가?

`DetailMain.controller.js`에서 다음 코드가 아무런 정의 없이도 동작하는 이유를 설명하는 핵심 파일이다.

```javascript
// DetailMain.controller.js 내부에서...
this.i18n.getText("button.submitReview");       // ← 어디서 왔나?
this.readOdata({ model, path, param });          // ← 어디서 왔나?
this._oCommonCode["GENRE"];                      // ← 어디서 왔나?
this.fnDeepCopy(oData);                          // ← 어디서 왔나?
```

**답**: 모두 `library_ott/src/controller/BaseController.js`에 정의되어 있고, 상속 체인을 통해 전달된다.

---

## 상속 체인 (3단계)

```
sap.ui.core.mvc.Controller              ← SAP UI5 프레임워크
    └─ common_ott.lib.controller.BaseController  ← ⭐ 진짜 모든 로직이 여기에
        │   (library_ott/src/controller/BaseController.js)
        │
        └─ detailPage.controller.BaseController  ← 빈 껍데기 (확장 포인트)
            │   (sysmgt_ott/.../detailPage/.../BaseController.js)
            │
            └─ detailPage.controller.DetailMain  ← 실제 비즈니스 컨트롤러
                (sysmgt_ott/.../detailPage/.../DetailMain.controller.js)
```

### 왜 빈 껍데기가 있는가?

```javascript
// detailPage/controller/BaseController.js (단 6줄!)
sap.ui.define([
    "common_ott/lib/controller/BaseController",
], function (Controller) {
    "use strict";
    return Controller.extend("detailPage.controller.BaseController", {});
});
```

**진짜 이유는 단순하다**: detailPage 안에 여러 view가 생길 경우, **그 view들이 공유할 공통 로직을 넣는 자리**다.

```
detailPage/webapp/controller/
├── BaseController.js         ← detailPage 전용 공통 로직 (현재는 빈 껍데기)
├── DetailMain.controller.js  ← 컨텐츠 상세
└── ReviewList.controller.js  ← 예: 리뷰 모아보기 (신규 view가 생기면 여기에 공통 코드 추가)
```

> **솔직히 말하면**: 템플릿 원본이 이 구조를 택했기 때문에 그대로 따라 한 것이고, 지금 당장은 삭제해도 기술적으로 아무 문제가 없다. `DetailMain.controller.js`가 `common_ott/lib/controller/BaseController`를 직접 상속해도 잘 돌아간다. 나중에 view가 늘어나면 그때 진가를 발휘할 자리일 뿐이다.

> myPage의 `BaseController.js`도 동일한 패턴 — `Controller.extend("myPage.controller.BaseController", {})`

## library_ott BaseController가 해주는 일

### 핵심: `onInit()` — 기초 데이터 5종 세팅

컨트롤러가 생성될 때 `onInit()`이 자동 실행되면서, 프론트엔드에서 기본적으로 필요한 데이터 5가지를 비동기로 로딩한다:

| 로더 | 모델명 | 담는 데이터 |
|------|:------:|------|
| `UserSessionLoader` | `"User"` | 로그인 사용자 정보 (ID, 이름, 권한) |
| `CommonCodeLoader` | `"CommonCode"` | 장르, 멤버십 상태 같은 공통 코드표 |
| `MenuLoader` | `"Menu"` | 메뉴 구조 |
| `MessageLoader` | `"Message"` | 다국어 시스템 메시지 |
| `MenuFunctionLoader` | `"MenuFunction"` | 메뉴별 버튼/기능 권한 |

각 로더는 `attachEventOnce('ready', callback)` 패턴을 써서 **딱 한 번만** 완료 이벤트를 받는다. 완료된 데이터는 `this.getView().setModel()`로 뷰에, `sap.ui.getCore().setModel()`로 전역에 저장된다.

추가로 `CommonCodeLoader`는 `this._oCommonCode` 라는 그릇에 따로 담아둔다. `fnMasterCodeValue()` 같은 메서드에서 빠르게 꺼내 쓰기 위해서다.

### 그 외 유용한 메서드들

| 메서드 | 하는 일 |
|--------|------|
| `getResourceBundle()` | `this.i18n.getText("키")`로 텍스트를 꺼낼 수 있게 i18n 번들 반환 |
| `readOdata()` | 콜백 기반 `model.read()`를 Promise로 감싸서 `await` 가능하게 |
| `fnDeepCopy()` | 객체 깊은 복사 (OData 바인딩 원본 손상 방지용) |
| `fnMasterCodeValue()` | `"01"`, `"GENRE"` 넣으면 `"액션"` 반환 |
| `resetTableSettings()` | 테이블 정렬·필터 초기화 |
| `switchTablePersonalization()` | 테이블 편집모드일 때 필터·정렬 임시 해제/복원 |

## DetailMain에서의 실제 사용 흐름

```
① DetailMain.view.xml 로드 → DetailMain.controller.js 인스턴스 생성
    │
② 상속 체인으로 library_ott의 BaseController.onInit() 실행
    ├── UserSessionLoader → User 모델 세팅
    ├── CommonCodeLoader  → _oCommonCode 그릇에 담기
    ├── MenuLoader, MessageLoader, MenuFunctionLoader
    └── this.i18n = this.getResourceBundle();
    │
③ DetailMain 자신의 onInit() 실행
    ├── this.i18n.getText(...)         ← ②에서 준비된 i18n 사용
    ├── this.readOdata({model, path})  ← BaseController의 Promise 래퍼
    ├── this.fnMasterCodeValue(...)    ← _oCommonCode 그릇에서 조회
    └── router.attachPatternMatched(...); ← URL 변경 감지 구독
```

> **핵심**: DetailMain.controller.js는 BaseController의 모든 메서드를 `this.메서드명()`으로 바로 사용할 수 있다. `this.i18n`, `this.readOdata()`, `this.fnDeepCopy()` 등 전부 상속받은 것이다.

## 핵심 암기 사항

| # | 내용 |
|---|------|
| 1 | **상속 체인**: `Controller → library_ott.BaseController → detailPage.BaseController(껍데기) → DetailMain` |
| 2 | **껍데기의 진짜 이유**: detailPage에 여러 view가 생기면 **공통 로직을 담을 자리**. 지금은 템플릿 따라 만든 빈 공간일 뿐 |
| 3 | **`onInit()`은 기초 데이터 5종 로딩**: User, CommonCode, Menu, Message, MenuFunction |
| 4 | **CommonCode는 `_oCommonCode` 그릇에 따로 담긴다** → `fnMasterCodeValue()`에서 빠르게 조회 |
| 5 | **`readOdata()`는 Promise 래퍼** → `await`로 깔끔하게 OData 호출 가능 |
| 6 | **DetailMain은 BaseController의 모든 메서드를 그냥 `this.XXX()`로 쓴다** |
