# 10. BTP Destination 설정

> **작성일**: 2026-07-13
> **역할**: approuter가 백엔드와 통신하기 위해 필요한 BTP Destination 설정. BTP Cockpit에서 **수동으로 구성**해야 하는 부분입니다.

---

## 📋 Destination 목록

| 이름 | URL | 연결 대상 |
|------|-----|-----------|
| `onCapService` | `https://org-...-com-cap-ott-core.cfapps.jp10.hana.ondemand.com` | core_ott (인증, 메뉴, 사용자) |
| `onOTTService` | `https://org-...-com-cap-ott.cfapps.jp10.hana.ondemand.com` | ott (콘텐츠, 멤버십, 정산) |

> ⚠️ URL은 실제 CF 배포 후 할당된 경로를 확인하여 입력해야 합니다.

---

## 🔧 공통 설정값

```
유형: HTTP
프록시 유형: Internet
인증: OAuth2JWTBearer
클라이언트 ID: sb-com-app-xsapp!t50732
토큰 서비스 URL: https://tf-joule-1on6yytw.authentication.jp10.hana.ondemand.com/oauth/token
토큰 서비스 URL 유형: Dedicated
추가 속성:
  tokenType = Bearer
```

---

## 🔄 OAuth2JWTBearer 동작 방식

```
① 사용자 portal_ott 접속 → XSUAA 로그인 → JWT 발급
② 사용자 OData API 호출 (예: /srv-api/odata/v2/detail/...)
③ approuter: xs-app-dev.json 라우트 매칭 → destination 확인
④ approuter: 사용자 JWT를 bearer assertion으로 XSUAA token service에 제시
⑤ XSUAA: 새 JWT 발급 (클라이언트 ID sb-com-app-xsapp 기반)
⑥ approuter: 새 JWT를 Authorization 헤더에 담아 destination으로 프록시
⑦ CAP 백엔드: JWT 검증 → req.user 설정 → 요청 처리
```

---

## 🔗 xs-app-dev.json과 Destination 연결

```json
// Route ①: ott-core → onCapService
{ "source": "^.*/srv-api/(odata/v[24]/ott-core/.*)$",
  "destination": "onCapService" }

// Route ②③: 그 외 → onOTTService
{ "source": "^.*/srv-api/(.*ott.*)$",
  "destination": "onOTTService" }
{ "source": "^.*/srv-api/(.*)$",
  "destination": "onOTTService" }
```

| 서비스 경로 | Destination | 백엔드 |
|------------|-------------|--------|
| `/srv-api/odata/v4/ott-core/...` | `onCapService` | core_ott |
| `/srv-api/odata/v2/ott/Mypage/...` | `onOTTService` | ott |
| `/srv-api/odata/v2/detail/...` | `onOTTService` | ott |
| `/srv-api/odata/v2/ott/FreeBoard/...` | `onOTTService` | ott |

---

## ⚠️ 주의사항

| # | 주의사항 |
|---|---------|
| 1 | Destination 이름은 **대소문자 구분** — `onCapService` ≠ `oncapservice` |
| 2 | URL은 CF 앱이 배포된 실제 경로와 정확히 일치해야 함 |
| 3 | `클라이언트 ID`는 `xs-security.json`의 `xsappname`과 연결 (`sb-{xsappname}!{랜덤}`) |
| 4 | QA 환경에서는 Destination 이름이 `onOttCoreService` / `onOttService`로 다름 (xs-app-qa.json 기준) |

```
   │         │     ├── sysmgt_ott (detailPage, myPage, mainPage)

   │         │     ├── settlement
   │         │     ├── trendAnalysis
   │         │     └── library_ott (common_ott.lib)
   │         │
   │         └── API 프록시 → BTP Destination
   │               ├── onCapService  → core_ott (HDI #1, port 8084)
   │               └── onOTTService  → ott (HDI #2, port 8083)
   │
   └── [XSUAA] ← 인증
```
