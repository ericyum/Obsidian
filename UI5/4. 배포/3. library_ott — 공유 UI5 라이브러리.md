# 3. library_ott — 공유 UI5 라이브러리

> **작성일**: 2026-07-13
> **역할**: 모든 프론트엔드 앱이 공유하는 UI5 라이브러리. BaseController, 유틸리티 함수, 공통 서비스를 제공합니다. portal_ott, sysmgt_ott 등 다른 프론트엔드 앱보다 **먼저 배포**되어야 합니다.

---

## 📂 모듈 개요

```
cap-app/library_ott/
├── mta.yaml             ← MTA 배포 정의
├── package.json         ← npm 스크립트 + 의존성
├── ui5-deploy.yaml      ← UI5 빌드 설정 (배포용)
├── xs-app.json          ← 정적 파일 라우팅 (배포 시 zip에 포함)
└── src/                 ← 실제 라이브러리 코드
    ├── library.js       ← UI5 라이브러리 선언
    └── controller/
        └── BaseController.js  ← 모든 앱의 부모 컨트롤러
```

## 🔑 핵심 배포 파일들

### ① `mta.yaml` — MTA 배포 정의

```yaml
_schema-version: "3.2"
ID: common_ottlib
description: BTP Launchpad - Shared Library
version: 0.0.1
parameters:
  deploy_mode: html5-repo              # ← HTML5 Application Repository에 배포
  enable-parallel-deployments: true

modules:
  # 모듈 1: Content (html5-apps-repo에 zip 업로드)
  - name: com-lib-content
    type: com.sap.application.content
    path: .
    requires:
      - name: com-lib-repo-host
        parameters:
          content-target: true
    build-parameters:
      build-result: resources
      requires:
        - name: common_ottlib
          artifacts:
            - common_ottlib.zip          # ← ui5-task-zipper가 생성한 zip
          target-path: resources/

  # 모듈 2: 빌드 모듈 (UI5 번들링)
  - name: common_ottlib
    type: html5
    path: .
    build-parameters:
      build-result: dist
      builder: custom
      commands:
        - npm install
        - npm run build:cf               # ← ui5 build preload ...

resources:
  - name: com-lib-repo-host
    type: org.cloudfoundry.managed-service
    parameters:
      service: html5-apps-repo
      service-plan: app-host
      service-name: com-lib-repo-host-${space}
```

**다른 프론트엔드 모듈과 차이점**:

| 특징 | 설명 |
|------|------|
| `deploy_mode: html5-repo` | 독립 배포 가능 (approuter 없이도) |
| `type: library` | ui5-deploy.yaml에서 type이 "application"이 아닌 "library" |
| `service-name`에 `${space}` | dev/qa/prod 환경별 별도 인스턴스 |

> ⚠️ library_ott는 **approuter_ott의 mta.yaml에 포함되지 않은 유일한 모듈**입니다. 자체 mta.yaml로 **독립 배포**됩니다. 이유는 공유 라이브러리이기 때문에 다른 앱의 배포와 무관하게 항상 먼저 준비되어 있어야 하기 때문입니다.

### ② `package.json` — 빌드 스크립트

```json
{
  "name": "common_ottlib",
  "version": "0.0.1",
  "sapuxLayer": "VENDOR",             // ← SAP UX Layer: VENDOR (공유 라이브러리)
  "scripts": {
    "build": "npm run build:cf",
    "build:cf": "ui5 build preload --clean-dest --config ui5-deploy.yaml --include-task=generateCachebusterInfo",
    "deploy:mta": "mbt build && cf deploy mta_archives/common_ottlib_0.0.1.mtar"
  },
  "devDependencies": {
    "ui5-task-zipper": "^3.3.1"       // ← zip 압축 태스크 (필수)
  }
}
```

**배포 명령어**:
```bash
cd cap-app/library_ott
npm run deploy:mta
```

---

### ③ `ui5-deploy.yaml` — UI5 배포 빌드 설정

```yaml
specVersion: '3.0'
metadata:
  name: common_ottlib
type: library                           # ← "application"이 아닌 "library"!
resources:
  configuration:
    propertiesFileSourceEncoding: UTF-8
builder:
  resources:
    excludes:
      - "/test/**"
      - "/localService/**"
      - "**/node_modules/**"
  customTasks:
    - name: ui5-task-zipper
      afterTask: buildThemes            # ← 일반 앱은 "generateCachebusterInfo" 뒤지만
      configuration:                    #    라이브러리는 "buildThemes" 뒤
        archiveName: common_ottlib
        additionalFiles:
          - xs-app.json                 # ← zip에 xs-app.json도 포함
```

**일반 프론트엔드 앱과의 차이**:

| 항목 | library_ott | 일반 앱 (board 등) |
|------|:--:|:--:|
| `type` | `library` | `application` |
| `afterTask` | `buildThemes` | `generateCachebusterInfo` |
| 의존성 | `ui5-task-zipper`만 | `@sap/ux-ui5-tooling` 등 추가 |
| `sapuxLayer` | `VENDOR` | `CUSTOMER_BASE` |

**`type: library`의 의미**: UI5 빌더가 라이브러리 전용 빌드 파이프라인을 사용합니다. `library.js` 파일을 통해 어떤 모듈을 노출할지 선언하고, 테마 빌드도 포함됩니다.

---

### ④ `xs-app.json` — 정적 파일 라우팅

```json
{
  "authenticationMethod": "route",
  "routes": [
    {
      "source": "^(.*)$",
      "target": "$1",
      "service": "html5-apps-repo-rt",
      "authenticationType": "xsuaa"
    }
  ]
}
```

**역할**: 이 파일은 **ui5-task-zipper에 의해 zip 아카이브에 포함**되어 BTP html5-apps-repo에 함께 업로드됩니다. 런타임에 html5-apps-repo-rt 서비스가 이 파일을 읽어 정적 파일 서빙 방식을 결정합니다.

## 🔄 library_ott가 제공하는 것들

모든 프론트엔드 앱은 `common_ott.lib` 네임스페이스를 통해 이 라이브러리를 참조합니다:

```javascript
// detailPage/Component.js
jQuery.sap.registerModulePath("common_ott.lib", "/common_ott.lib/");

// index.html
data-sap-ui-resourceroots='{"common_ott.lib":"/common_ott.lib/"}'
```

제공하는 주요 자원:
- `common_ott.lib.controller.BaseController` — 모든 컨트롤러의 부모
- `readOdata()` — Promise 기반 OData read 래퍼
- `getResourceBundle()` — i18n 편의 메서드
- `fnDeepCopy()`, `fnMasterCodeValue()` 등 유틸리티

## 🔗 BTP에서의 URL 경로

```
로컬 (dev):
  /common_ott.lib/library.js
    → approuter localDir → library_ott/src/library.js

BTP:
  /common_ott.lib/library.js
    → approuter xs-app-dev.json 라우트 매칭
    → html5-apps-repo-rt
    → /common_ottlib-1.0.0/library.js
```

`mta.yaml` ID(`common_ottlib`) + version(`0.0.1`) 조합이 BTP Repository 경로가 됩니다. 단, `ui5-task-zipper`의 `archiveName`이 소문자 정규화되어 `common_ottlib`가 됩니다.
