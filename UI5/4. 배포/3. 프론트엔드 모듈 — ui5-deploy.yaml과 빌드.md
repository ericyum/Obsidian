# 프론트엔드 모듈 — ui5-deploy.yaml과 빌드

> **작성일**: 2026-07-13
> **목적**: 각 프론트엔드 HTML5 모듈의 빌드 설정과 MTA 통합 방식을 이해한다.

---

## 1. 공통 빌드 명령어

```json
// 각 모듈의 package.json
"build:cf": "ui5 build preload --clean-dest --config ui5-deploy.yaml --include-task=generateCachebusterInfo"
```

- `ui5 build preload`: Component-preload.js 번들 생성
- `--clean-dest`: dist/ 폴더 초기화
- `--config ui5-deploy.yaml`: 배포 전용 설정 사용
- `--include-task=generateCachebusterInfo`: 캐시 무효화 토큰 생성

---

## 2. ui5-deploy.yaml 공통 구조

```yaml
# 예: board/ui5-deploy.yaml
specVersion: '3.0'
metadata:
  name: board
type: application
resources:
  configuration:
    propertiesFileSourceEncoding: UTF-8
builder:
  resources:
    excludes:
      - "/test/**"
      - "/localService/**"
  customTasks:
    - name: ui5-task-zipper
      afterTask: generateCachebusterInfo
      configuration:
        archiveName: board
        additionalFiles:
          - xs-app.json
```

### ui5-task-zipper

빌드 결과물(`dist/`)을 **zip 아카이브**로 압축합니다. MTA 빌드가 이 zip을 HTML5 Application Repository에 업로드합니다.

| 모듈 | archiveName | 배포 경로 |
|------|------------|-----------|
| portal_ott | comportalott | `/comportalott-1.0.0/` |
| library_ott | common_ottlib | `/common_ottlib-1.0.0/` |
| sysmgt_ott | comsysmgtott | `/comsysmgtott-1.0.0/` |
| board | board | `/board-1.0.0/` |
| settlement | settlement | `/settlement-1.0.0/` |
| trendAnalysis | trendanalysis | `/trendanalysis-1.0.0/` |

> 주의: `archiveName`은 소문자로 정규화되며, `mta.yaml`의 `ID`-`version` 조합이 최종 URL 경로가 됩니다.

## 3. MTA 통합 — 각 모듈이 approuter_ott에 연결되는 방식

```yaml
# approuter_ott/mta.yaml (일부)
modules:
  - name: com-board-content
    type: com.sap.application.content
    requires:
      - name: com-board-repo-host
        parameters:
          content-target: true
    build-parameters:
      requires:
        - name: board
          artifacts:
            - board.zip
          target-path: resources/

  - name: board
    type: html5
    path: ../board              # ← 각 모듈의 실제 경로
    build-parameters:
      build-result: dist
      builder: custom
      commands:
        - npm install
        - npm run build:cf      # ← ui5 build preload ...
```

**흐름**:
```
board/ 디렉토리
  → npm install
  → npm run build:cf (ui5 build preload)
  → dist/ 생성 (Component-preload.js 포함)
  → ui5-task-zipper → board.zip
  → approuter_ott/mta.yaml → resources/board.zip
  → mbt build → MTA 아카이브에 포함
  → cf deploy → html5-apps-repo-rt에 업로드
```

## 4. sysmgt_ott 특이사항 — 중첩 앱 구조

sysmgt_ott는 단일 모듈이지만 내부에 여러 개의 독립 앱을 포함합니다:

```
sysmgt_ott/webapp/
├── detailPage/webapp/    ← detailPage.Component
├── myPage/webapp/        ← myPage.Component
└── mainPage/webapp/      ← mainPage.Component
```

`ui5 build preload`는 sysmgt_ott 루트의 `Component-preload.js`만 생성합니다. **각 하위 앱(detailPage, myPage, mainPage)의 Component-preload.js는 생성되지 않습니다.** BTP에서 UI5가 이 파일을 요청하면 404가 발생하지만, UI5는 자동으로 개별 파일로 폴백하므로 큰 문제는 아닙니다.

## 5. 핵심 요약

| # | 내용 |
|---|------|
| 1 | 각 모듈은 `ui5 build preload`로 번들링 → `ui5-task-zipper`로 zip 압축 |
| 2 | `mta.yaml`의 `artifacts`로 zip을 참조 → MTA 아카이브에 포함 |
| 3 | `archiveName`이 html5-apps-repo-rt의 배포 경로를 결정 |
| 4 | sysmgt_ott의 하위 앱은 개별 Component-preload.js가 생성되지 않음 (UI5 폴백) |
| 5 | `build:cf` 명령어는 모든 모듈에서 동일한 패턴 |
