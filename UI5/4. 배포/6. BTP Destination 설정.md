# BTP Destination 설정

> **작성일**: 2026-07-13
> **목적**: BTP Cockpit의 Destination 설정 구조와 각 destination의 역할을 이해한다.

---

## 1. Destination 목록

| 이름 | URL | 용도 |
|------|-----|------|
| `onCapService` | `https://org-...-com-cap-ott-core.cfapps.jp10.hana.ondemand.com` | core_ott 백엔드 (인증, 메뉴, 사용자) |
| `onOTTService` | `https://org-...-com-cap-ott.cfapps.jp10.hana.ondemand.com` | ott 백엔드 (콘텐츠, 멤버십, 정산) |

---

## 2. 공통 설정

```
유형: HTTP
프록시 유형: Internet
인증: OAuth2JWTBearer
클라이언트 ID: sb-com-app-xsapp!t50732
토큰 서비스 URL: https://tf-joule-1on6yytw.authentication.jp10.hana.ondemand.com/oauth/token
토큰 서비스 URL 유형: Dedicated
추가 속성: tokenType = Bearer
```

---

## 3. OAuth2JWTBearer 동작 방식

```
① 사용자가 portal_ott 접속 → XSUAA 로그인 → JWT 토큰 발급
② 사용자가 OData API 호출 (예: /srv-api/odata/v2/ott/Mypage)
③ approuter: xs-app-dev.json 라우트 매칭 → destination 확인
④ approuter: 사용자 JWT를 bearer assertion으로 XSUAA token service에 제시
⑤ XSUAA: 새 JWT 토큰 발급 (클라이언트 ID 기반)
⑥ approuter: 새 JWT를 Authorization 헤더에 담아 destination으로 프록시
⑦ CAP 백엔드: JWT 검증 → req.user 설정 → 요청 처리
```

---

## 4. xs-app-dev.json과 Destination 연결

```json
// Route ①: ott-core → onCapService (core_ott)
{ "source": "^.*/srv-api/(odata/v[24]/ott-core/.*)$",
  "destination": "onCapService" }

// Route ②③: 그 외 → onOTTService (ott)
{ "source": "^.*/srv-api/(.*ott.*)$",
  "destination": "onOTTService" }
{ "source": "^.*/srv-api/(.*)$",
  "destination": "onOTTService" }
```

| 서비스 | Destination | 백엔드 |
|--------|-------------|--------|
| UserManagement, MenuManagement, CommonCode | `onCapService` | core_ott |
| DetailService, MembershipService, FreeBoard | `onOTTService` | ott |

---

## 5. 주의사항

- Destination 이름은 **대소문자 구분**. 오타가 있으면 approuter가 destination을 찾지 못하고 500 에러 발생
- `URL`은 CF 앱이 배포된 실제 경로와 정확히 일치해야 함
- `클라이언트 ID`는 `xs-security.json`의 `xsappname`과 연결됨
