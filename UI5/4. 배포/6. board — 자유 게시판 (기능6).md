# 6. board — 자유 게시판 (기능6)

> **작성일**: 2026-07-13
> **역할**: OTT 사용자 간 자유 게시판 기능. 표준적인 UI5 애플리케이션으로, 가장 단순한 배포 구조를 가지고 있습니다.

---

## 📂 모듈 개요

```
cap-app/board/
├── mta.yaml             ← MTA 배포 정의
├── package.json         ← npm 스크립트
├── ui5-deploy.yaml      ← UI5 빌드 설정
├── ui5.yaml             ← 로컬 개발용 설정
├── xs-app.json          ← 정적 파일 라우팅
└── webapp/              ← 앱 코드
```

---

## 🔑 핵심 배포 파일들

### ① `mta.yaml` — MTA 배포 정의

```yaml
_schema-version: "3.2"
ID: comboard
version: 1.0.0
parameters:
  deploy_mode: html5-repo
  enable-parallel-deployments: true

modules:
  - name: com-board-content
    type: com.sap.application.content
    path: .
    requires:
      - name: com-board-repo-host
        parameters:
          content-target: true
    build-parameters:
      build-result: resources
      requires:
        - name: board
          artifacts:
            - board.zip
          target-path: resources/

  - name: board
    type: html5
    path: .
    build-parameters:
      build-result: dist
      builder: custom
      commands:
        - npm install
        - npm run build:cf

resources:
  - name: com-board-repo-host
    type: org.cloudfoundry.managed-service
    parameters:
      service: html5-apps-repo
      service-plan: app-host
      service-key: com-board-repo-host-key
```

> `service-key`가 하드코딩되어 있습니다. `${space}`를 사용하지 않으므로 dev/qa/prod 환경 구분이 되지 않습니다. 멀티 환경 배포가 필요하면 `service-name: com-board-repo-host-${space}`로 변경하는 것이 좋습니다.

### ② `package.json` — 빌드 스크립트

```json
{
  "name": "comboard",
  "version": "1.0.0",
  "description": "OTT 자유 게시판 (기능6)",
  "scripts": {
    "start:ui": "ui5 serve --port 5003",
    "build:cf": "ui5 build preload --clean-dest --config ui5-deploy.yaml --include-task=generateCachebusterInfo",
    "deploy:mta": "mbt build && cf deploy mta_archives/comboard_1.0.0.mtar"
  },
  "devDependencies": {
    "@sap/ux-ui5-tooling": "1",
    "ui5-task-zipper": "^3.3.1",
    "@sap/ui5-builder-webide-extension": "^1.1.9"   // ← devDependency에 있음
  }
}
```

---

### ③ `ui5-deploy.yaml` — UI5 배포 빌드 설정

```yaml
specVersion: '3.0'
metadata:
  name: board
type: application
resources:
  configuration:
    propertiesFileSourceEncoding: UTF-8
builder:
  resources:
    excludes:
      - "/test/**"
      - "/localService/**"
  customTasks:
    - name: ui5-task-zipper
      afterTask: generateCachebusterInfo
      configuration:
        archiveName: board
        additionalFiles:
          - xs-app.json
```

> ✅ board의 `ui5-deploy.yaml`은 **깨끗합니다**. `webide-extension-task-updateManifestJson`가 없습니다. `@sap/ui5-builder-webide-extension`이 package.json devDependencies에 있지만, ui5-deploy.yaml에서 참조되지 않으면 실행되지 않으므로 안전합니다.

---

### ④ `ui5.yaml` — 로컬 개발용 설정

```yaml
specVersion: "3.1"
metadata:
  name: comboard
type: application
framework:
  name: SAPUI5
  version: "1.136.0"
  libraries:
    - name: sap.m
    - name: sap.ui.core
    - name: sap.ui.unified
```

**최소한의 설정**: 로컬 개발 시 fiori-tools-proxy 등 없이도 SAPUI5 CDN만으로 실행 가능한 단순 구조입니다.

## 🔗 approuter와의 연결

```json
// approuter_ott/xs-app-dev.json
{
  "source": "^.*/board/~[0-9]+~/(.*)$",
  "target": "/board-1.0.0/$1",
  "service": "html5-apps-repo-rt",
  "authenticationType": "xsuaa"
},
{
  "source": "^.*/board/(.*)$",
  "target": "/board-1.0.0/$1",
  "service": "html5-apps-repo-rt",
  "authenticationType": "xsuaa"
}
```

BTP에서 board는 `/board-1.0.0/` 경로로 서빙됩니다 (`mta.yaml`의 `ID`-`version` 조합).

## 📊 board의 배포 특징 (다른 모듈과 비교)

| 항목 | board | sysmgt_ott | portal_ott |
|------|-------|------------|------------|

## ⚠️ 배포 시 주의사항

| # | 주의사항 |
|---|---------|
| 1 | `service-key`가 하드코딩 — 멀티 환경 필요시 `${space}`로 변경 |
| 2 | `ui5-deploy.yaml`은 깨끗함 — `webide-extension` 없음 확인 완료 |
| 3 | board는 approuter_ott의 **통합 배포**에 포함 (자체 mta.yaml도 있지만, approuter의 mta.yaml로도 배포됨) |
