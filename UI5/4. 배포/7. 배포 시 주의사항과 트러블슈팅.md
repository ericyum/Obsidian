# 배포 시 주의사항과 트러블슈팅

> **작성일**: 2026-07-13
> **목적**: BTP 배포 과정에서 발생한 실제 문제들과 해결 방법을 기록한다.

---

## 1. `webide-extension-task-updateManifestJson` — OData URI 파괴

### 증상
모든 OData 호출이 404. Network 탭에 전혀 엉뚱한 URL로 요청이 감.

### 원인
`ui5-deploy.yaml`의 `webide-extension-task-updateManifestJson`가 `manifest.json`의 URI에서 맨 앞 `/`를 제거해 상대경로로 만듦.

### 해결
해당 customTask를 `ui5-deploy.yaml`에서 제거.

> 상세: `4. webide-extension-task-updateManifestJson — 절대경로 파괴 버그.md`

---

## 2. BTP Destination 이름 오타

### 증상
MenuLoader 500 에러. `$metadata` 호출 실패.

### 원인
BTP Cockpit의 Destination 이름이 `xs-app-dev.json`에서 참조하는 이름과 불일치 (오타).

### 해결
Cockpit에서 Destination 이름 확인 후 수정. 이름은 **대소문자 구분**됨에 주의.

## 3. `@restrict`로 인한 빈 데이터

### 증상
myPage에서 레이아웃은 보이지만 데이터가 나오지 않음 (빈 화면). 콘솔에 OData 에러 없음.

### 원인
`MembershipService.cds`의 `UserSet`에 `@restrict: user_id = $user` 적용. BTP에서 `$user`는 XSUAA 이메일인데, ott의 Users 테이블에 해당 이메일 레코드가 없음.

### 해결
ott Users 테이블에 BTP 사용자 이메일로 레코드 추가.

> 상세: `5. 백엔드 — CAP 배포와 HANA CSV 데이터.md`

## 4. `Component-preload.js` 404 (무시 가능)

### 증상
중첩 컴포넌트(detailPage, myPage, mainPage) 로드 시 콘솔에 `Component-preload.js 404` 발생.

### 원인
`ui5 build preload`는 sysmgt_ott 루트의 Component-preload.js만 생성. 하위 앱(detailPage/webapp/ 등)의 번들은 생성하지 않음.

### 결론
**무시 가능.** UI5는 preload 실패 시 자동으로 개별 파일로 폴백하여 정상 작동한다. `Component-preload.js`는 성능 최적화용 번들이며, 없어도 기능에 지장이 없다.

## 5. `menu_display_flag`가 DB에서 `false`인데도 메뉴에 보임

### 증상
Menu.csv에서 `menu_display_flag = false`로 설정하고 core_ott를 재배포했는데, 상세 페이지가 여전히 메뉴바에 표시됨.

### 원인
`Layout.controller.js`의 `buildTree()` 함수가 `menu_display_flag`를 확인하지 않음. DB 값이 프론트엔드 코드에서 전혀 사용되지 않음.

### 해결
`setHeaderNavigation()`과 `createSideNavigation()`에서 `buildTree()` 호출 전에 필터링 추가:

```javascript
const filteredMenuList = aMenuList.filter(item => item.menu_display_flag !== false);
const oMenuTree = this.buildTree(filteredMenuList, aFavoriteMenuList);
```

portal_ott 재배포 필요.

## 6. `x-forwarded-for: "undefined"` 500 에러

### 증상
로컬에서 `$batch` OData 요청이 500으로 실패.

### 원인
approuter가 로컬 dev 모드에서 `x-forwarded-for: "undefined"` 헤더를 전송. CAP이 이 값을 잘못된 IP로 해석해 거부.

### 해결
`core_ott/srv/server.ts`와 `ott/srv/server.ts`에 패치 추가:

```typescript
if (req.headers['x-forwarded-for'] === 'undefined') {
    delete req.headers['x-forwarded-for'];
}
```
