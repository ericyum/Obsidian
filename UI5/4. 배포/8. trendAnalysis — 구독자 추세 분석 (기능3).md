## 🔑 핵심 배포 파일들

### ① `mta.yaml` — MTA 배포 정의

```yaml
_schema-version: "3.2"
ID: comtrendanalysis
version: 1.0.0
parameters:
  deploy_mode: html5-repo
  enable-parallel-deployments: true

modules:
  - name: com-trendanalysis-content
    type: com.sap.application.content
    path: .
    requires:
      - name: com-trendanalysis-repo-host
        parameters:
          content-target: true
    build-parameters:
      build-result: resources
      requires:
        - name: trendanalysis
          artifacts:
            - trendanalysis.zip
          target-path: resources/

  - name: trendanalysis
    type: html5
    path: .
    build-parameters:
      build-result: dist
      builder: custom
      commands:
        - npm install
        - npm run build:cf

resources:
  - name: com-trendanalysis-repo-host
    type: org.cloudfoundry.managed-service
    parameters:
      service: html5-apps-repo
      service-plan: app-host
      service-key: com-trendanalysis-repo-host-key   # ← 하드코딩
```

### ② `package.json` — 빌드 스크립트

```json
{
  "name": "trendAnalysis",
  "version": "1.0.0",
  "description": "OTT 구독자 추세 분석 대시보드 (기능3) — Admin 전용",
  "scripts": {
    "build": "ui5 build -a --clean-dest",
    "build:cf": "ui5 build preload --clean-dest --config ui5-deploy.yaml --include-task=generateCachebusterInfo",
    "deploy:mta": "mbt build && cf deploy mta_archives/comtrendanalysis_1.0.0.mtar",
    "start": "fiori run --open \"test/flpSandbox.html?sap-ui-xx-viewCache=false#trendAnalysis-tile\"",
    "start-local": "fiori run --config ./ui5-local.yaml --open \"...\""
  },
  "devDependencies": {
    "@sap/ux-ui5-tooling": "1",
    "ui5-task-zipper": "^3.3.1",
    "@sap/ui5-builder-webide-extension": "^1.1.9"
  }
}
```

**trendAnalysis의 특이한 점**: `start` 스크립트가 `fiori run`을 사용하며 FLP Sandbox에서 실행됩니다. 다른 모듈들은 `ui5 serve`를 사용하는 것과 대비됩니다.

### ③ `ui5-deploy.yaml` — UI5 배포 빌드 설정

```yaml
specVersion: '3.0'
metadata:
  name: trendanalysis
type: application
resources:
  configuration:
    propertiesFileSourceEncoding: UTF-8
builder:
  resources:
    excludes:
      - "/test/**"
      - "/localService/**"
  customTasks:
    - name: ui5-task-zipper
      afterTask: generateCachebusterInfo
      configuration:
        archiveName: trendanalysis
        additionalFiles:
          - xs-app.json
```

> ✅ `webide-extension-task-updateManifestJson` 없음. 깨끗합니다.

### ④ `ui5.yaml` — 로컬 개발용 설정

```yaml
specVersion: "3.1"
metadata:
  name: trendanalysis
type: application
server:
  customMiddleware:
    - name: fiori-tools-proxy
      afterMiddleware: compression
      configuration:
        backend:
          - path: /srv-api
            url: http://localhost:8083        # ← ott 백엔드 (8083)
            pathReplace: "/"
    - name: fiori-tools-preview
      afterMiddleware: fiori-tools-appreload
      configuration:
        component: trendAnalysis
        ui5Theme: sap_horizon
    - name: fiori-tools-servestatic
      afterMiddleware: compression
      configuration:
        paths:
          - path: /common.lib
            src: "../library_ott/src"          # ← common.lib → library_ott (실수 가능성)
```

> ⚠️ `fiori-tools-servestatic`에서 `/common.lib`를 `../library_ott/src`로 매핑하고 있습니다. 원래 `/common_ott.lib`이어야 하지만, `/common.lib`으로 잘못 기재되었을 가능성이 있습니다. (템플릿 코드 그대로 남아있음)

## 🔗 approuter와의 연결

```json
// approuter_ott/xs-app-dev.json
{
  "source": "^.*/trendAnalysis/~[0-9]+~/(.*)$",
  "target": "/trendanalysis-1.0.0/$1",
  "service": "html5-apps-repo-rt",
  "authenticationType": "xsuaa"
},
{
  "source": "^.*/trendAnalysis/(.*)$",
  "target": "/trendanalysis-1.0.0/$1",
  "service": "html5-apps-repo-rt",
  "authenticationType": "xsuaa"
}
```

BTP에서 trendAnalysis는 `/trendanalysis-1.0.0/` 경로로 서빙됩니다. `mta.yaml`의 ID는 `comtrendanalysis`지만 `ui5-task-zipper`의 `archiveName`은 `trendanalysis`(소문자)입니다. 최종 URL 경로는 `archiveName`이 아닌 `ID-version` 조합으로 결정되므로 `/trendanalysis-1.0.0/`가 됩니다.
