# 9. approuter_ott — 배포 총괄 오케스트레이터

> **작성일**: 2026-07-13
> **역할**: ZEN-OTT 전체 배포의 **진입점이자 오케스트레이터**. 모든 프론트엔드 모듈의 MTA 빌드를 총괄하고, BTP에서 HTTP 라우팅·인증·정적 파일 서빙을 담당합니다. **가장 마지막에 배포**되지만, 가장 먼저 이해해야 하는 모듈입니다.

## 📂 모듈 개요

```
cap-app/approuter_ott/
```

## 🔑 핵심 배포 파일들

### ① `package.json` — 배포 스크립트와 xs-app 덮어쓰기

```json
{
  "name": "approuter",
  "dependencies": {
    "@sap/approuter": "^20.0.1"         // ← SAP AppRouter (Node.js)
  },
  "scripts": {
    "start": "node node_modules/@sap/approuter/approuter.js",
    "start:local": "node node_modules/@sap/approuter/approuter.js --workingDir ./dev --port 5000",

    "build:cf": "mbt build",            // ← 프로덕션 빌드 (xs-app.json 그대로)
    "build:dev": "node -e \"require('fs').copyFileSync('xs-app-dev.json','xs-app.json')\" && mbt build",
                                         // ← ★ 개발 빌드 (xs-app-dev.json → xs-app.json 덮어쓰기!)

    "deploy:cf": "cf deploy mta_archives/com-approuterott_1.0.0.mtar",
    "deploy:mta": "npm run build:dev && npm run deploy:cf"
                                         // ← ★ 통합 배포 명령어
  }
}
```

**핵심 메커니즘 — xs-app.json 덮어쓰기**:

```
개발자가 수정하는 파일:  xs-app-dev.json  (BTP 개발 환경용)
                        xs-app-qa.json   (BTP QA 환경용)

            build:dev 실행 시
                 │
    node -e "copyFileSync('xs-app-dev.json', 'xs-app.json')"
                 │
                 ▼
        xs-app.json (빌드 시점에 덮어써짐)
                 │
            mbt build
                 │
                 ▼
        MTA 아카이브에 포함 → cf deploy
```

**루트의 `xs-app.json`은 의미가 없습니다.** `build:dev` 스크립트가 `xs-app-dev.json`으로 완전히 덮어쓰기 때문입니다. `xs-app.json`을 직접 수정해도 배포 시 사라집니다.

이렇게 한 이유:
- 개발 환경(dev)과 QA 환경(qa) 각각 다른 라우팅 규칙 필요
- `build:dev` → `xs-app-dev.json` 사용 / `build:qa` → `xs-app-qa.json` 사용 (필요시 스크립트 추가)
- 유동적으로 테스트 가능

### ② `mta.yaml` — ★ 전체 배포의 설계도

```yaml
ID: com-approuterott
_schema-version: "3.2"
version: 1.0.0

modules:
  # ===== 모듈 1: approuter 자체 (Node.js) =====
  - name: com-approuterott
    type: approuter.nodejs
    path: .
    properties:
      IAS_XSUAA_XCHANGE_ENABLED: true
      COOKIE_BACKWARD_COMPATIBILITY: true
    requires:
      - name: com-app-repo-runtime       # ← html5-apps-repo (정적 파일)
      - name: com-app-destination        # ← BTP Destination 서비스
      - name: com-app-xsuaa              # ← XSUAA 인증
      - name: com-app-connectivity       # ← Connectivity (Cloud Connector)

  # ===== 모듈 2~: 각 프론트엔드 앱 (content + build) =====
  # sysmgt_ott 예시 (board, settlement, trendAnalysis, portal_ott도 동일 패턴)
  - name: com-sysmgtott-content          # ← Content 모듈 (zip 업로드 담당)
    type: com.sap.application.content
    path: .
    requires:
      - name: com-sysmgtott-repo-host
        parameters:
          content-target: true
    build-parameters:
      build-result: resources
      requires:
        - name: comsysmgtott
          artifacts:
            - comsysmgtott.zip
          target-path: resources/

  - name: comsysmgtott                   # ← Build 모듈 (UI5 번들링)
    type: html5
    path: ../sysmgt_ott                  # ← 상대경로로 각 앱 참조
    build-parameters:
      build-result: dist
      builder: custom
      commands:
        - npm install
        - npm run build:cf               # ← ui5 build preload ...
      supported-platforms: []

resources:
  # BTP 서비스 인스턴스 생성
  - name: com-app-repo-runtime
    type: org.cloudfoundry.managed-service
    parameters:
      service: html5-apps-repo
      service-plan: app-runtime
      service-name: com-app-repo-runtime-${space}

  - name: com-app-destination
    type: org.cloudfoundry.managed-service
    parameters:
      service: destination
      service-plan: lite

  - name: com-app-xsuaa
    type: org.cloudfoundry.managed-service
    parameters:
      path: ./xs-security.json           # ← XSUAA 설정
      service-plan: application
      service: xsuaa
      config:
        xsappname: 'com-app-xsapp'       # ← 고정 xsappname (기존 인스턴스 호환)

  - name: com-app-connectivity
    type: org.cloudfoundry.managed-service
    parameters:
      service: connectivity
      service-plan: lite

  # 각 앱의 repo-host
  - name: com-sysmgtott-repo-host
    type: org.cloudfoundry.managed-service
    parameters:
      service: html5-apps-repo
      service-plan: app-host
      service-name: com-sysmgtott-repo-host-${space}
  # (board, settlement, trendAnalysis, portal_ott도 동일 패턴)
```

**approuter_ott의 mta.yaml에 포함된 앱들**:

| 모듈 | path | 비고 |
|------|------|------|
| approuter 자체 | `.` | Node.js approuter |
| portal_ott | `../portal_ott` | 포털 셸 |
| sysmgt_ott | `../sysmgt_ott` | detailPage, myPage, mainPage |
| board | `../board` | 자유 게시판 |
| settlement | `../settlement` | 정산 관리 |
| trendAnalysis | `../trendAnalysis` | 추세 분석 |

> ⚠️ **library_ott는 포함되지 않음!** library_ott는 자체 mta.yaml로 **독립 배포**해야 합니다.

**배포 시 BTP 서비스 구조**:
```
com-approuterott (approuter)
  ├── com-app-repo-runtime   ← html5-apps-repo (정적 파일 서빙)
  ├── com-app-destination    ← BTP Destination (백엔드 주소록)
  ├── com-app-xsuaa          ← XSUAA (사용자 인증)
  └── com-app-connectivity   ← Connectivity (Cloud Connector)
```

### ③ `xs-app-dev.json` — ★★★ 실제 BTP 라우팅 규칙

이 파일이 **실제 배포되는 라우팅 규칙**입니다. `build:dev`로 빌드하면 이 내용이 `xs-app.json`을 덮어씁니다.

```json
{
  "welcomeFile": "/index.html",
  "sessionTimeout": 60,
  "authenticationMethod": "route",
  "logout": {
    "logoutEndpoint": "/user/logout",
    "logoutPage": "/"
  },
  "errorPage": [
    {"status": [400,401,402,403,404], "file": "/error/404.html"},
    {"status": [500,501,502,503], "file": "/error/500.html"}
  ],
  "routes": [
    // ===== API 라우트 (3-way 분기) =====

    // ① ott-core 경로 → core_ott 백엔드
    {
      "source": "^.*/srv-api/(odata/v[24]/ott-core/.*)$",
      "target": "$1",
      "destination": "onCapService",
      "authenticationType": "none",
      "csrfProtection": false
    },

    // ② "ott" 포함 경로 → ott 백엔드
    {
      "source": "^.*/srv-api/(.*ott.*)$",
      "target": "$1",
      "destination": "onOTTService",
      "authenticationType": "none",
      "csrfProtection": false
    },

    // ③ 그 외 모든 API → ott 백엔드 (XSUAA 인증)
    {
      "source": "^.*/srv-api/(.*)$",
      "target": "$1",
      "destination": "onOTTService",
      "authenticationType": "xsuaa",
      "csrfProtection": false
    },

    // ===== 정적 파일 라우트 (html5-apps-repo-rt) =====

    // portal_ott
    { "source": "^.*/portal_ott/~[0-9]+~/(.*)$",
      "target": "/comportalott-1.0.0/$1",
      "service": "html5-apps-repo-rt", "authenticationType": "xsuaa" },
    { "source": "^.*/portal_ott/(.*)$",
      "target": "/comportalott-1.0.0/$1",
      "service": "html5-apps-repo-rt", "authenticationType": "xsuaa" },

    // library_ott (경로에 공백 없음!)
    { "source": "/common_ott.lib/(.*)",
      "target": "/common_ottlib-1.0.0/$1",
      "service": "html5-apps-repo-rt", "authenticationType": "xsuaa" },

    // sysmgt_ott
    { "source": "^.*/sysmgt_ott/~[0-9]+~/(.*)$",
      "target": "/comsysmgtott-1.0.0/$1",
      "service": "html5-apps-repo-rt", "authenticationType": "xsuaa" },
    { "source": "^.*/sysmgt_ott/(.*)$",
      "target": "/comsysmgtott-1.0.0/$1",
      "service": "html5-apps-repo-rt", "authenticationType": "xsuaa" },

    // board, settlement, trendAnalysis (동일 패턴)
    ...

    // ===== Catch-all =====
    {
      "source": "(.*)",
      "target": "/comportalott-1.0.0/$1",
      "service": "html5-apps-repo-rt",
      "authenticationType": "xsuaa"
    }
  ]
}
```

**라우트 우선순위 (위에서 아래로)**:

```
1. /srv-api/odata/v4/ott-core/...  → onCapService (core_ott)
2. /srv-api/...ott...              → onOTTService (ott) — 인증 없음
3. /srv-api/...                    → onOTTService (ott) — XSUAA 인증
4. /portal_ott/...                 → html5-apps-repo-rt
5. /common_ott.lib/...             → html5-apps-repo-rt
6. /sysmgt_ott/...                 → html5-apps-repo-rt
7. /board/...                      → html5-apps-repo-rt
8. /settlement/...                 → html5-apps-repo-rt
9. /trendAnalysis/...              → html5-apps-repo-rt
10. (.*)                           → portal_ott (catch-all)
```

### ④ `xs-app-qa.json` — QA 환경용 라우팅

```json
{
  // xs-app-dev.json과 거의 동일하지만:
  "routes": [
    {
      "source": "^.*/srv-api/(odata/v[24]/ott-core/.*)$",
      "target": "$1",
      "destination": "onOttCoreService",     // ← dev는 "onCapService", QA는 "onOttCoreService"
      "authenticationType": "xsuaa",         // ← dev는 "none", QA는 "xsuaa"
      "csrfProtection": false
    },
    {
      "source": "^.*/srv-api/(.*)$",
      "target": "$1",
      "destination": "onOttService",         // ← 같은 destination
      "authenticationType": "xsuaa",         // ← dev는 일부 "none", QA는 전부 "xsuaa"
      "csrfProtection": false
    }
    // ...
  ]
}
```

**dev vs qa 차이**:
| 항목 | xs-app-dev.json | xs-app-qa.json |
|------|:--:|:--:|
| Destination 이름 | `onCapService` / `onOTTService` | `onOttCoreService` / `onOttService` |
| API 인증 | 일부 `none` | 전부 `xsuaa` |
| 사용 목적 | 개발 테스트 (인증 완화) | QA 검증 (실제 인증) |

### ⑤ `xs-security.json` — XSUAA 인증 설정

```json
{
  "xsappname": "com-app-xsapp",
  "tenant-mode": "dedicated",
  "scopes": [
    {
      "name": "$XSAPPNAME.Admin",
      "description": "OTT 관리자 권한"
    }
  ],
  "role-templates": [
    {
      "name": "Admin",
      "description": "OTT 관리자 (정산·구독자 추세 분석)",
      "scope-references": ["$XSAPPNAME.Admin"]
    }
  ],
  "role-collections": [
    {
      "name": "OTT_Settlement_Admin",
      "description": "OTT 관리자. BTP Cockpit에서 실제 관리자 계정에 할당",
      "role-template-references": ["$XSAPPNAME.Admin"]
    }
  ],
  "oauth2-configuration": {
    "token-validity": 43200,
    "refresh-token-validity": 43200,
    "redirect-uris": [
      "https://*.cfapps.jp10.hana.ondemand.com/**",
      "https://*.jp10.applicationstudio.cloud.sap/**"
    ]
  }
}
```

**핵심**:
- `xsappname: "com-app-xsapp"` — 이 이름이 BTP Destination의 **Client ID**가 됩니다 (`sb-com-app-xsapp!t50732`)
- `role-collections` — BTP Cockpit에서 사용자에게 할당하는 권한 그룹
- `redirect-uris` — 로그인 후 리다이렉트 허용 도메인

### ⑥ `dev/` — 로컬 개발 환경

#### `dev/default-env.json` (로컬 Destination)
```json
{
  "destinations": [
    { "name": "onOttService", "url": "http://localhost:8083" },
    { "name": "onOttCoreService", "url": "http://localhost:8084" }
  ]
}
```

로컬에서 approuter를 실행할 때 BTP Destination 대신 이 파일을 사용합니다.

#### `dev/xs-app.json` (로컬 라우팅)
```json
{
  "authenticationMethod": "none",
  "routes": [
    // API → localhost destinations
    { "source": "^.*/srv-api/(.*ott-core/.*)$", "destination": "onOttCoreService" },
    { "source": "^.*/srv-api/(.*)$", "destination": "onOttService" },

    // 정적 파일 → localDir (디스크 직접 읽기)
    { "source": "/common_ott.lib/(.*)", "localDir": "../../library_ott/src" },
    { "source": "^(.*)/sysmgt_ott/(.*)$", "localDir": "../../sysmgt_ott/webapp" },
    { "source": "^(.*)/portal_ott/(.*)$", "localDir": "../../portal_ott/webapp" },
    // ...
  ]
}
```

**로컬 vs BTP**:

| | 로컬 (`dev/xs-app.json`) | BTP (`xs-app-dev.json`) |
|---|---|---|
| 정적 파일 | `localDir`: 디스크 직접 읽기 | `service: "html5-apps-repo-rt"`: Repository |
| Destination | `default-env.json` (localhost) | BTP Cockpit Destination 서비스 |
| 인증 | `"none"` → CAP mocked auth | `"xsuaa"` → 실제 XSUAA |

### `deploy:mta` 내부 단계

```
① npm run build:dev
   ├─ xs-app-dev.json → xs-app.json (덮어쓰기!)
   └─ mbt build
        │
        ├── approuter_ott 자체 빌드
        ├── portal_ott → npm install → npm run build:cf → dist/ → zip
        ├── sysmgt_ott → npm install → npm run build:cf → dist/ → zip
        ├── board → npm install → npm run build:cf → dist/ → zip
        ├── settlement → npm install → npm run build:cf → dist/ → zip
        └── trendAnalysis → npm install → npm run build:cf → dist/ → zip
        │
        └──→ mta_archives/com-approuterott_1.0.0.mtar 생성

② npm run deploy:cf
   └─ cf deploy mta_archives/com-approuterott_1.0.0.mtar
        │
        ├── html5-apps-repo-rt에 각 앱의 zip 업로드
        ├── approuter Node.js 앱 배포
        └── XSUAA/Destination/Connectivity 서비스 바인딩
```

## 🔗 BTP Destination 설정

approuter_ott의 `xs-app-dev.json`이 참조하는 Destination은 **BTP Cockpit에서 별도로 설정**해야 합니다:

| Destination | URL (예시) | 연결 백엔드 |
|-------------|-----------|------------|
| `onCapService` | `https://org-...-com-cap-ott-core.cfapps.jp10.hana.ondemand.com` | core_ott |
| `onOTTService` | `https://org-...-com-cap-ott.cfapps.jp10.hana.ondemand.com` | ott |

**공통 설정**:
```
유형: HTTP
인증: OAuth2JWTBearer
클라이언트 ID: sb-com-app-xsapp!t50732
토큰 서비스 URL: https://tf-joule-...authentication.jp10.hana.ondemand.com/oauth/token
```

## ⚠️ 배포 시 주의사항

| # | 주의사항 |
|---|---------|
| 1 | **`xs-app.json`은 직접 수정하지 말 것** — `xs-app-dev.json`을 수정해야 함 |
| 2 | `build:dev`와 `build:cf`의 차이: 전자는 xs-app 덮어쓰기 + mbt build, 후자는 mbt build만 |
| 3 | **library_ott는 approuter mta.yaml에 포함되지 않음** — 별도 배포 필수 |
| 4 | BTP Destination 이름이 `xs-app-dev.json`과 **정확히 일치**해야 함 (대소문자 구분) |
| 5 | `xsappname` 변경 시 기존 XSUAA 인스턴스와 충돌 — 변경 신중히 |
| 6 | `authenticationType: "none"`인 라우트는 인증 없이 백엔드 접근 가능 → 보안 주의 |
| 7 | Catch-all `(.*)` → portal_ott로 모든 미매칭 요청이 portal로 감 |
