## 1. `sap.m` — 기본 UI 컨트롤 (가장 많이 사용)

### 레이아웃 컨테이너

| 컨트롤 | 용도 | 사용 예시 |
|--------|------|----------|
| **`HBox`** | 가로 방향 박스 (flex row) | `alignItems="Center"`로 수직 중앙 정렬. Label+Value 나란히 배치 |
| **`VBox`** | 세로 방향 박스 (flex column) | Card 그룹, 폼 필드, 리뷰 아이템 내부 구조 |
| **`FlexBox`** | 고급 flex 컨테이너 | `wrap="Wrap"` → 반응형 카드 그리드. `items` aggregation으로 배열 반복 렌더링 |
| **`Page`** | 전체 페이지 셸 | PlanChange에서만 사용. `showNavButton="true"` → 뒤로가기 버튼 자동 생성 |

### 텍스트·레이블

| 컨트롤 | 용도 | 주요 속성 |
|--------|------|----------|
| **`Title`** | 제목 텍스트 | `level="H2"/"H3"/"H6"` (HTML heading 레벨) |
| **`Text`** | 일반 텍스트 | `wrapping="false"` → 줄바꿈 금지. `textAlign="Center"` |
| **`Label`** | 입력 필드의 레이블 | `design="Bold"` → 굵게. `class="sapMLabelShowColon"` → `:` 자동 추가 |

### 버튼

| 컨트롤 | 용도 | 주요 속성 |
|--------|------|----------|
| **`Button`** | 일반 버튼 | `type="Emphasized"/"Default"/"Transparent"`, `icon="sap-icon://...". `press="메서드명"` → 컨트롤러 연결 |

### 상태 표시

| 컨트롤 | 용도 | 주요 속성 |
|--------|------|----------|
| **`ObjectStatus`** | 상태 배지/태그 | `state="Success"/"Warning"/"Error"/"None"/"Information"`, `icon="sap-icon://..."` |
| **`MessageStrip`** | 안내 배너 | `type="Information"/"Warning"/"Error"`, `showIcon="true"` |
| **`MessageToast`** | 하단 팝업 메시지 (controller) | `MessageToast.show("메시지")` — 3초 후 자동 소멸 |
| **`MessageBox`** | 확인/경고 다이얼로그 (controller) | `.warning()` / `.confirm()` — `actions`, `emphasizedAction`, `onClose` |

### RatingIndicator

| 컨트롤 | 용도 | 주요 속성 |
|--------|------|----------|
| **`RatingIndicator`** | 별점 표시/입력 | `value="{...}"` (양방향 바인딩). `maxValue="5"`. `enabled="false"` → 읽기 전용. `iconSize="0.875rem"/"1.5rem"` |

> **양방향 바인딩**: `enabled` 기본값은 `true`. `value="{viewModel>/NewReview/rating}"`으로 바인딩하면 사용자가 별을 클릭할 때마다 viewModel 값이 즉시 업데이트된다. 추가 코드 없이 `setProperty` 호출과 동일한 효과.

### 리스트

| 컨트롤 | 용도 | 주요 속성 |
|--------|------|----------|
| **`List`** | 스크롤 가능한 목록 | `items="{배열경로}"`, `noDataText="{i18n>...}"` |
| **`CustomListItem`** | List의 자유 형식 항목 | 내부에 VBox/HBox/Text 자유롭게 배치 |

### 입력

| 컨트롤 | 용도 | 주요 속성 |
|--------|------|----------|
| **`TextArea`** | 여러 줄 텍스트 입력 | `maxLength="1000"`, `rows="3"`, `width="100%"`, `placeholder="{i18n>...}"` |

### 툴바

| 컨트롤 | 용도 | 주요 속성 |
|--------|------|----------|
| **`OverflowToolbar`** | 오버플로우 지원 툴바 | 공간 부족 시 버튼이 `...` 메뉴로 접힘 |
| **`ToolbarSpacer`** | 툴바 내 공간 확보 | 양쪽에 두면 가운데 정렬 효과. 한쪽만 두면 우측 정렬 |

## 2. `sap.uxap` — ObjectPage 패턴

Fiori 마스터-디테일 화면의 표준 구조. **사용자 정보/상세 정보를 표시하는 SAP의 공식 패턴**.

| 컨트롤 | 용도 | 주요 속성 |
|--------|------|----------|
| **`ObjectPageLayout`** | 전체 페이지 구조의 루트 | `upperCaseAnchorBar="false"` → 섹션명 소문자 유지. `toggleHeaderOnTitleClick="true"`. `showFooter="false"` |
| **`ObjectPageDynamicHeaderTitle`** | 동적 헤더 (확장/축소) | `snappedContent`(스크롤 시 축소) + `expandedContent`(펼친 상태) + `heading`(항상) + `actions`(우측 버튼) |
| **`ObjectPageSection`** | 섹션 구분 | `title="{i18n>...}"`, `titleUppercase="false"` |
| **`ObjectPageSubSection`** | 섹션 내 하위 영역 | `title="..."`, `visible="{= 조건 }"` |

### 헤더 전환 원리

```
[스크롤 최상단]
┌──────────────────────────────────────┐
│ expandedContent (MOVIE ⭐3.75 리뷰4개) │  ← 넓은 정보
│ heading       (타이틀)                │  ← 항상 보임
├──────────────────────────────────────┤
│ [기본 정보] [출연진] [리뷰]            │  ← Anchor Bar
│ ...                                  │

        ↓ 스크롤 내리면 ↓

├──────────────────────────────────────┤
│ snappedContent (MOVIE)  타이틀        │  ← 한 줄로 축소
├──────────────────────────────────────┤
│ [기본 정보] [출연진] [리뷰]            │
```

## 3. `sap.ui.table` — 데이터 테이블

`t:` 접두어로 사용. `sap.m.Table`과 달리 **정렬·열 고정·열 리사이즈가 내장된 데이터 그리드**.

| 컨트롤 | 용도 | 주요 속성 |
|--------|------|----------|
| **`t:Table`** | 데이터 그리드 | `rows="{...}"`, `visibleRowCount="{= ...}"`, `enableColumnFreeze="true"`, `selectionMode="None"` |
| **`t:Column`** | 컬럼 정의 | `hAlign="Left"`, `width="12rem"`, `sortProperty="필드명"` |
| **`t:template`** | 컬럼의 셀 템플릿 | 각 행마다 이 안의 컨트롤이 반복 생성됨 |
| **`t:extension`** | 테이블 상단/하단 확장 영역 | 툴바나 버튼을 넣을 수 있음 |

### `sap.m.Table` vs `sap.ui.table.Table` 선택 기준

| 요구사항 | 선택 |
|----------|------|
| 단순 목록 출력, 적은 행 (≤20) | `sap.m.Table` |
| 정렬 필요 | `sap.ui.table.Table` |
| 열 고정(freeze) 필요 | `sap.ui.table.Table` |
| 사용자 열 너비 조절 필요 | `sap.ui.table.Table` |
| 반응형 모바일 대응 | `sap.m.Table` |

## 4. `sap.ui.layout.form` — 폼 레이아웃

`form:` 접두어로 사용.

| 컨트롤 | 용도 | 주요 속성 |
|--------|------|----------|
| **`form:SimpleForm`** | 폼 컨테이너 | `layout="ResponsiveGridLayout"`, `editable="true"/"false"`, `labelSpanL="4"`, `columnsL="2"` |
| **`form:content`** | 폼 필드들의 컨테이너 | 이 안에 VBox/HBox로 필드 배치 |

### ResponsiveGridLayout 동작 원리

`SimpleForm`이 `layout="ResponsiveGridLayout"`일 때 내부적으로 **12컬럼 그리드**를 생성한다. `columnsL="2"`면 Large 화면에서 2열, 각 필드는 `GridData span="L6"`으로 6컬럼씩 반씩 차지한다.

```xml
<form:SimpleForm layout="ResponsiveGridLayout" columnsL="2" columnsM="2">
    <form:content>
        <VBox>
            ...필드...
            <layoutData>
                <layout:GridData span="L6 M6 S12" />
            </layoutData>
        </VBox>
    </form:content>
</form:SimpleForm>
```

## 5. `sap.ui.layout` — 배치 도구

`layout:` 접두어로 사용.

| 컨트롤 | 용도 | 주요 속성 |
|--------|------|----------|
| **`layout:GridData`** | 폼 내 필드의 열 점유 지정 | `span="L6 M6 S12"` → Large 6/12, Medium 6/12, Small 12/12 |

### span 값 규칙

| 값 | 의미 |
|:--:|------|
| `L12 M12 S12` | 모든 화면에서 한 줄 전체 |
| `L6 M6 S12` | 넓은 화면 반쪽, 좁은 화면 전체 |
| `L4 M4 S12` | 넓은 화면 1/3, 좁은 화면 전체 |

> ⚠️ `sap.ui.layout.cssgrid`는 UI5 1.136 CDN에 **없음**. 순수 CSS Grid(`display: grid`)로 대체.

## 8. `sap/ui/export/Spreadsheet` — 엑셀 다운로드

코드에서만 사용 (XML 아님).

```javascript
var oSheet = new Spreadsheet({
    fileName: "detail_info",
    workbook: { columns: [{ label: "제목", property: "label" }] },
    dataSource: [{ label: "컨텐츠명", value: "어벤져스" }]
});
oSheet.build().finally(function () { oSheet.destroy(); });
```

| 속성 | 의미 |
|------|------|
| `fileName` | 다운로드 파일명 (`.xlsx` 자동 추가) |
| `workbook.columns` | 컬럼 정의 배열 `[{ label, property }]` |
| `dataSource` | 데이터 배열 (객체의 property가 컬럼과 매핑) |
| `.build()` | 엑셀 생성 → 다운로드 트리거 |
| `.destroy()` | 메모리 해제 (`.finally()`에서 호출) |

> ⚠️ UI5 1.136 CDN의 Spreadsheet는 **멀티시트 미지원**. 여러 시트가 필요하면 2개 버튼으로 분리.

## 12. CSS 유틸리티 클래스 (sapUi*)

SAP UI5가 기본 제공하는 마진·패딩 클래스. 별도 CSS 파일 없이 사용 가능.

| 클래스 | 효과 |
|--------|------|
| `sapUiTinyMarginBegin` | 왼쪽에 작은 마진 (LTR 기준) |
| `sapUiTinyMarginEnd` | 오른쪽에 작은 마진 |
| `sapUiTinyMarginBottom` | 아래에 작은 마진 |
| `sapUiTinyMarginTopBottom` | 위아래 작은 마진 |
| `sapUiSmallMargin` | 작은 마진 (전방향) |
| `sapUiSmallMarginBegin` | 왼쪽에 중간 마진 |
| `sapUiSmallMarginBottom` | 아래에 중간 마진 |
| `sapUiSmallMarginTopBottom` | 위아래 중간 마진 |
| `sapUiMediumMargin` | 중간 마진 (전방향) |
| `sapUiLargeMarginBottom` | 아래에 큰 마진 |
| `sapUiContentPadding` | 컨텐츠 영역 패딩 |
| `sapUiNoMarginBottom` | 아래 마진 제거 |
| `sapUiResponsiveMargin` | 반응형 마진 |

> 패턴: `sapUi` + `[크기]` + `[Margin/Padding]` + `[방향]`. 예: `sapUiTinyMarginBegin`

## 13. 컨트롤러 전용 — MessageBox 패턴

```javascript
// ✅ 표준 패턴 (Fiori 가이드라인)
MessageBox.warning(sMessage, {
    title: sTitle,
    actions: [MessageBox.Action.OK, MessageBox.Action.CANCEL],
    emphasizedAction: MessageBox.Action.OK,     // Enter 키 기본 액션
    contentWidth: "30rem",                       // 텍스트 잘림 방지
    onClose: function (sAction) {
        if (sAction === MessageBox.Action.OK) { /* 실행 */ }
    }
});

// ⚠️ 구형 패턴 (아이콘 '?' 고정, width 제한)
MessageBox.confirm(sMessage, {
    onClose: function (sAction) { ... }
});
```
