# Joule Studio 최신 업데이트

[https://news.sap.com/2026/05/new-joule-studio-enterprise-scale-agentic-development/](https://news.sap.com/2026/05/new-joule-studio-enterprise-scale-agentic-development/)

[https://www.sap.com/topics/events/sapphire/innovation-news-guide-2026#joule](https://www.sap.com/topics/events/sapphire/innovation-news-guide-2026#joule)

- SAP Autonomous Suite
    - 재무 관리 : Joule Assistant가 자율 금융을 주도
    - 지출 관리 : Joule Assistant는 지출 라이프사이클 전반에 걸쳐 자율적인 지출 관리를 가능하게 함.
        - SAP Ariba 구매 및 계약 솔루션 전반에 걸친 업데이트 (Joule Agent 포함)
    - 인적 자원 관리 : Joule의 새로운 Assistant를 통해 자율적인 HCM이 현실화
    - 고객 경험 : 새로운 Joule Assistant가 자율적인 고객 경험(CX)을 지원
    - 공급망 관리 : Joule Assistant, Joule Agent 및 산업 AI 시나리오를 통해 자율형 SCM 구현

- SAP Business AI Platform (Joule Studio 포함 : 내용 분리됨)
    - **SAP Domain Model** : SAP 환경에 최적화된 도메인 인텔리전스와 코드를 제공
        - Joule, Joule Studio, Joule Agent 및 SAP 애플리케이션에 심층적인 SAP 지식 제공
        - SAP S/4HANA(Cloud Public Edition) 및 SAP Ariba에서 정보 조회/사용자 정의 확장 기능 생성
            - Joule Studio의 개발자는 SAP S/4HANA 및 SAP Ariba용 특수 모델을 사용하여 자연어로부터 깔끔하고 핵심 규격을 준수하는 코드를 이해, 생성
            - 고객은 Joule에서 자연어 프롬프트를 사용하여 기본 데이터 모델 및 비즈니스 컨텍스트에 기반한 고객 데이터에 액세스 가능
    - SAP Business Data Cloud(BDC) : Agentic AI 기능이 데이터 관리, 분석 및 계획 시나리오에서 데이터 전문가를 지원
        - Joule Agent는 사용자 쿼리에 응답하고 데이터 제품 생성, 데이터셋 준비 또는 대시보드 생성과 같은 작업을 자동화 → 고객이 여러 도구와 워크플로우를 사용하지 않고도 결과를 얻을 수 있음
        - Joule은 다양한 SAP BDC 구성 요소에 대한 통합 진입점을 제공 → 고객이 조직 전체에서 데이터가 어떻게 연결되고 관리되는지 더 쉽게 이해할 수 있도록 지원
            - Joule은 SAP BDC 데이터 제품과 **SAP Knowledge Graph** 활용 → 완전 관리형 온톨로지를 사용하여 기업 전체의 프로세스, 관계 및 컨텍스트를 파악 → 더 정확한 비즈니스 인사이트 제공
        - SAP BDC와 AWS 통합
        - SAP HANA Coud가 SAP BDC에서 기본적으로 제공됨.

### Joule Work : AI Agent가 단순 반복 작업을 처리하는 중앙 작업 공간

- 사용자가 결과 도출에 집중할 수 있도록 지원
- Joule Agents와 Joule Assistants를 사용하여 관련 데이터를 표시
    - Joule Assistant는 심층적인 역할 및 비즈니스 컨텍스트를 통해 Joule Agent를 조율
    - Assistant의 지시를 받으면, Agent는 관련 데이터, 도구 및 애플리케이션을 연결하여 데이터 분석, 작업 실행, 시스템 상호 작용, 워크플로우 단계 실행과 같은 특정 작업을 수행
    - Joule Assistant는 Joule Work를 통해 나타남
- SAP 및 비SAP 시스템 전반에서 워크플로우를 안정적으로 자동화
- 주요 구성 요소(Component)
    - 대화(Conversations) : 대화 기능을 통해 사용자는 의도를 표현하고, Joule과의 대화를 통해 달성하고자 하는 목표 탐색 가능
        - Joule은 SAP 및 타사 시스템의 지침과 결과를 통합
        - 대화 기능은 더욱 복잡한 워크플로우를 위한 Spaces 진입점으로 활용 가능
    - **공간(Spaces)** : 간단하고 복잡한 문제 해결을 위한 역동적인 작업 공간
        - 각 Space는 특정 목표에 필요한 데이터, 도구 및 컨텍스트를 제공하기 위해 실시간으로 구성됨
        - 사용자는 애플리케이션과 수동적인 사용자 인터페이스를 탐색하는 대신 현재 목표(사용자 의도)에 맞춰 조정되는 환경에서 작업 가능
    - 개발(Develop) : Joule Work는 Joule Studio에 편리하게 접근할 수 있는 진입점을 제공하여 팀이 맞춤형 AI를 구축하고 실행할 수 있는 통합된 방법을 제공
        - Joule Studio의 Pro-Code/Low-Code 도구를 통해 SAP 및 타사 시스템 전반에 걸쳐 AI Agent, 앱 및 자동화 생성 가능
- Joule Work 워크플로우
    - 사용자는 자연어로 원하는 목표를 표현
    - Joule Assistant가 Joule Agent 팀을 구성하여 적절한 인사이트를 도출 → 비즈니스 영역 및 시스템 전반에 걸쳐 반복적인 작업을 자동화하여 목표를 달성하도록 지원
        - 사용자를 기능별로 구성된 스마트 팀원과 같은 Joule Assistant와 연결
        - 이러한 Assistant는 컨텍스트를 활용하여 사용자의 의도를 파악하고, 비즈니스 전반에 걸쳐 적절한 Joule Agent를 조율하여 작업을 수행
    - 이러한 과정은 실시간으로 구축되는 의도 기반의 적응형 작업 공간(Space)에서 이루어짐 → 팀은 의사 결정 및 성과 창출에 집중
- Joule Work를 통해 고객은 Joule Studio 솔루션에 액세스하여 MCP 및 A2A 프로토콜을 사용하는 맞춤형 Agent를 구축하고, 도구 및 타사 Agent 활용 가능
    - *Joule Work 및 Joule A2A 기능은 2026년 4분기에 일반에 공개될 예정*
- 웹 클라이언트 / 데스크톱 앱 / 모바일 앱으로 사용 가능
    - *Joule Work는 현재 SAP Early Adopter Care 프로그램에 참여하는 고객에게 제공
    Joule Work 데스크톱 앱의 SAP Early Adopter Care 프로그램은 2026년 2분기에 시작될 예정이며, 두 앱 모두 2026년 하반기에 일반 출시될 예정
    Joule Work 모바일 앱은 현재 일반 출시되어 있음*

### Joule Studio 관련

- *Joule Studio는 현재 얼리 액세스 프로그램을 통해 이용 가능하며, 2026년 3분기에 일반 출시될 예정입니다.*
    - Joule Studio 2.0
- SAP Business AI Platform의 핵심 구성 요소
- Agent 및 애플리케이션 구축, 확장, Orchestration을 위한 통합 기능 활용 → AI 기반 전체 라이프사이클 관리
- Joule Studio는 Joule Work(참여 레이어)에 내장되어 AI 기능을 제공함으로써 Agent 및 애플리케이션 개발, 통합, Agent 기반 워크플로우를 위한 통합된 환경 구현
    - Joule Work는 개발 및 배포 수명 주기 전반에 걸쳐 Agent의 가시성을 제공, MCP 서버 생성 간소화, 상호 운용 가능한 Agent 연결을 지원하는 새로운 MCP 빌더 포함
- Joule Studio는 이제 비즈니스 컨텍스트를 활용한 의도 기반 개발 기능을 개발자에게 제공
    - 사용자는 자연어로 비즈니스 의도 설명 가능
    - 사용자는 Joule Work 참여 계층의 개발 공간 내에서 Joule Studio 설계 시간 툴에 액세스하여 Agent 및 확장 기능 생성
- Joule Studio를 사용하는 개발자는 LangGraph, AutoGen, LlamaIndex 등의 선호하는 Agent 프레임워크를 사용하여 Low-Code 및 Pro-Code 기능을 활용해 Agent 생성 가능
    - 새로운 구축 환경은 SAP 도메인 모델과 SAP Cloud SDK for Python을 사용
    - Agent는 일반적인 LLM 및 도구로는 구현할 수 없는 SAP 고유의 코드, 아키텍처 및 데이터 모델을 기본적으로 이해 → SAP 시스템 전반에서 더 정확하게 추론, 실행 가능
    - Joule Studio에서 생성된 Agent는 MCP 및 A2A 프로토콜을 기본적으로 지원 → 타사 도구 및 Agent와 연결 및 협업 가능, 상호 운용성과 확장성을 통해 다중 Agent 실행 가능
- Joule Studio는 SAP Signavio 및 SAP LeanIX와 같은 비즈니스 전환 관리 솔루션과 통합 → 고객이 기업 환경 전반에 걸쳐 SAP 및 비SAP AI Agent를 검색, 모니터링 및 관리할 수 있도록 지원
    - **SAP Signavio**의 프로세스 인사이트 : Joule Studio는 SAP Signavio와 통합하여 실제 프로세스 모델을 시각화
        - 병목 현상, 시간 낭비 요소, 오류 발생 지점을 파악 가능
        - 이러한 기반 위에 구축된 Agent는 일반적인 해결책이 아닌, 운영상의 실제 마찰 지점을 제거하도록 설계됨.
    - **SAP LeanIX**의 아키텍처 컨텍스트 : SAP LeanIX는 IT 환경을 매핑 → 어떤 시스템이 연결되어 있는지, 로드맵에는 무엇이 포함되어 있는지, 클린 코어를 유지해야 하는 부분은 무엇인지 보여줌.
        - Joule Studio는 에이전트가 아키텍처에 맞춰 설계되도록 보장
- **n8n** : 엔터프라이즈 규모의 시각적 워크플로우 오케스트레이션
    - Joule Studio 솔루션 내에서 완전 관리형 환경으로 제공 예정 (*2026년 3분기)*
    - Joule Studio 내의 n8n 캔버스에서 직접 작업하여 AI 워크플로우를 시각적으로 구축, Agent 기능을 프로세스 자동화와 결합 → SAP를 조직에서 사용하는 도구 및 서비스 환경과 연결
    - SAP 클라우드 인프라에서 n8n 임베디드 환경이 실행됨
- **Vercel** : Next.js와 같은 프레임워크에서 속도와 맞춤형 디자인을 우선시하는 유연한 웹 경험 구축 가능 (*2026년 하반기)*
    - 일관성과 표준이 중요한 경우 기존의 SAPUI5 및 SAP Fiori UX 사용
- **Joule Studio Runtime** : Joule Studio에 직접 연결되어 고객이 안전하고 운영 환경에 바로 적용 가능한 완전 관리형 환경에서 Agent, 애플리케이션, 워크플로우를 배포할 수 있도록 지원
(*2026년 3분기)*
    - Joule Studio 런타임에 NVIDIA OpenShell 통합 → 보안 및 기업 거버넌스 강화
    - Joule Studio 런타임에 배포된 Agent는 영구적인 장기 메모리 탑재 가능
        - SAP HANA Cloud에서 기능 지원 → Agent는 세션 및 작업 전반에 걸쳐 정보와 사용자 기본 설정을 저장, 검색 및 재사용 가능 → 고도로 개인화된 서비스 제공 가능
- Joule Studio로 구현 가능한 전체 범위
    - 결정론적 앱 : 엄격하고 미리 정해진 규칙을 따름.
        - SAP UI, React 프론트엔드, CAP 백엔드 구축 가능
        - 로직에 대한 완전한 제어권 유지
    - 하이브리드 솔루션 : 고정된 논리와 Agent 기반 단계를 결합
    - 대화형 Agent : 엄격한 가이드라인 내에서 작동, 사용자가 정의한 특정 기술 및 도구로 제한됨
    - 시스템 트리거 Agent : 비즈니스 이벤트, API, 또는 일정에 따라 백그라운드에서 실행되는 능동적인 자동화를 위해 설계됨 → 사람의 개입 거의 필요 X
    - 다중 Agent 시스템 : 복잡하고 여러 단계를 거치는 문제를 해결하기 위해 워크플로우 내에서 협력하는 전문 Agent들의 조정된 네트워크
        - SAP Integration Suite는 구축하는 모든 시스템에 대한 연결을 처리하여 API, MCP 및 A2A를 통해 Agent를 SAP 및 비SAP 시스템에 연결
        - Agent는 독립적으로 작동하지 않고, 처음부터 엔터프라이즈 시스템에 연결됨.

### Joule Studio 관련 FAQ(7/7)

- 새로운 Joule Studio — 고객이 에이전트 기반 솔루션을 구축할 수 있도록 지원,
맞춤형 AI 에이전트, 애플리케이션, 워크플로우를 생성하고 실행할 수 있는 통합 AI 우선 개발 솔루션 제공
*아래와 같은 새로운 기능들은 올해 후반 출시 예정*
    - 의도 기반 개발(Intent-driven development)을 통해 에이전트, 앱, 워크플로우를 개발
    에이전트 개발 솔루션 및 프레임워크(Claude, Vercel, VS Code 등) 사용 가능
        - 의도 기반 개발 — (코드/기술 사양 이전에) 자연어 프롬프트로 시작되는 소프트웨어 구축 방식
        AI와 깊은 비즈니스 컨텍스트를 활용하여 비즈니스 의도를 지속적으로 진화, 컨텍스트 인식 솔루션으로 변환
    - SAP Signavio, SAP LeanIX, AI Agent와 통합 → 에이전트 기반 솔루션 제작, 거버넌스 지원
    - SAP 도메인 모델 및 SAP 지식 그래프를 활용 → SAP 시맨틱스와 엔터프라이즈 컨텍스트 기반의 확장 가능한 AI 에이전트 및 에이전트 기반 솔루션 제공
        - SAP LeanIX — 엔터프라이즈 아키텍처에 대한 에이전트 생성의 유효성을 검증
        - SAP Signavio — 프로세스 인사이트와 명확하게 정의된 장애 요인에 기반한 에이전트 생성 가능
        - SAP Knowledge Graph, SAP Domain Models, SAP Business Data Cloud(BDC) — 모든 개발 산출물에 자동으로 흐르는 보완적 컨텍스트 제공
    - 서드파티 커넥터에 대한 검증된 노드(통합/커넥터/MCP) 라이브러리와 함께 내장된 n8n의 에이전트 워크플로우 기능을 제공
    - 의도 기반 프롬프트 및 프로코드 개발자 도구를 통해 CAP 애플리케이션 생성 가능
    - Joule Studio Runtime → 에이전트 기반 AI 솔루션 실행 제공

- SAP Build와 새로운 Joule Studio의 차이점
    
    
    | **기능** | **SAP Build (비주얼-우선)** | **Joule Studio (AI & 코드-우선)** |
    | --- | --- | --- |
    | 주요 방식 | **로우코드 & 프로코드**: 시각적 도구와 사전 구축된 콘텐츠를 활용해 빠르게 생성하고, 복잡한 논리를 위한 프로코드 환경을 제공합니다. | **AI 기반** **&** **프로코드:** **"vibe** 코딩"(자연어 프롬프트) 와 전문 코딩을 사용합니다. |
    | AI 중점 | **내장형 AI:** 단일 턴 상호작용을 통해 워크플로우, 앱, 콘텐츠 기반 에이전트 생성을 지원합니다. | **대화형 AI:** 중앙화된 진입점에서 자연어 프롬프트를 사용해 앱, 에이전트, 워크플로우를 생성합니다. 
    다양한 개발 단계 전반에 걸쳐 엔드 투 엔드 AI-가이드 경험: 아이디어 도출, 개발, 테스트. 대화형 특성. |
    | 컨텍스트 인지  | 제한적: 비즈니스 데이터 연결을 위해 수동 구성이 필요합니다.  | 심층 통합: 귀하의 SAP 비즈니스 컨텍스트와 IT 환경을 자동으로 이해하며, SAP AI Agent Hub 및 SAP Signavio 와의 네이티브 통합을 지원합니다.  |
    | 맞춤형 AI 에이전트  | SAP Build는 노코드 에이전트 빌더(SAP Build 내 Joule Studio, 클래식 에디션)에서 AI 에이전트 생성을 지원합니다. 
    에이전트는 MCP 서버에 연결할 수 있고, 문서 그라운딩을 통해 기반을 둘 수 있습니다. 
    단, 에이전트 정의는 비즈니스 에이전트 프레임워크(BAF)를 기반으로 제한됩니다.  | Joule Studio는 이제 에이전트에 대한 프로코드 개발을 지원하며, 개발자가 LangChain, LangGraph, Pydantic AI 및 Llamaindex와 같은 원하는 에이전트 프레임워크를 사용할 수 있도록 하여 더욱 유연한 개발이 가능합니다. 
    MCP 및 문서 그라운딩 구성 외에도, 개발자는 지속적 메모리와 개인화를 활성화하여 더 맥락에 맞는 결과를 도출할 수 있습니다.  |
    | 배포  | 고객 관리형: 고객이 롤아웃 관리를 직접 담당합니다.  | SAP 관리형: SAP가 Joule Studio 런타임을 통해 배포 프로세스를 관리합니다.  |
    | 일반적인 산출물  | 앱 확장, 워크플로우 및 RPA 봇, 워크스페이스, 기본 AI 어시스턴트 및 에이전트  | 고급 맞춤형 지능형 솔루션 및 복잡한 소프트웨어.  |
    - SAP Build의 Joule Studio : 클래식 에디션, SAP Build의 기능으로 출시 → 로우코드 환경에서 Joule Skill과 AI 에이전트 설계 가능하도록 함
        - 개발자는 도구, 문서 그라운딩, MCP 서버 및 기타 구성을 에이전트 정의에 추가 가능
    - 새로운 Joule Studio → AI 에이전트뿐만 아니라 앱과 워크플로우도 하나의 환경에서 설계 가능
        - 의도 기반 개발, 선호하는 개발자 도구, 자연어 프롬프트를 비즈니스 및 기술 요구사항과 명세로 변환하는 기능 제공
        - 생성된 에이전트의 코드 확인 가능
        - LangChain, LangGraph, Pydantic AI, LlamaIndex 등 원하는 에이전트 프레임워크 사용 가능
        - 지속적 메모리와 개인화를 지원하여 더 맥락에 맞는 응답 제공
        - 고객이 Joule Skill로 구축한 기능들(결정론적 API 호출, 워크플로우 실행, 예측 가능한 출력 등)은 새로운 아키텍처에서 MCP 도구와 n8n 워크플로우를 통해 보존/강화됨

- Joule Studio와 연동:
    - SAP Business AI Platform — 세 가지 역량 기둥 중 Build 기둥 내에 Joule Studio 위치
    - Joule Work — 한 곳에서 모든 작업을 처리할 수 있는 단일 공간 제공, 의도 기반의 상황별 경험 제공

- 가격 모델
    - Joule Work — 현재 기능 무료로 사용, 향후 추가 기능에 과금 가능
    - Joule Studio 디자인 타임 도구(Design-time tooling) — 공정 사용 한도까지 무료로 제공
    한도 초과 시 Joule Studio 런타임 SKU 지표를 사용하여 AI 기능(LLM, 도메인 모델, 지식 그래프) 사용에 대해 비용을 지불
    - Joule Studio Runtime — 실제 리소스 소비량 기준으로 과금
    측정 방식은 컴퓨트, 스토리지, 트랜잭션, AI 토큰 등 제한된 소비 지표를 사용
