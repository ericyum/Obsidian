### 모델 튜닝의 세 가지 축

- **데이터 불균형 해소**: 모델 안정성 확보를 위한 필수 조치

### 데이터 불균형 및 샘플링 기법

- **정확도(Accuracy)의 함정**: 99% 정상 데이터에서 무조건 "정상"으로 예측해도 정확도 99% → 이상치 탐지 불가
- **SMOTE(오버샘플링)**: 소수 클래스 데이터를 가상으로 생성하여 비율 조정
    - 장점: 원본 데이터 삭제 없이 특징 강화
    - 단점: 경계선 노이즈 증폭 가능성, 피처 수 증가 시 복잡도 상승
- **변형 오버샘플링**: Borderline SMOTE(경계선 데이터 집중 복제), ADASYN(학습 어려운 영역 집중)
- **언더샘플링 결합 기법**: Tomek Links(가장 가까운 이종 쌍 제거), ENN(다수결 기반 제거) — 노이즈 감소 목적
    - 공격적 언더샘플링 시 원본 데이터의 경향성·시계열 연계성 손실 위험
- **POC 적용 방식**: 리샘플링 대신 **가중치(Class Weight + Sample Weight)** 방식 채택 → 원본 데이터 보존, 시계열 흐름 유지
    - POC 데이터 불균형 수준: 다수 클래스 58.2% vs 소수(상승) 41.9% → 심각한 불균형 아님

---

### AUC 해석 기준 (원자재 가격 예측)

- **0.55 미만**: 사용 불가 (동전 던지기 수준)
- **0.56 ~ 0.60**: 최소한 설명 가능한 수준
- **0.63 ~ 0.69**: 매우 준수
- **0.70 이상**: 과적합 의심 — 변동성 높은 시장에서는 오히려 경계

---

### 임계값(Threshold) 조정과 Precision/Recall 트레이드오프

- Threshold를 **0.5 → 0.3**으로 낮추면: Recall(재현율) 상승, Precision(정밀도) 하락
- **Recall 중시**: 가격 상승 이벤트를 놓치지 않는 조기 감지 목적에 유리
- **Precision 중시**: 오탐 비용(잘못 구매 등)이 클 경우 확실한 신호에만 반응
- Confusion Matrix 변화: Threshold 낮추면 True Positive 증가, False Positive(상승 예측 후 하락) 증가
- **구매 담당자 관점**: 무조건 구매해야 하는 포지션이므로 가격 상승 탐지(Recall 우선)가 핵심
- Threshold 조정은 모델 분별력(AUC) 향상이 아닌 **의사결정 성향 조정**임

---

### 모델 보정 (Calibration)

- **정의**: 모델 예측 확률값을 실제 발생 확률에 맞게 보정하는 작업 → AUC/정밀도/재현율은 변하지 않음
- **필요성**: 모델이 52%로 예측해도 실제는 75%일 수 있음(과소평가), 반대로 99% 확신이 실제 20%일 수 있음(과잉확신)
- **XGBoost**: 캘리브레이션 미적용
- **TFT 모델 캘리브레이션** (가격 범위 예측):
    - Bias Calibration: P50 중앙값 편향 보정 (POC에서 -29.08달러 보정 적용)
    - Conformal Calibration: 예측 구간(P10~P90) 커버리지 보정 → 커버리지 84% 달성
    - Volatility Adjustment + Rolling: 변동성에 따라 구간폭 조정, 최신 데이터 가중 반영
    - Alpha Tuning: 커버리지 유지하면서 예측 구간 폭을 최소화하는 최적 파라미터 탐색

---

### 기간별 검증 (Time-Series Validation)

- **무작위 검증 금지**: 시계열 데이터에서 미래 데이터 누출(Data Leakage) 및 Concept Drift 위험
- **Expanding Window**: 학습 기간을 누적 확장하며 검증
- **Sliding Window**: 고정 길이 윈도우를 시간 순으로 이동하며 검증
- **POC 적용 절차**:
    1. Single Time Split으로 기본 AUC 산출
    2. Expanding Walk-Forward 수행 → 분산(편차) 과대 문제 발생
    3. Multi Cutoff(9개 고정 커트오프)로 전환 → 최근 데이터일수록 AUC 향상 확인
- **목적**: 과적합 방지, 모델 재학습 시점 선제 탐지, 안정적 장기 운용

### 피처 엔지니어링: GVZ 도입 실험

- **문제 의식**: 기존 피처에 포함된 VIX(S&P500 변동성 지수)와 대두 가격의 연관성에 의문 → 금선물 변동성 지수 **GVZ**로 대체/보완 시도
- **피처 생성**: GVZ 종가 기반으로 1일 수익률, 7/20/60일 이동평균, 7/21일 표준편차 등 파생 피처 생성
- **스모크 테스트**: 피처 조합 전 AUC 개선 가능성 사전 평가
- **스크리닝 기준**:
    - Correlation Screening: 상관관계 높은 피처 중 하나 제거 (기존 피처 우선 보존)
    - SHAP Value + Feature Importance 랭킹으로 우위 피처 선택
    - Combo 조합은 Top20 내 2개 이상, Top10 내 1개 이상 진입 조건
- **결과**: Combo 조합이 Replace보다 AUC 개선 우위 → Replace 방식 보류
- **추가 검증 필요**: AUC 외 전체 지표 포함 최종 검증 후 적용 여부 결정
- **XGBoost의 피처 평가 활용**: 특정 모델에 종속되지 않은 피처 유효성 검증 도구로도 활용 가능
