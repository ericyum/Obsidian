- [ ] 예측 오차 감소를 위한 모델 튜닝 또는 모델 교체 검토

- [ ] 교육 세션 종료 후 팀원들이 직접 모델 실습 진행
- [ ] 고객 미팅 전 DQN 관련 질문 항목 정리 — 리워드 KPI, 액션 정의, 구매 가이드라인 등 확인

### 세션 개요

- 3일간 진행된 교육 세션의 마지막 날로, RAG 구성 사례와 DQN 강화학습 모델 전반을 다룸
- 핵심 메시지: **데이터가 가장 중요**하며, 피처 조합 → 모델 성능 향상 → DQN 최적화 순으로 진행
- 모델은 대두(Soybean) 구매 의사결정 보조 목적으로 개발됨

---

### RAG 구성 사례

- **첫 번째 RAG**: XGBoost 모델 출력값(상승/Buy/Wait 등)을 조합해 쿼리를 생성하고, 해당 의사결정에 가장 유사한 뉴스 기사를 벡터 검색으로 연결
- **두 번째 RAG**: 6주간의 가격 변동 구간(상승 시작일, 하락 시작일)을 날짜 단위로 명시하여 LLM에 전달, 뉴스 기사 검색 기반으로 가격 변동 원인 설명 생성
    - LLM이 단순 시계열 가격 나열을 잘 해석하지 못하므로, 변곡점 날짜를 명시적으로 제공하는 것이 중요

---

### DQN 핵심 개념

- **State (상태)**: 현재 기업 환경 — 가격 예측, 재고, 납기, 구매 가능 물량 등 12차원으로 구성
- **Action (액션)**: Hold, Buy, 분할구매(1배/2배/3배) 총 5가지 행동 정의
    - 3배 액션(6,000톤)은 실제로 발생하지 않아 수정 예정
- **Reward (리워드)**: 미래 평균 가격 대비 실제 구매 가격의 차이 — 싸게 살수록 보상값이 큼
    - 페널티: 물량 미달 구매, 너무 빠른 전량 구매, 최대 구매량 초과
- **Q-Value**: 특정 상태에서 특정 행동을 선택했을 때의 기대 가치
- **Q-Network (DQN)**: Q값을 출력하는 딥러닝 신경망 — 모든 경우의 수를 저장하는 대신 근사치를 학습

### 신경망 구조

- 입력층: 12차원 상태값
- 은닉층 1: 128노드 (ReLU 활성화) → 은닉층 2: 64노드
- 출력층: 5개 액션별 Q값
- 손실함수: Huber Smooth L1 — 초기에는 큰 오차를 완만하게, 후반부로 갈수록 오차 제곱 적용

---

### Double DQN

- 단일 신경망 사용 시 정답 Q값이 학습마다 변동되어 과대평가 발생
- 해결책: **Main Network** (선택 담당)와 **Target Network** (평가 담당)를 분리
- Target Network는 일정 주기(50스텝)마다 Main Network와 동기화 → 안정적인 학습 보장
- Git의 master/dev 브랜치 분리와 유사한 안정화 구조

### RL Cache & Replay Buffer

|구분|RL Cache|
|---|---|
|역할|실시간 데이터 임시 저장|
|크기|소형 (배치 단위)|
|저장 기간|에피소드 종료 시 삭제|
|추출 방식|날짜 순|

- 무작위 샘플링을 통해 시계열 상관관계를 제거하고 과적합 방지
- Replay Buffer가 가득 차면 가장 오래된 데이터부터 삭제
- `.npz` 파일로 백업하여 추론 환경에서 빠르게 로드 가능

### 하이퍼파라미터

- **감마(γ)**: 미래 가치 할인율 — 먼 미래의 Q값일수록 현재 가치로 할인하여 합산
- **학습률**: 0.001 (기본값) — 너무 크면 Q값이 불안정해짐
- **배치 사이즈**: 64 (기본값) — 너무 작으면 학습 불안정, 너무 크면 속도 저하
- **메모리 사이즈**: 10,000건
- **Target 업데이트 주기**: 50스텝마다 동기화

---

### Epsilon-Greedy 탐색 전략

- **탐색(Exploration)**: 새로운 행동 무작위 선택 vs **활용(Exploitation)**: 기존 최선 행동 선택
- 초기 ε = 1.0 (100% 무작위 탐색) → 점진적 감소 → 최종 ε = 0.05 (5%만 무작위)
- 감소율: 0.005씩 감소
- 학습 초기에는 탐색 위주, 후반부에는 검증된 좋은 정답 위주로 학습

### 운영 추론 환경 구성

- 운영 흐름: 마스터 데이터 갱신 → XGBoost/TFT 예측 갱신 → RL Cache 갱신 → DQN 추론
- **Calibration**: 재고 비율, 가격 차이, 남은 기간, 분할 구매 후 잔여 물량 등 업무 현실 반영 보정
- **세 가지 출력 통합**: 룰 기반 추천 + DQN 모델 선택 액션 + LLM 해석 문장 → LLM에 전달
- **해석 매트릭스**: 시나리오 엔진과 DQN 결과가 다를 경우 해석하여 출력
    - 예시: 재고 타이트 → Buy 신호이지만 DQN이 Hold → "가격 하락 가능성 있으나 입고 시급, 구매 권고"
- **과거 유사 케이스 RAG**: 현재 상황과 유사한 Top3 과거 사례를 벡터 검색으로 제공, 당시 판단 및 실제 결과 비교

### 현재 문제점 및 향후 과제

- **폴백 메커니즘 버그**: 과거 날짜 시뮬레이션 시 Cache의 최신 행(현재 날짜 데이터)을 참조하는 오류 가능성
- **예측 오차**: P50 예측값과 실제 종가 간 오차가 20% 이상인 구간 존재
- **결품(품절) 리스크 미반영**: DQN의 리워드가 가격 중심으로 설계되어 결품 방지 로직 부재 → 보조 모델로만 활용
- **개선 과제 목록**: 피처 재검토, 모델 튜닝/교체, 리포트 내 그래프 표시, Fallback 메커니즘 수정
