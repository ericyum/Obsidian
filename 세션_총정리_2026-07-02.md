## 2. 리뷰 섹션 구조 변경

### 변경 전
```
리뷰 섹션
├── 리뷰 목록
├── "내 리뷰" 하위섹션  ← MessageStrip (HasReviewed일 때)
└── 리뷰 작성 폼
```

### 변경 후
```
리뷰 섹션
├── MessageStrip  ← 최상단 (HasReviewed일 때)
├── 리뷰 목록
└── 리뷰 작성/수정 폼  ← !HasReviewed 또는 수정모드일 때
```

- "내 리뷰" 하위 섹션 **완전히 삭제**
- MessageStrip이 리뷰 섹션 맨 위에 표시
- 수정 버튼 누르면 MessageStrip 유지 + 수정 폼 표시

**변경 파일**: `DetailMain.view.xml`

## 5. 평균 평점 SQL 버그 수정 🔥

### 문제
C001의 리뷰를 하나 삭제했는데 평균 별점이 **4.0**, 리뷰 개수가 **9**로 표시됨 (실제로는 리뷰 1개, 별점 3.0이어야 함)

### 원인
`DetailService.handler.ts`의 SQL에서 Review 테이블의 FK 컬럼명을 잘못 씀
```sql
-- 잘못된 코드 (Review 테이블에 content_id 컬럼 없음 → 필터링 실패 → 전체 리뷰 집계)
WHERE content_id = ?

-- 올바른 코드 (CAP이 Association to Contents로 생성한 실제 컬럼명)
WHERE content_content_id = ?
```

### 영향
`submitReview`와 `deleteReview` 두 액션 모두에서 발생. 컬럼명 오류로 WHERE절 필터링이 안 돼서 **모든 콘텐츠의 전체 리뷰 평균과 개수**가 반영되고 있었음.

**변경 파일**: `DetailService.handler.ts`

## 🎯 핵심 교훈

1. **세션 시작 시 `git log` + `git status` 먼저** — pull 안 된 상태로 작업하면 큰일 난다
2. **UI5 1.136 CDN 제약** — `sap.ui.layout.cssgrid` 없음, `f:Card` 기본 aggregation 없음 → CSS로 우회

6. **SQL raw query 쓸 때 컬럼명 더블체크** — CDS 모델과 DB 스키마의 컬럼명은 다를 수 있다
