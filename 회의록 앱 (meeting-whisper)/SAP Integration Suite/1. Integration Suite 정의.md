# 1. SAP Integration Suite란 무엇인가 (정의)

> **작성일**: 2026-07-20
> **출처**: SAP 공식 문서, Discovery Center, Help Portal 종합

---

## 한 줄 정의

**SAP Integration Suite**는 기업의 온프레미스·클라우드·SaaS 애플리케이션과 프로세스를 하나로 연결하는 **iPaaS(Integration Platform as a Service)** 통합 솔루션입니다. SAP BTP(Business Technology Platform)의 핵심 서비스 중 하나로, 다양한 통합 시나리오를 중앙에서 설계·실행·모니터링할 수 있는 엔터프라이즈급 통합 플랫폼입니다.

---

## 역사적 배경

### 기존의 SAP 통합 도구들

| 도구 | 시기 | 특징 | 한계 |
|------|------|------|------|
| **SAP PI (Process Integration)** | 2000년대 | Java 기반 ESB, 온프레미스 | 무겁고 복잡, Java 스택 의존 |
| **SAP PO (Process Orchestration)** | 2010년대 | PI + BPM + BRM 통합 | 온프레미스 중심, 클라우드 대응 부족 |
| **SAP CPI (Cloud Platform Integration)** | 2015년경 | 최초의 클라우드 통합 | 통합 기능만 제공, 별도 도구 필요 |

### 통합으로의 진화

SAP는 2021년부터 CPI를 **Integration Suite**로 확장·리브랜딩했습니다. 단순한 EAI(Enterprise Application Integration) 도구를 넘어, iPaaS라는 더 큰 개념으로 발전했습니다. CPI는 현재 Integration Suite의 한 구성 요소인 **Cloud Integration**으로 편입되었습니다.

```
SAP PI → SAP PO → SAP CPI → SAP Integration Suite (현재)
                              ├── Cloud Integration (구 CPI)
                              ├── API Management
                              ├── Event Mesh
                              ├── Trading Partner Management (B2B/EDI)
                              ├── Integration Advisor
                              └── (Open Connectors, Edge Integration Cell...)
```

---

## 핵심 개념: iPaaS (Integration Platform as a Service)

### iPaaS란?

iPaaS는 다양한 환경(온프레미스, 클라우드, SaaS)에 분산된 애플리케이션, 데이터, 프로세스를 **통합하는 클라우드 기반 플랫폼**입니다.

### 시장 포지션

SAP Integration Suite는 Gartner Magic Quadrant for iPaaS와 Forrester Wave에서 지속적으로 **리더(Leader)** 로 평가받고 있습니다. 특히 SAP 생태계와의 긴밀한 통합이 최대 강점입니다.

## 통합 패턴 5가지

SAP Integration Suite가 지원하는 통합 방식:

| # | 패턴 | 설명 | 예시 |
|---|------|------|------|
| 1 | **프로세스 통합** (A2A) | 애플리케이션 간 비즈니스 프로세스 연계 | S/4HANA ↔ SuccessFactors |
| 2 | **API 기반 통합** | REST/OData API 노출·관리 | 타사 앱이 SAP 데이터 조회 |
| 3 | **이벤트 기반 통합** | 실시간 이벤트 발행·구독 | 주문 생성 → 재고 업데이트 이벤트 |
| 4 | **B2B/EDI 통합** | 거래 파트너와 전자문서 교환 | 구매주문서 EDI 교환 |
| 5 | **하이브리드 통합** | 온프레미스 ↔ 클라우드 연결 | On-prem ERP ↔ Cloud CRM |

## SAP BTP와의 관계

Integration Suite는 **SAP BTP의 핵심 서비스** 중 하나입니다:

```
SAP Business Technology Platform (BTP)
├── Application Development   (CAP, Build, SAP Build Apps)
├── Automation                (SAP Build Process Automation)
├── Data & Analytics          (SAP Analytics Cloud, Datasphere)
├── AI                        (SAP AI Core, AI Launchpad)
├── Integration               ← ★ Integration Suite
└── Database Management       (HANA Cloud)
```

BTP의 계정 모델과 글로벌 네트워크를 기반으로 동작하며, 동일한 XSUAA 인증·권한 체계, 동일한 CF/Kyma 런타임을 사용합니다.

## 배포 방식

| 모델 | 설명 |
|------|------|
| **Public Cloud (Multi-tenant)** | SAP가 완전 관리, 가장 일반적인 방식 |
| **Private Cloud (Single-tenant)** | 전용 인스턴스, 규제 산업용 |
| **Edge Integration Cell** | 하이브리드 시나리오용, 온프레미스 에지에서 실행 |
| **Neo 환경 (Legacy)** | 구 SAP Cloud Platform, 현재 마이그레이션 중 |

## 표준 vs 엔터프라이즈

| **Standard** | Cloud Integration + API Management + Integration Advisor (기본) |
| **Enterprise** | Standard + Trading Partner Management + Integration Advisor (고급) + Event Mesh 포함 |

## FSD 기준 전체 Capability 목록

| # | Capability | 설명 |
|---|-----------|------|
| 1 | **Cloud Integration** | End-to-end 프로세스 통합 (iFlow) |
| 2 | **API Management** | API 설계·게시·보안·분석·과금 |
| 3 | **Graph** | SAP 제품군 통합 데이터 그래프 API |
| 4 | **Event Mesh** | 이벤트 기반 Pub-Sub 메시징 |
| 5 | **OData Provisioning** | SAP Gateway 서비스 클라우드 OData 게시 |
| 6 | **Open Connectors** | 150+ Non-SAP SaaS Pre-built 커넥터 |
| 7 | **Integration Advisor** | AI 기반 B2B 인터페이스·매핑 설계 |
| 8 | **Trading Partner Management** | B2B 파트너 프로필·협정·모니터링 |
| 9 | **Integration Assessment** | ISA-M 방법론 기반 통합 랜드스케이프 설계 |
| 10 | **Migration Assessment** | PI/PO 7.5 → Integration Suite 마이그레이션 평가 |
| 11 | **Edge Integration Cell** | On-premise Kubernetes에서 iFlow/API 실행 |
| 12 | **Data Space Integration** | Catena-X/Eclipse Tractus-X 데이터 교환 |
| 13 | (Security Framework) | Tenant Isolation, 암호화, RBAC, 멀웨어 스캔, ISO/SOC |
