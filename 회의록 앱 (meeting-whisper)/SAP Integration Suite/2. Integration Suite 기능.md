## Capability 1: Cloud Integration (구 CPI)

### Integration Content 설계
- **Predefined Content**: SAP 제공 재사용 가능 통합 패키지
- **Integration Flow Designer**: 그래픽 기반 Low-code 통합 설계
- **다중 플랫폼 배포**: Cloud Integration runtime + SAP Process Orchestration 7.5
- **Transport**: CTS+, Cloud Transport Management, 수동 export/import
- **시뮬레이션**: 배포 없이 iFlow 테스트 (trace 지원)

### Message Processing — 전체 기능

#### 변환 (Transformation)
| 기능 | 설명 |
|------|------|
| **Mapping** | 그래픽 에디터, XSLT, Custom Script (XSD/EDMX) |
| **ID Mapping** | 소스→타겟 메시지 ID 매핑 (Exactly Once) |
| **Content Modifier** | 헤더·바디·Exchange 속성 수정 |
| **XML Modifier** | DTD 제거, XML 선언 제거 |
| **Converter** | XML↔JSON, XML↔CSV, XML↔EDI(EDIFACT/ASC-X12) |
| **Decoder** | Base64, GZIP, ZIP, MIME Multipart |
| **Encoder** | Base64, GZIP 등 |
| **Filter** | XPath 기반 노드 추출 |
| **Message Digest** | 페이로드 해시 계산 |
| **Script** | Java/Groovy 커스텀 스크립트 |

#### 외부 호출
| 기능 | 설명 |
|------|------|
| **Request-Reply** | 동기 외부 호출 |
| **Send** | 비동기 전송 (응답 불필요) |
| **Content Enricher** | 외부 데이터 병합 |
| **Poll Enrich** | 폴링 후 메시지 보강 |
| **Process Call** | 로컬 서브프로세스 호출 |
| **Looping Process Call** | 반복 서브프로세스 |
| **Idempotent Process Call** | 중복 메시지 감지·처리 |

#### 라우팅
| 기능 | 설명 |
|------|------|
| **Router** | 조건부 라우팅 (Content-based) |
| **Multicast** | Parallel/Sequential 멀티캐스트 |
| **Splitter** | General, Iterating, PKCS#7, IDoc, EDI, ZIP, TAR |
| **Join** | 여러 경로 메시지 병합 |
| **Gather** | 전략 기반 메시지 결합 |

#### 보안
| 기능 | 설명 |

#### 기타
| 기능 | 설명 |

### API-Based Integration Development

Cloud Integration은 API 우선 개발도 지원:
- **OData API Editor**: SOAP/REST/OData 데이터소스를 OData로 노출
- **REST API Editor**: HTTPS sender 기반
- **SOAP API Editor**: SOAP 1.x sender 기반

### B2B Integration 지원
- **Integration Advisor** 연동
- **Partner Directory**: 파트너 정보 저장·iFlow 파라미터화
- **EDI 표준**: EDIFACT, EANCOM, ASC-X12
- **EDI Splitter/Converter**

### 모니터링 & 운영
- 메시지 처리 모니터링 (trace, log level)
- 통합 콘텐츠 관리 (deploy/undeploy)
- 보안 아티팩트 관리 (credential, PGP, SSH, OAuth2)
- Keystore 관리 (X.509 인증서, 키 페어, SSH 키, 백업/복원)
- Access Policy (메시지 로그 접근 제어)
- JDBC 데이터소스 관리
- 연결 테스트 (TLS, SSH, FTP, SMTP, IMAP, POP3, AMQP, Kafka)
- Store 관리 (Data Stores, Variables, Message Queues, Number Ranges)
- Lock 관리 (중복 처리 방지)
- 데이터 아카이빙 (외부 CMS 연동)
- Resource Consumption 모니터링

### API
- **Java API**: Java 8 표준 라이브러리, XML DOM
- **OData API**: 모니터링 데이터 접근 (OData V2, Basic/OAuth)

## Capability 3: Graph ⭐

> **SAP 공식 FSD에만 있는 Capability**

SAP 제품군의 비즈니스 데이터를 **하나의 통합 데이터 그래프**로 접근:

```
SAP S/4HANA ────┐
SAP Sales Cloud ─┼──→ Graph ──→ 통합 API (단일 엔드포인트)
SuccessFactors ──┘
```

- **단일 API**로 SAP 제품군 전체 데이터 접근
- **의미적 연결**(semantically connected) 데이터 모델
- SAP 파트너·고객 개발자 대상
- 재사용 가능한 확장·클라이언트 앱 개발 비용·복잡성 감소

## Capability 12: Data Space Integration ⭐

Catena-X/Eclipse Tractus-X 기반 데이터 공간:

- **Assets**: 제공할 데이터 자산 정의
- **Policies**: 소비 제한 조건
- **Contract Definitions**: 오퍼 정의
- **Agreements**: 계약 모니터링
- **Company Policies**: 오퍼 요청 규제

> ⚠️ 오픈소스(Eclipse Tractus-X) 기반이므로 SAP가 지속적 제공을 보장하지 않음
