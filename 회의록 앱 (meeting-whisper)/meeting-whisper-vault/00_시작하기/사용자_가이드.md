# 회의록 사용법 안내

작성일: 2026-06-30

최종 갱신: 2026-07-20

소스 기준: `meeting_whisper` `0182231` (2026-07-16)

이 문서는 현재 배포된 Meeting Whisper 기준으로 팀원이 회의록을 생성, 전사, 요약, 공유, 수정하는 방법을 안내하기 위한 사용자용 문서다.

## 접속

운영 화면은 Approuter URL로 접속한다.

```text
https://org-build-tf-joule-1on6yytw-dev-meeting-whisper-approuter.cfapps.jp10.hana.ondemand.com
```

접속하면 회사 계정으로 로그인한다. 로그인 후 우측 상단에서 현재 사용자 프로필과 로그아웃 버튼을 확인할 수 있다.

## 세션 만료와 연장

- Approuter 세션 기준은 180분이다.
- 화면은 마지막 정상 API 통신을 기준으로 만료 약 10분 전에 경고를 표시한다.
- 경고의 `세션 연장`을 누르면 로그인 상태를 다시 확인한다.
- 녹음 중에는 약 30분 간격으로 세션 유지 요청을 보낸다.
- 파일 업로드, 녹음 업로드와 재시도 직전에도 세션을 먼저 확인한다.
- 세션이 실제로 만료됐거나 연장에 실패하면 다시 로그인한 뒤 작업을 계속한다.

세션 경고는 브라우저가 계산한 안내다. 회사 로그인 정책이나 네트워크 상태에 따라 실제 만료 시점은 달라질 수 있다.

## 회의 목록 찾기

목록 화면에서는 다음 방법으로 회의록을 찾을 수 있다.

- 제목, 참여자와 요약·전사 내용 검색
- 완료, 처리 중, 실패 상태 필터
- 카테고리 필터
- 회의 카드의 카테고리 배지를 눌러 같은 카테고리만 보기

검색어나 필터는 현재 URL에 반영되므로 상세 화면에서 목록으로 돌아와도 조건을 유지할 수 있다.

## 새 회의록 만들기

1. 목록 화면에서 `+ 새 회의록`을 누른다.
2. 회의 카테고리를 선택한다. 목록에 없으면 직접 입력할 수 있다.
3. 회의 제목을 입력한다. 제목을 직접 입력하지 않으면 카테고리를 바탕으로 제목이 자동 생성될 수 있다.
4. 참여자와 공유자를 입력한다.
5. 녹음으로 진행하거나 기존 녹음 파일을 업로드한다.

기본 카테고리는 주간회의, 고객미팅, 프로젝트회의, 내부회의, 인터뷰, 내부 세미나 참석이다. 참여자와 공유자는 이름 또는 이메일로 입력할 수 있으며, 기존에 입력했던 값은 자동완성 후보로 나타날 수 있다.

## 권한 기준

현재 배포 기준 권한은 아래와 같다.

| 구분 | 열람 | 제목·카테고리·요약·전사 수정 | 참여자/공유자 관리 | 삭제 |
|---|---:|---:|---:|---:|
| 생성자 | 가능 | 가능 | 가능 | 가능 |
| 참여자 | 가능 | 가능 | 불가 | 불가 |
| 공유자 | 가능 | 불가 | 불가 | 불가 |

삭제는 생성자만 할 수 있다. 참여자는 회의록 내용과 카테고리는 수정할 수 있지만 접근 권한 관리나 삭제는 할 수 없다.

## 녹음으로 전사하기

1. 새 회의록 화면에서 카테고리, 제목과 참여자를 입력한다.
2. `회의 시작`을 누른다.
3. 브라우저 마이크 권한을 허용한다.
4. 회의가 끝나면 종료한다.
5. 원본 녹음의 로컬 백업 여부를 선택한다.
6. 녹음 파일이 업로드되고 전사 대기열에 등록된다.

`다운로드 후 계속`을 선택하면 원본 녹음 파일을 사용자 기기에 저장한 뒤 전사를 계속한다. `저장하지 않고 계속`을 선택해도 업로드와 전사는 진행된다. 이 안내는 브라우저에서 직접 녹음한 경우에만 표시된다.

녹음 중 표시되는 실시간 임시 전사는 브라우저 음성 인식 기능을 이용한 참고용 미리보기다. 임시 전사 내용을 화면에서 수정해도 서버로 전송되지 않으며 최종 전사본이나 회의록 요약에 반영되지 않는다. 최종 전사본은 녹음 종료 후 업로드된 원본 오디오를 CF 전사 Worker가 SAP AI Core STT로 다시 전사한 결과를 기준으로 생성한다. 최종 결과를 고치려면 처리 완료 후 최종 전사본을 수정하고 요약을 다시 생성해야 한다.

## 파일 업로드로 전사하기

1. 새 회의록 화면에서 녹음 파일을 선택한다.
2. 파일을 선택하면 상단 버튼이 `파일 전사 시작`으로 바뀐다.
3. `파일 전사 시작`을 누르면 업로드 후 자동으로 전사 대기열에 등록된다.

지원 파일 예시는 `m4a`, `mp3`, `wav`, `webm`, `mp4` 등이다. 현재 업로드 한도는 150MB다. 150MB를 넘는 ALAC M4A 등은 원본을 보관한 뒤 AAC LC, mono, 32kHz, 64kbps 정도로 변환하는 방법을 권장한다.

## 현재 전사 방식

- 전사 엔진: SAP AI Core STT
- 기본 STT 모델: `gemini-2.5-flash`
- 자동 화자분리: 꺼짐
- Worker 동시 처리 slot: 2개
- 12분을 넘는 오디오는 내부적으로 나눠 처리

전사 모델, 화자 수와 자동 화자분리 설정은 현재 사용자 화면에 노출하지 않는다. 동시에 처리할 수 있는 slot이 모두 사용 중이면 추가 회의록은 대기열에 남는다.

## 처리 상태 이해하기

회의록 처리 중에는 상태가 아래 순서로 바뀐다.

1. 업로드 대기
2. 전사 대기
3. 전사 중
4. 요약 생성 중
5. 완료

여러 사용자가 동시에 파일을 올리면 Worker의 두 처리 slot과 CAP 대기열을 사용한다. 처리 slot이 모두 사용 중이거나 Worker가 일시적으로 바쁘면 새 회의록은 전사 대기 상태로 남고, 화면에 대기 순서와 진행 안내가 표시된다.

## 회의록 화면에서 확인할 수 있는 정보

회의록 상단에는 아래 정보가 표시된다.

- 생성일시
- 생성자
- 회의 카테고리
- 회의 길이
- 참여자
- 전체 처리 시간
- 요약 생성 시간
- dispatch와 Worker 진행 또는 오류 메시지

기존 완료 회의 중 처리 시간 값이 저장되어 있지 않은 회의는 시간이 표시되지 않을 수 있다. 새로 처리되는 회의는 완료 시점에 처리 시간이 저장된다.

## 제목·카테고리와 공유 범위 수정

- 제목은 목록 또는 상세 화면에서 수정할 수 있다.
- 카테고리는 상세 화면에서 변경하면 바로 저장된다.
- 생성자는 참여자와 공유자를 변경할 수 있다.
- 참여자는 회의 내용과 카테고리를 수정할 수 있지만 참여자·공유자 목록은 관리할 수 없다.

## 요약본 수정

요약 탭에서는 아래 항목을 바로 수정할 수 있다.

- 핵심 요약
- 결정 사항
- 액션 아이템
- 안건과 포인트

안건 추가, 액션 아이템 추가, 결정 사항 추가 버튼으로 내용을 더 넣을 수 있다. 수정한 내용은 자동 저장된다.

## 전사본과 화자 수정

전사본 탭에서는 발화 단위로 전사 내용을 수정할 수 있다. 전사 문장을 저장하면 요약이 다시 생성된다.

편집 권한이 있는 사용자는 각 발화의 화자 버튼을 눌러 기존 참여자·화자를 선택하거나 새 화자 이름을 입력할 수 있다. 현재 화면에는 과거의 전체 화자 일괄 변경·삭제 관리 영역은 노출하지 않는다. 발화의 화자만 변경하면 기존 요약은 유지된다.

자동 화자분리는 운영 기본값에서 꺼져 있으므로, 전사 결과에 화자 정보가 없으면 필요한 발화만 수동으로 지정한다.

## 검토 후보 사용

요약본에서 `검토 후보 생성`을 누르면 요약본 안의 이름, 회사·제품명, 기술 용어, 숫자·날짜와 어색한 표현 후보를 생성한다.

검토 후보는 전사본으로 이동하는 기능이 아니라 요약본 내용을 다듬기 위한 추천 기능이다. 후보를 적용하면 요약본에 반영되고, 적용 후 남은 후보 목록도 저장된다.

## 다운로드·복사·Outlook 내보내기

회의록이 완료되면 다음 기능을 사용할 수 있다.

- `Markdown 다운로드`: 요약본을 `.md` 파일로 저장
- `요약 복사`: 요약본을 클립보드에 복사
- `Outlook 앱으로 보내기`: OS 기본 메일 앱에 `[회의록] <회의제목>` 메일 작성 화면 열기
- 전사본 텍스트 저장

Outlook 내보내기는 `mailto:`를 사용한다. 본문은 최대 6,000자이며 Markdown의 굵게·표 같은 서식은 메일 서식으로 변환되지 않는다. 긴 요약은 Markdown 파일을 내려받아 첨부한다.

## 실패했을 때

회의록이 실패 상태가 되면 상세 화면에 오류 메시지가 표시된다.

편집 권한이 있는 사용자에게는 `전사 재시도` 버튼이 표시된다. 원본 오디오가 남아 있으면 다시 전사 대기열에 등록할 수 있다. 이미 원본이 정리된 회의는 파일을 다시 업로드해야 한다.

반복해서 실패하는 경우에는 회의 제목, 생성 시간과 화면의 오류 문구를 운영 담당자에게 전달한다.

## 사용 팁

- 긴 회의는 내부 분할, 전사와 요약 때문에 처리 시간이 오래 걸릴 수 있다.
- 동시에 여러 파일을 올리면 뒤에 올린 회의록은 전사 대기 상태가 될 수 있다.
- 공유자보다 참여자로 추가하면 해당 사용자가 회의록 내용을 수정할 수 있다.
- 생성자만 삭제할 수 있으므로 삭제가 필요하면 생성자에게 요청한다.
- 전사본 문장을 수정하면 요약이 다시 생성되므로 완료 안내를 기다린다.
- 브라우저 직접 녹음은 종료 시 원본을 내려받아 별도로 보관하는 것을 권장한다.

## 현재 운영상 제약

- 자동 화자분리는 꺼져 있으며 전사 모델·화자 수 옵션도 화면에서 선택할 수 없다.
- 대용량 또는 장시간 회의는 처리 시간이 길어질 수 있다.
- 업로드 원본 오디오는 처리 완료 또는 운영 정리 시점에 삭제되므로 원본 파일은 별도로 보관한다.
- 이미 잘못된 raw JSON 형태로 저장된 과거 transcript는 자동으로 교정하지 않는다.
- Outlook 내보내기는 기본 메일 앱 설정과 URL 길이 제한의 영향을 받는다.
