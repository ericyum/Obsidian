# 2026-07-20 세션 관리 및 운영 도구 기준

소스 기준: `meeting_whisper` `0182231` (`main`, 2026-07-16)

이 문서는 2026-07-08~16 사이 소스에 반영됐지만 기존 Vault에서 빠졌던 세션 관리, 목록 UX와 운영용 AI usage·재전사 도구를 정리한다.

## 세션 관리

### 운영값

| 항목 | 현재 값 |
| --- | ---: |
| Approuter `SESSION_TIMEOUT` | `180`분 |
| UI 만료 사전 경고 | 만료 약 `10`분 전 |
| 녹음 중 keep-alive | `30`분 간격 |
| 세션 상태 확인 | `30`초 간격 |

관련 소스:

- `mta.yaml`: Approuter `SESSION_TIMEOUT=180`
- `app/meeting-ui/js/session.js`: 경고, 수동 연장, 만료 modal, 녹음 keep-alive
- `app/meeting-ui/js/api.js`: 정상 API 응답의 session touch와 `401` 만료 event
- `app/meeting-ui/js/app.js`: 파일·녹음·복구 업로드 전 세션 확인

### 사용자 동작

1. UI는 마지막 정상 API 통신 시각을 브라우저 `localStorage`에 기록한다.
2. 세션 만료 약 10분 전에 남은 시간과 `세션 연장` 버튼을 표시한다.
3. 브라우저 녹음을 시작할 때 세션을 확인하고, 녹음 중에는 약 30분 간격으로 keep-alive 요청을 보낸다.
4. 파일 업로드, 녹음 완료 업로드, draft 복구 업로드와 재시도 업로드 직전에도 세션을 확인한다.
5. API가 `401`을 반환하거나 세션 연장이 실패하면 재로그인 안내를 표시한다.

UI 경고는 마지막 정상 API 통신을 기준으로 계산한 보조 안내다. 실제 인증 만료 시점은 XSUAA, IdP와 회사 로그인 정책에 따라 달라질 수 있으므로 운영 smoke에서 함께 확인한다.

## 목록·카테고리 UX

- 제목, 참여자와 요약·전사 내용을 통합 검색한다.
- 완료, 처리 중, 실패 상태를 필터링한다.
- 기본·사용자 정의 카테고리로 필터링한다.
- 검색어와 필터를 URL query에 반영해 상세 화면에서 돌아온 뒤에도 조건을 유지한다.
- 회의 상세의 카테고리는 변경 시 즉시 CAP `updateMeetingCategory` action으로 저장한다.
- polling 중에는 사용자가 보고 있는 상세 화면을 불필요하게 다시 그리지 않도록 보호한다.

## AI Core usage 내보내기

회의별 AI Core 호출 결과, 모델, token, latency와 오류를 로컬 JSON으로 내보낼 수 있다.

```powershell
cds bind --exec --profile hybrid -- node scripts/export-remote-ai-usage.js <meetingId>
```

출력:

```text
data/remote-debug/<meetingId>/ai-usage-logs.json
```

내보내는 주요 필드:

- 호출 source, service와 action
- model/config/request ID
- prompt/completion/total token
- success/error 상태와 오류 메시지
- response preview
- 시작·종료 시각과 latency

이 파일은 장애 분석용 로컬 산출물이다. prompt나 response preview에 회의 내용이 포함될 수 있으므로 외부 공유와 Git 커밋 전에 내용을 확인한다.

## 운영자 강제 재전사

사용자 UI의 `전사 재시도`로 해결되지 않는 경우, 원본 오디오가 남아 있는 회의만 운영 스크립트로 다시 queue에 넣을 수 있다.

```powershell
cds bind --exec --profile hybrid -- node scripts/ops-retry-transcription.js `
  <meetingId> small false https://<meeting-whisper-srv-route>/internal
```

인자:

```text
<meetingId> [small|medium|large-v3-turbo] [true|false] <capInternalBaseUrl>
```

현재 운영 엔진은 AI Core STT이므로 `small|medium|large-v3-turbo` 값은 legacy/local Whisper 호환 metadata다. 실제 운영 STT 모델은 Worker의 `AICORE_STT_MODEL=gemini-2.5-flash` 환경변수가 결정한다.

### 실행 전 확인

- 정확한 meeting ID인지 확인한다.
- `AudioObjects`에 원본 오디오가 남아 있는지 확인한다.
- CAP internal URL이 Approuter가 아닌 srv 직접 route의 `/internal`인지 확인한다.
- 사용자가 편집한 transcript, summary와 review를 보존해야 하는지 먼저 확인한다.

### 스크립트가 변경하는 데이터

이 스크립트는 다음 작업을 수행하므로 되돌리려면 별도 백업이 필요하다.

1. 기존 `Transcripts`, `Summaries`, `ReviewItems` 삭제
2. Meeting을 `queued`, `10%`로 변경하고 오류 초기화
3. 기존 `TranscriptionDispatches` 삭제
4. 새 queued dispatch 생성

운영자가 임의로 DB row를 직접 고치는 대신 이 스크립트를 사용하되, 파생 데이터 삭제가 승인된 회의에만 실행한다.

## 운영 smoke

- 실제 로그인 상태에서 세션 경고와 `세션 연장` 동작
- 30분 이상 녹음 중 keep-alive와 종료 후 업로드
- 만료된 탭에서 파일 업로드 직전 재로그인 안내
- 목록 검색·상태·카테고리 필터와 URL 상태 복원
- 상세 카테고리 변경 후 새로고침 시 저장값 유지
- AI usage export의 meeting ID 필터와 출력 파일 생성
- 원본이 남은 테스트 회의에서 강제 재전사 후 dispatch 진행
