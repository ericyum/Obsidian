## 데이터 모델 (`cap/db/schema.cds`)

### 엔티티 관계도

```
Meetings (중심 — cuid + managed)
│
├── Composition [1:1] → AudioObjects
│   audio: Composition of one AudioObjects on audio.meeting = $self
│
├── Composition [1:1] → Transcripts
│   transcript: Composition of one Transcripts on transcript.meeting = $self
│
├── Composition [1:1] → Summaries
│   summary: Composition of one Summaries on summary.meeting = $self
│
├── Composition [1:N] → ReviewItems
│   reviewItems: Composition of many ReviewItems on reviewItems.meeting = $self
│
└── (Association [1:1]) → TranscriptionDispatches
    (별도 엔티티, meeting Association으로 연결)

AiCoreUsageLogs (독립 — meetingId UUID로 논리 연결)
```

## 핵심 라이브러리 (`cap/srv/lib/`)

### transcription-dispatcher.js

**Dispatcher 상태 머신**:

```
Dispatcher polling (5초 간격)
  ↓
SELECT queued rows WHERE nextAttemptAt ≤ now
  ↓ (limit 2개)
dispatchOne()
  ├─ UPDATE → dispatching
  ├─ POST Worker /transcribe
  ├─ Worker 429 → 다시 queued, backoff (base^attempt)
  ├─ 성공 → dispatched
  └─ 실패 → backoff or maxAttempts 초과 → failed
```

**Backoff 전략**:
```javascript
backoff = base * 2^(attempts-1), 최대 10분
```

### ai-core.js

**두 가지 호출 모드**:

| 작업 | 모드 | 모델 결정 | 환경변수 |
|------|------|----------|----------|
| 요약 | `template` | AI Core 등록 template | `CAP_AICORE_SUMMARY_ORCHESTRATION_MODE=template` |
| 검토 | `direct` | `CAP_LLM_MODEL` | `CAP_LLM_MODEL=anthropic--claude-4.7-opus` |

**요약 파이프라인**:
```
summarizeMeeting(transcript, category, participants)
  ↓
긴 transcript (> CAP_LLM_SUMMARY_CHUNK_CHARS=12000)?
  ├─ YES: chunk 분할 → partial summary → reduce (16,000자)
  └─ NO:  단일 호출
  ↓
normalizeNote() → JSON 정규화
  ↓
noteToMarkdown() → Markdown 변환
  ↓
Summaries 저장 (content + markdown)
```

**카테고리별 guidance**: `CATEGORY_SUMMARY_GUIDANCE`로 카테고리별 요약 강조점 제공.

### meeting-note.js

Transcript/Note 검증 및 변환:

- `validateTranscriptContent()`: segments 구조, timestamp 순서, raw JSON 중첩 차단
- `normalizeNote()`: AI Core 응답을 정규화된 meeting note 구조로 변환
- `noteToMarkdown()`: JSON → Markdown
- `parseTranscriptContent()`: transcript JSON 파싱

### audio-store.js

오디오 저장 및 권한 관리:

- `saveAudioForMeeting()`: DB 또는 Object Store에 저장
- `deleteAudioForMeeting()`: 오디오 삭제
- `assertOwnedMeeting()`: 생성자 확인
- `canAccessAllMeetings()`: Admin 권한 확인

## server.js

CAP 서버 진입점:

```javascript
cds.on("bootstrap", (app) => {
  mountBinaryUploadRoute(app);  // 바이너리 업로드 핸들러

});
```

- `MAX_AUDIO_BYTES` 기반 body parser limit 동적 설정 (최대 157,286,400 bytes ≈ 150MiB)
