# 1. app — 프론트엔드

## 책임

이 폴더는 저장소의 `app` 관련 소스를 파일 단위로 추적한다.

## 문서 목록

- [[app › meeting-ui › index.html 완전 분석]] — Vanilla JS SPA의 화면 골격, 다이얼로그, 입력 요소와 모듈 진입점을 정의한다.
- [[app › meeting-ui › js › api.js 완전 분석]] — MeetingService OData action/entity 호출을 fetch 기반 함수로 캡슐화한다.
- [[app › meeting-ui › js › app.js 완전 분석]] — 프론트엔드 오케스트레이터. 상태, 이벤트, 업로드, polling, 목록/상세 렌더링을 묶는다.
- [[app › meeting-ui › js › drafts.js 완전 분석]] — IndexedDB에 장시간 녹음 chunk와 복구용 메타데이터를 보관한다.
- [[app › meeting-ui › js › recorder.js 완전 분석]] — MediaRecorder와 Web Audio API를 감싸 녹음 blob과 음량 레벨을 만든다.
- [[app › meeting-ui › js › session.js 완전 분석]] — 세션 만료 감지, keep-alive, 탭 간 만료 전파와 업로드 전 세션 확인을 담당한다.
- [[app › meeting-ui › js › ui.js 완전 분석]] — 상태 진행률, 전사, 요약, 회의 목록을 HTML로 변환하는 표현 계층이다.
- [[app › meeting-ui › site.webmanifest 완전 분석]] — 상위 모듈의 실행 또는 설정을 보조하는 파일이다.
- [[app › meeting-ui › styles.css 완전 분석]] — 반응형 레이아웃, 화면 상태, 녹음기, 회의 목록과 상세 화면의 시각 규칙을 담당한다.

## 읽는 법

1. 진입점과 선언형 계약을 먼저 읽는다.
2. 실제 handler 또는 실행 함수를 읽는다.
3. 호출하는 라이브러리와 테스트로 내려간다.
4. 환경 변수는 루트 `.env.example`과 `mta.yaml`의 운영 값을 함께 비교한다.
