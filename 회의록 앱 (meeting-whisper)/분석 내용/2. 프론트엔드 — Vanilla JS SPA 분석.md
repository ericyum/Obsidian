> **분석일**: 2026-07-20
> **기준 소스**: `meeting_whisper` `0182231`

---

## 기술 선택 배경

UI5 대신 **순수 JavaScript SPA**를 선택. 이유:
- 빠른 프로토타이핑과 개발 속도
- 가벼운 번들 크기 (UI5 CDN 불필요)
- 브라우저 API 직접 활용 (MediaRecorder, SpeechRecognition)
- BTP 배포 시 정적 파일만 html5-apps-repo에 업로드

## JS 모듈 상세

### `app.js` — 앱 진입점 & 생명주기

```javascript
// 모듈 import
import { fetchMeetings, ... } from './api.js';
import { renderList, renderResult, ... } from './ui.js';
import { startRecording, ... } from './recorder.js';
import { loadDraft, saveDraft, ... } from './drafts.js';
import { initSession, ... } from './session.js';

// 초기화
initSession();
renderCategoryFilter();
loadDraft();  // 복구 가능한 드래프트 확인

// View 라우팅 (URLSearchParams 기반)
// ?view=list, ?view=entry, ?view=result&id=xxx
```

**주요 이벤트 바인딩**:
| 이벤트 | 타겟 | 액션 |
|--------|------|------|
| `#new-meeting` 클릭 | → `#view-entry` 표시 |
| `#start` 클릭 | → 녹음 시작 → `#view-recording` |
| `#stop` 클릭 | → 녹음 종료 → 업로드 → dispatch |
| `[data-back]` 클릭 | → 목록으로 복귀 (검색·필터 유지) |
| 폴링 타이머 | active meeting 상태 갱신 |

### `api.js` — CAP API 통신

```javascript
// CAP action 호출 패턴
await callAction('uploadAudio', { meetingId, mimeType, size, content });
await callAction('getMeetingStatus', { meetingId });
await callAction('saveTranscript', { meetingId, content, ... });

// 세션 터치 (정상 응답 시마다)
localStorage.setItem('lastApiSuccess', Date.now());

// 401 감지 → session.js에 만료 이벤트 발행
```

### `ui.js` — UI 렌더링

| 함수 | 역할 |
|------|------|
| `renderList(meetings)` | 회의 카드 목록 렌더링 |
| `renderResult(meeting, status)` | 상세 화면 렌더링 (요약 탭/전사 탭) |
| `renderSummaryTab(meeting)` | 요약 JSON 파싱 → 편집 가능 UI |
| `renderTranscriptTab(meeting)` | 전사 segments → 발화별 편집 UI |
| `renderReviewItems(items)` | 검토 후보 리스트 |
| `updateSteps(status)` | 진행 상태 표시줄 |

**요약 편집 UI**: summary, decisions[], action_items[], topics[]를 각각 편집 가능한 필드로 분리.

**전사 편집 UI**: segments 배열을 타임라인 형태로 표시, 각 발화에 화자 변경 버튼.

### `recorder.js` — 브라우저 녹음

```javascript
// MediaRecorder API
const mediaStream = await navigator.mediaDevices.getUserMedia({ audio: true });
const recorder = new MediaRecorder(mediaStream, { mimeType: 'audio/webm' });

// 녹음 중 실시간 임시 전사
const recognition = new webkitSpeechRecognition();  // 또는 SpeechRecognition
recognition.continuous = true;
recognition.interimResults = true;
// → #live-transcript에 표시 (서버 전송 안 함)

// 타이머
setInterval(() => updateTimer(), 1000);

// 종료 시 원본 백업 옵션 (download link)
```

### `drafts.js` — 로컬 드래프트

```javascript
// localStorage 기반
saveDraft({ category, title, participants, sharedWith, file? });
const draft = loadDraft();  // 복구 가능한 드래프트 확인

// draft-banner: "이전에 작성 중이던 회의록이 있습니다. 복구하시겠습니까?"
```

### `session.js` — 세션 관리

```javascript
// Approuter SESSION_TIMEOUT = 180분
const SESSION_TIMEOUT_MIN = 180;
const WARNING_BEFORE_MIN = 10;  // 만료 10분 전 경고

// 마지막 API 성공 시간 기준 계산
const lastSuccess = localStorage.getItem('lastApiSuccess');
const remaining = SESSION_TIMEOUT_MIN * 60 * 1000 - (Date.now() - lastSuccess);

if (remaining < WARNING_BEFORE_MIN * 60 * 1000) {
    showSessionWarning(remaining);  // 모달: "세션이 곧 만료됩니다. 연장하시겠습니까?"
}

// 녹음 중 keep-alive: 30분 간격으로 /api/... 호출
// 업로드 직전 세션 확인 후 진행
```


## URL 상태 관리

목록 화면의 검색어·필터는 URL query parameter로 유지:

```
/?view=list&search=프로젝트&stage=active&category=주간회의
```

상세 화면에서 목록으로 돌아올 때 `history.back()`으로 복원.
