# 3. cap — 데이터와 백엔드

## 책임

이 폴더는 저장소의 `cap` 관련 소스를 파일 단위로 추적한다.

## 문서 목록

- [[cap › db › package.json 완전 분석]] — 상위 모듈의 실행 또는 설정을 보조하는 파일이다.
- [[cap › db › schema.cds 완전 분석]] — Meetings를 중심으로 음원, dispatch, 전사, 요약, 검토, AI 사용량을 모델링한다.
- [[cap › srv › ai-usage-service.cds 완전 분석]] — 관리자에게 AI Core 호출 기록을 읽기 전용으로 노출한다.
- [[cap › srv › lib › ai-core.js 완전 분석]] — 전사를 chunk/reduce 방식으로 요약하고 검토 항목을 생성하며 AI Core 응답 JSON을 검증·복구한다.
- [[cap › srv › lib › ai-usage-log.js 완전 분석]] — 상위 모듈의 실행 또는 설정을 보조하는 파일이다.
- [[cap › srv › lib › audio-store.js 완전 분석]] — 회의 소유권을 확인하고 Object Store 음원 메타데이터와 payload를 관리한다.
- [[cap › srv › lib › audio.js 완전 분석]] — 상위 모듈의 실행 또는 설정을 보조하는 파일이다.
- [[cap › srv › lib › binary-upload-route.js 완전 분석]] — 상위 모듈의 실행 또는 설정을 보조하는 파일이다.
- [[cap › srv › lib › cleanup.js 완전 분석]] — 만료 음원, stale dispatch/worker/meeting을 주기적으로 복구 또는 실패 처리한다.
- [[cap › srv › lib › meeting-note.js 완전 분석]] — 전사 JSON 검증, 텍스트 chunk화, 요약 정규화와 Markdown 출력을 담당한다.
- [[cap › srv › lib › object-store.js 완전 분석]] — VCAP 바인딩을 해석해 S3 호환 Object Store 업로드·서명 URL·삭제를 수행한다.
- [[cap › srv › lib › status.js 완전 분석]] — 상위 모듈의 실행 또는 설정을 보조하는 파일이다.
- [[cap › srv › lib › transcription-dispatcher.js 완전 분석]] — DB queue를 polling하고 동시성·재시도·backoff·lock을 제어한다.
- [[cap › srv › lib › worker-client.js 완전 분석]] — CAP에서 Worker /transcribe 엔드포인트로 작업을 넘긴다.
- [[cap › srv › meeting-service.cds 완전 분석]] — 사용자용 OData 서비스의 projection, complex type, action 계약을 정의한다.
- [[cap › srv › meeting-service.js 완전 분석]] — 소유권, 업로드, dispatch, 상태 조회, 요약·검토, 편집·삭제의 실제 비즈니스 로직이다.
- [[cap › srv › worker-internal-service.cds 완전 분석]] — Worker가 claim, heartbeat, audio 조회, 결과 저장, 실패 보고에 쓰는 내부 계약이다.
- [[cap › srv › worker-internal-service.js 완전 분석]] — Worker callback을 검증하고 상태 전이·전사 저장·비동기 요약을 수행한다.

## 읽는 법

1. 진입점과 선언형 계약을 먼저 읽는다.
2. 실제 handler 또는 실행 함수를 읽는다.
3. 호출하는 라이브러리와 테스트로 내려간다.
4. 환경 변수는 루트 `.env.example`과 `mta.yaml`의 운영 값을 함께 비교한다.
