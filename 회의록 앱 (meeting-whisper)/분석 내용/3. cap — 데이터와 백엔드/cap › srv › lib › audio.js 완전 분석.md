# audio.js 완전 분석

> 소스: `cap/srv/lib/audio.js`  
> 역할: 상위 모듈의 실행 또는 설정을 보조하는 파일이다.

## 1. 이 파일이 있는 이유

상위 모듈의 실행 또는 설정을 보조하는 파일이다.

이 파일은 **3. cap — 데이터와 백엔드** 영역에 속한다. 독립적으로 보기보다 아래의 호출자·의존 대상·데이터 계약과 함께 읽어야 한다.

## 2. 파일 개요

| 항목 | 값 |
|---|---|
| 경로 | `cap/srv/lib/audio.js` |
| 형식 | `javascript` |
| 전체 줄 수 | 73 |
| 주석 줄 수 | 0 |
| 주요 심볼 수 | 7 |
| 환경 변수 참조 수 | 0 |

## 4. 주요 선언과 책임

| 선언 | 읽을 때 확인할 점 |

| `normalizeAudioContent` | 입력, 상태 변경, 반환값과 실패 경로를 확인한다. |
| `streamToBuffer` | 입력, 상태 변경, 반환값과 실패 경로를 확인한다. |

| `getMaxAudioBytes` | 입력, 상태 변경, 반환값과 실패 경로를 확인한다. |

## 8. 변경 시 영향 범위
+
- 공개 계약, 상태명, 환경 변수, URL을 바꾸면 호출 측과 테스트를 함께 수정해야 한다.
- 저장 데이터의 형식이 바뀌면 기존 HANA 행과 진행 중인 작업의 호환성을 확인한다.
- 인증·소유권 검사는 편의상 우회하면 안 된다. Approuter 인증과 CAP 역할 검사는 서로 다른 계층이다.
