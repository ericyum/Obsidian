# server.js 완전 분석

> 소스: `server.js`  
> 역할: CAP 서버 부트스트랩. 바이너리 업로드 라우트와 정적 UI를 Express에 연결하고 요청 본문 크기를 조정한다.

## 1. 이 파일이 있는 이유

CAP 서버 부트스트랩. 바이너리 업로드 라우트와 정적 UI를 Express에 연결하고 요청 본문 크기를 조정한다.

이 파일은 **9. docs와 루트 설정** 영역에 속한다. 독립적으로 보기보다 아래의 호출자·의존 대상·데이터 계약과 함께 읽어야 한다.

- `MAX_AUDIO_BYTES`: 배포 환경과 로컬 실행 값이 달라질 수 있으므로 `mta.yaml`, `.env.example`과 함께 확인한다.
- `CDS_BODY_PARSER_LIMIT`: 배포 환경과 로컬 실행 값이 달라질 수 있으므로 `mta.yaml`, `.env.example`과 함께 확인한다.
- `CDS_BODY_PARSER_LIMIT_BYTES`: 배포 환경과 로컬 실행 값이 달라질 수 있으므로 `mta.yaml`, `.env.example`과 함께 확인한다.

## 7. 코드 흐름 상세

### 7.1 `requireBinaryUploadRoute`

- 위치: 6~24행
- 의미: 이 블록의 전제 조건, 외부 호출, DB/메모리 상태 변경, 예외 처리 순서가 핵심이다.

### 7.2 `positiveNumber`

- 위치: 25~60행
- 의미: 이 블록의 전제 조건, 외부 호출, DB/메모리 상태 변경, 예외 처리 순서가 핵심이다.

## 8. 변경 시 영향 범위
+
- 공개 계약, 상태명, 환경 변수, URL을 바꾸면 호출 측과 테스트를 함께 수정해야 한다.
- 저장 데이터의 형식이 바뀌면 기존 HANA 행과 진행 중인 작업의 호환성을 확인한다.
- 인증·소유권 검사는 편의상 우회하면 안 된다. Approuter 인증과 CAP 역할 검사는 서로 다른 계층이다.
