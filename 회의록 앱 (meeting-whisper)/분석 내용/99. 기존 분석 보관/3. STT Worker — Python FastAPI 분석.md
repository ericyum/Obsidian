### `/transcribe` 처리 흐름

```
① 인증: TRANSCRIPTION_WORKER_TOKEN (Bearer) 검증
   401 → token 없음
   403 → token 불일치

② 동시성 체크: asyncio.Semaphore
   full → 429 "Worker is busy; retry later"

③ CAP claimTranscription(meetingId, workerRunId)
   502 → claim 실패

④ Background Task 실행 (_run_transcription_job)
   ├─ Timeout 감시 (WORKER_JOB_TIMEOUT_SEC=5400s)
   │   └─ 초과 → fail + os._exit(124) (워커 재시작)
   │
   └─ _process_transcription()
       ├─ Stage: downloading (25%)
       │   └─ CAP getAudio() → AudioPayload
       │       ├─ downloadUrl → httpx stream 다운로드
       │       └─ contentBase64 → base64 decode → temp file
       │
       ├─ Stage: transcribing (30% → 74%, progress callback)
       │   └─ run_transcription()
       │       ├─ 실제/예상 길이 > 720초 → chunk 분할
       │       ├─ 각 chunk → AI Core STT
       │       │   ├─ 실패 → 2회 재시도
       │       │   └─ 계속 실패 → 6분 소분할
       │       ├─ chunk 병합 (timestamp offset)
       │       └─ on_progress callback → heartbeat
       │
       ├─ 검증: _validate_transcript_for_storage()
       │   ├─ JSON object 확인
       │   ├─ segments 배열 확인
       │   ├─ text 존재 확인
       │   ├─ start/end 유효성 (finite, non-negative, end ≥ start)
       │   ├─ timestamp 순서 확인
       │   ├─ duration 범위 확인 (tolerance: 5% or 30s)
       │   └─ raw JSON 중첩 차단 (_looks_like_transcript_json)
       │
       ├─ AI usage 로깅: _pop_ai_usage_logs() + logAiUsage()
       │
       └─ Stage: saving (75%)
           └─ CAP saveTranscript(meetingId, workerRunId, language, content)

⑤ 완료 → Semaphore 해제
```

### Heartbeat 메커니즘

```python
# 정기 heartbeat (60초 간격)
_heartbeat_loop(): stop 신호까지 60초마다 stage="transcribing" heartbeat

# 진행률 heartbeat (10% 이상 변경 시)
_progress_heartbeat_loop(): progress_queue에서 이벤트 수신 → 즉시 heartbeat
```

### 오류 처리

```python
try:
    await _process_transcription(...)
except Exception as error:
    await _mark_failed(cap, meeting_id, error, worker_run_id)
    # CAP failTranscription() 호출
    # 오류 메시지: base64 data URL redaction 처리
```

### 오디오 다운로드 전략

| 크기 | 방식 |
|------|------|
| ≤ 25MB (`AICORE_STT_INLINE_MAX_AUDIO_BYTES`) | contentBase64 (인라인) |
| > 25MB | signed download URL → httpx stream |
| MIME 타입 추론 | Magic bytes로 WebM/WAV/MP4/MP3 판별 |

## `aicore_stt.py` — AI Core STT

### chunk 분할 로직

```python
def should_chunk(audio_duration_sec, expected_duration_sec):
    # 실제 측정 길이 또는 CAP 전달 예상 길이 중 하나가 720초 초과
    if audio_duration_sec > 720 or expected_duration_sec > 720:
        return True
    return False
```

### STT 호출 & 재시도

```python
async def transcribe_chunk(chunk_path, ...):
    for attempt in range(max_attempts):  # 2회
        try:
            result = await ai_core_stt_request(...)
            validate_stt_response(result)
            return result
        except Exception:
            if attempt == max_attempts - 1:
                # 소분할 시도 (6분 단위)
                return await transcribe_subchunks(...)
```
